Speaker diarization — who spoke when

Diarization answers a different question from "is someone talking?": it works out how many people are speaking and which of them is speaking at each moment — including where two voices overlap. This is pyannote's segmentation model, ~6 MB, running on the CPU in your browser, turning audio into a colour-coded speaker timeline with nothing uploaded.

Why it matters

Diarization is the layer that turns a transcript into a conversation: it tells you who said what. Paired with a speech recogniser, it's how you get "Speaker 1: … / Speaker 2: …" from a raw recording — and doing it on-device keeps the audio private.

  • Meeting notes — attribute each line of a transcript to a speaker, locally, with no recording uploaded.
  • Interview & podcast tooling — auto-generate speaker-labelled transcripts and chapter markers.
  • Turn-taking analytics — measure who talked how long, and where people talked over each other.
  • Overlap detection — flag crosstalk for call-quality or accessibility captioning.

How the API works

It's an audio-frame-classification model with a diarization post-processor. Decode audio to the model's sample rate, run it, and let the processor turn the per-frame powerset logits into speaker segments: 

import { AutoProcessor, AutoModelForAudioFrameClassification, read_audio }
  from "@huggingface/transformers";

const model_id = "onnx-community/pyannote-segmentation-3.0";
const model = await AutoModelForAudioFrameClassification.from_pretrained(model_id);
const processor = await AutoProcessor.from_pretrained(model_id);

const audio = await read_audio(url, processor.feature_extractor.config.sampling_rate); // 16 kHz mono
const inputs = await processor(audio);
const { logits } = await model(inputs);            // [1, frames, 7]  (powerset of up to 3 speakers)

const result = processor.post_process_speaker_diarization(logits, audio.length);
// [ [ { id: 0, start: 0, end: 1.05, confidence: 0.82 }, { id: 2, start: 1.05, end: 2.34, … }, … ] ]

This page runs exactly that in a Web Worker (decoding audio with Web Audio first), and draws the segments as a colour-coded speaker timeline.
