ResNet-50 — the network that made deep deep learning

Same job as the ViT and ConvNeXt demos — name what's in an image across ImageNet's 1,000 classes — but this is the 2015 original that made 50-layer networks trainable at all. ResNet's trick is the skip connection: each block learns a small residual on top of an identity shortcut, so gradients flow through dozens of layers without vanishing. Drop an image, read the labels, then run the backbone lineage race — ResNet 2015 vs ConvNeXt 2022 vs ViT — on one picture.

Backbone lineage race — ResNet vs ConvNeXt vs ViT

The whole point of this page: run the same image through three generations of ImageNet backbone and watch how the field evolved. ResNet-50 (2015) is the residual CNN that everything else was measured against; ConvNeXt (2022) is a ResNet modernised block-by-block to match transformers; ViT is the pure transformer that dropped convolutions entirely. Same 1,000 classes, three very different machines. Where all three agree, trust the label; where they split, the image is genuinely hard. Loading ConvNeXt adds ~29 MB and ViT ~88 MB the first time.

The residual block & skip connection — why 50 layers works

Before ResNet, stacking more layers made networks worse, not better — gradients vanished on the way back and deep plain nets were harder to optimise than shallow ones. The fix is almost embarrassingly simple: instead of asking a block to learn a full transformation H(x), ask it to learn the residual F(x) = H(x) − x, then add the untouched input back: output = F(x) + x. That + x is the skip (identity) connection. It gives gradients a direct highway to every earlier layer, so a 50-layer network trains as easily as a shallow one, and any block that has nothing useful to add can fall back to the identity for free. ResNet-50 stacks bottleneck blocks — a 1×1 conv squeezes the channels, a 3×3 conv does the spatial work, a 1×1 conv expands them back — across four stages (3, 4, 6, 3 blocks). This one idea unlocked networks with hundreds of layers and is still wired into ConvNeXt, every transformer's residual stream, and most modern architectures. The race above lets you watch how much (or how little) accuracy the seven years since bought on any given image.

Why it matters

ResNet-50 is the reference backbone of computer vision — the model almost every detector, segmenter, and feature extractor of the late 2010s was built on top of, and the number every new architecture still reports against. It's small, fast on the WASM backend, extremely well understood, and its ImageNet features transfer to almost any vision task.

  • A dependable on-device tagger — the classic, battle-tested choice for auto-tagging when you want a known quantity, not the newest thing.
  • A transfer-learning starting point — ResNet-50 features are the default warm-start for custom classifiers; running it in the browser lets you prototype that pipeline with nothing installed.
  • An architecture baseline — race it against ConvNeXt and ViT to make the "is the new model actually better here?" question concrete on your own images.
  • Private-by-default vision — everything runs in the tab; no image leaves the device.

Use cases

Classify a sample and read the top-k — feel the original residual net name what's in an image.

Wild

Texture vs shape: strip colour and warp the image and watch a CNN's texture bias show.

Multi-model

Three-way vote: ResNet + ConvNeXt + ViT on one image for a robust majority answer.

How the API works

The convenience path is two lines with Transformers.js:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "image-classification",
  "Xenova/resnet-50",
  { dtype: "q8" },
); // downloads + caches ~26 MB once

const output = await classify(imageURL, { top_k: 5 });
// → [{ label: "tiger cat", score: 0.96 }, { label: "tabby, tabby cat", score: 0.01 }, …]

This page goes one level deeper for "See inside": instead of the pipeline it runs the model's own processor then model() in a single forward pass, so it can read the raw logits tensor (all 1,000 classes) and compute the softmax, entropy, and margin itself. Same weights, same result — it just keeps the intermediate numbers. Inference runs in a Web Worker so the page never janks.
