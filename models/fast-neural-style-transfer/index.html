Fast Neural Style Transfer — repaint a photo as art

Give this model any photo and it repaints it in a famous artistic style — the swirls of Rain Princess, the mosaic tiles of Mosaic, the pointillist dots of Pointilism. These are the classic fast neural style transfer networks (Johnson et al. 2016) — the feed-forward models that first made real-time style transfer possible: one tiny generator per style, each trained to reproduce one painting's look. Runs entirely on your device via ONNX Runtime Web; the photo never leaves the tab.

See inside

Each style is a separate trained network, not a filter with parameters. Johnson's method trains one small feed-forward generator per target painting, so at run time a single pass produces the style — no slow optimisation loop like the original Gatys method. That's why it's called fast style transfer, and why each style is its own ~6.6 MB model. Switch styles below and re-run the same photo to see how differently each network paints it.

How to read it

Notice the model doesn't just recolour — it imposes the target painting's brushwork: mosaic tiles, pointillist dots, swirling strokes. That structural texture, learned from one artwork, is what makes it read as a painting rather than a filtered photo. The fixed 224² network keeps it fast; the result is resized to your photo's shape.

Why it matters

Turning a photo into a painting on-device makes art from any picture, privately and free per image:

  • Creative & social apps — a one-tap "paint my photo" that runs client-side, no server, no per-image cost.
  • Art & print — turn holiday snaps into stylised prints or posters entirely in the browser.
  • Backgrounds & assets — restyle stock photos into painterly backdrops for slides, sites or games.
  • Teaching — a live, tangible demo of the paper that kicked off real-time neural style transfer.

These are the original fixed-style networks — one model per look. They render a stylistic interpretation, and the fixed 224² input means very fine detail is simplified.

How the API works

transformers.js has no style-transfer class and no neural-style task, so we run the ONNX graph directly with ONNX Runtime Web. Note the fixed 224×224 input and the 0–255 range:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(styleBytes, { executionProviders: ["wasm"] });

// photo stretched to 224×224 → [1,3,224,224] float32 in [0,255] (NCHW, un-normalised bytes)
const input = new ort.Tensor("float32", toNCHW255(photo, 224, 224), [1, 3, 224, 224]);
const { output1 } = await session.run({ input1: input });

// output1 is [1,3,224,224] in ~[0,255] → clamp to RGBA and resize back to the photo's aspect
const painting = tensorToImage(output1.data, 224, 224);

Inference and the whole output composite run in a Web Worker so the UI never blocks; each ~6.6 MB style is fetched through Cache Storage the first time you pick it, so a returning visit auto-initialises with no re-download.
