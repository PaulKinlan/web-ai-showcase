VAD → embed (Silero → WavLM-SV)

Silence and noise dilute a speaker embedding. This chains two models on-device: Silero VAD finds the speech regions, we keep only those samples, then WavLM-SV embeds the cleaned speech. Both the raw and VAD-trimmed cosine are shown side by side — the gap is largest on silence- or noise-heavy clips; on already-clean speech it's small (and imperfect trimming can even nudge it down). Real numbers, no cherry-picking.

Both models run in the browser — the audio and embeddings never leave the tab. VAD trims leading/trailing/inter-word non-speech so the speaker embedding sees voice, not room tone.

Why pre-process with VAD

A speaker embedder is trained on speech; feed it silence, breaths, or background noise and those frames pull the embedding toward a generic "room" point, blurring the speaker. Running a tiny VAD first — cheap, on-device — keeps only the frames that carry the voice, which helps most when a clip is padded with silence or room tone. On clips that are already wall-to-wall speech the embedding barely moves. Detect-then-embed is the same idea as detect-then-read in OCR: let a small specialist clean the input for the bigger model.
