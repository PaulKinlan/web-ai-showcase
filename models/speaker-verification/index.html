Speaker verification — same voice, or not?

WavLM-SV turns a voice clip into a 512-number embedding — a point in "voice space" where the same speaker's clips land close together and different speakers land apart. Compare two clips and you get a similarity signal: likely the same speaker, or likely not. It runs entirely on your device — the audio and the embeddings never leave the tab. This is voice similarity, not identification, lookup, or surveillance.

Why it matters

A speaker embedding is the building block behind "is this the same voice?" — and doing it on-device means the biometric never touches a server. Once a voice is a 512-number point you can compare, cluster, and threshold, all client-side:

  • Private voice-note organising — group recordings by who's speaking without uploading a second of audio.
  • Consent-first re-auth signal — a "does this sound like the enrolled user?" hint that stays a signal, with the real decision made by a proper, liveness-checked system.
  • Diarisation seed — tell "who spoke" apart across clips of a meeting locally.
  • Look-alike & spoof study — measure how close two voices are, and see where the model is fooled (the Wild demo).

These are similarity tools. Real voice authentication needs consent, liveness / anti-spoofing, bias auditing, and calibration this demo deliberately does not provide.

Use cases

Basics

Two clips in, one similarity number out — see same vs different in "voice space".

Practical

On-device "voice unlock" concept — enrol a reference voice, then check a probe against a threshold (a signal, never a verdict).

Wild

Impersonation & look-alikes — find the different speakers the model scores as suspiciously close.

Multi-model

VAD → embed: trim silence with voice-activity detection first, then embed only the speech.

How the API works

transformers.js registers WavLMForXVector but has no audio-xvector pipeline, so we load the model + processor directly. The two things a pipeline would hide — feature extraction and L2-normalizing the output — are a few lines:

import { AutoModelForXVector, AutoProcessor } from "@huggingface/transformers";

const processor = await AutoProcessor.from_pretrained("Xenova/wavlm-base-plus-sv");
const model = await AutoModelForXVector.from_pretrained("Xenova/wavlm-base-plus-sv", { dtype: "q8" });

// audioA / audioB are 16 kHz mono Float32Arrays (decoded in-page)
const { embeddings: eA } = await model(await processor(audioA));
const { embeddings: eB } = await model(await processor(audioB));

const similarity = cosine(l2norm(eA.data), l2norm(eB.data)); // ∈ [-1, 1]; ≥0.86 ⇒ likely same speaker

Both embeddings are computed in a Web Worker so the UI never blocks, and the model is cached by Transformers.js, so a returning visit auto-initialises with no re-download.
