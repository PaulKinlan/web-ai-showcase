LongT5 — the long-context gist machine

A different summarizer family from DistilBART: LongT5 uses transient-global attention to read far longer inputs, and it was pretrained with Pegasus's own gap-sentence objective — the closest browser-runnable relative of the Pegasus/XSum "one-sentence gist" style. Paste something and watch the gist stream in, token by token. Nothing leaves your device.

A quick, honest note on the model

The original ask was Pegasus XSum, but it has no transformers.js/ONNX build, so it can't run in a browser today. LongT5 is the closest distinct, runnable family: same Pegasus-style gap-sentence pretraining, plus long-input attention DistilBART lacks. This is the base checkpoint (not a summarization fine-tune), so its gists lean extractive — it tends to lift and lightly compress the single most salient sentence rather than freely rewrite. That's a real, useful behaviour; it just isn't the heavy abstraction a fine-tuned Pegasus would give.

Why it matters

Summarization turns a wall of text into something you can act on in seconds — and doing it on-device means you can summarize private, sensitive, or offline content without shipping a word to a server. LongT5's long-context attention makes it a natural fit for the big inputs a ~1024-token model can't take in one pass.

  • Headline / TL;DR generation — squeeze an article down to a one-line gist for a feed or notification.
  • Long-document digests — reports, transcripts, and threads that overflow a short-context summarizer.
  • Private, offline recaps — summarize confidential docs with no API key and no per-call bill.
  • Pipeline stage — pair with speech-to-text to go from a recording straight to a gist.

How the API works

Summarization is one pipeline call; length is controlled with generation parameters:

import { pipeline } from "@huggingface/transformers";

const summarizer = await pipeline(
  "summarization",
  "Xenova/long-t5-tglobal-base",
); // downloads + caches ~335 MB once

const out = await summarizer(longText, {
  max_new_tokens: 45,
  min_new_tokens: 10,
  num_beams: 1,          // greedy; raise for tighter (slower) output
  no_repeat_ngram_size: 3,
});
// → [{ summary_text: "…a concise gist…" }]

For the live token stream and the decode timeline, this page passes a TextStreamer so it can timestamp each generated token — all inside a Web Worker so the UI never janks.
