IBM Granite 3.0 2B Instruct — an enterprise-tuned LLM in your browser

Granite is IBM's open model family, built and licensed (Apache-2.0) for enterprise work: structured extraction, retrieval-augmented generation, tool/function calling and summarisation over business data. This page runs the 2B Instruct model entirely in your browser via Transformers.js — it takes the WebGPU fast path (q4f16) when your device has a GPU and falls back to the universal WebAssembly/CPU path (q8) otherwise. Chat with it, steer it with a system prompt and sampling controls, and watch the real probability distribution behind every token. Nothing is uploaded; there is no API key and no server.

A distinct family from the other chat models here (Qwen2, Llama, Gemma, Mistral, Falcon3): Granite is IBM's own granite architecture (GraniteForCausalLM), tuned for the kinds of governed, on-prem tasks enterprises actually ship.

A 2B model is heavier than the sub-1B chat models here — on the WASM/CPU path it's a ~2.5 GB download and streams at a modest pace. On a machine with a WebGPU adapter it uses the smaller, much faster q4f16 build automatically. Both paths produce real, on-device output.

Why it matters

Granite was designed for the enterprise problem the big consumer chatbots don't solve: governed, auditable, commercially-licensed models you can run where the data lives. Running the 2B Instruct in the browser takes that to its logical end — the data never leaves the device at all. Apache-2.0 means no gating and no per-token bill; the enterprise tuning means it's genuinely good at the unglamorous tasks businesses actually pay for.

  • On-device document processing — extract fields, classify, and summarise sensitive documents without them ever hitting a server.
  • Grounded RAG — answer from a user's own policies/contracts with citations, all in the tab (see the multi-model demo).
  • Structured output — reliable JSON for forms, APIs and workflows, at low temperature.
  • Commercial-safe — Apache-2.0 with no gating makes it straightforward to embed in a product.

How the API works

It's the standard Transformers.js text-generation pipeline — pass a messages array (the pipeline applies Granite's chat template), a TextStreamer for token-by-token output, and your sampling settings. The only device-specific choice is device + dtype, which this page picks by probing for a real WebGPU adapter: 

import { pipeline, TextStreamer } from "@huggingface/transformers";

// Probe for a real adapter — WebGPU (q4f16) for a 2B model, else universal WASM (q8) on the CPU.
const adapter = navigator.gpu && await navigator.gpu.requestAdapter().catch(() => null);
const [device, dtype] = adapter ? ["webgpu", "q4f16"] : ["wasm", "q8"];

const generator = await pipeline(
  "text-generation", "onnx-community/granite-3.0-2b-instruct", { device, dtype },
);

const messages = [
  { role: "system", content: "You are a precise enterprise assistant." },
  { role: "user", content: "Classify this ticket as billing/bug/feature: 'charged twice'." },
];

const streamer = new TextStreamer(generator.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });

await generator(messages, {
  max_new_tokens: 220, do_sample: true, temperature: 0.4, top_p: 0.9, streamer,
});

The "see inside" panel runs one raw forward pass — generator.model(generator.tokenizer(prompt)) — reads the last position's logits, softmaxes them, and shows the real top-k. Everything runs in a Web Worker so generation never blocks the page; the loader auto-inits an already-cached copy and never silently re-downloads.
