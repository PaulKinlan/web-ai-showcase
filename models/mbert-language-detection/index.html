mBERT language detection — what language is this, across 201?

Paste any text and this model tells you which of 201 languages it's written in, with a confidence score. It's a fine-tune of a multilingual BERT encoder with a classification head over the FLORES-200 label set (codes like fra_Latn, zho_Hans), so it reads the actual linguistic signal — script, morphology, function words — rather than matching a keyword list. A distinct model and a much wider net than the 20-language XLM-RoBERTa demo: different architecture, ten times the languages. Nothing leaves your device.

See inside — the probability distribution

The model doesn't just pick a language; the classification head emits a real logit per class ([1, 201]), softmaxed into a probability over all 201. A confident detection puts almost all the mass on one bar. On short or mixed text — or between closely related languages that share a script — the mass spreads; that spread is the model telling you it's unsure, and it's genuinely useful signal. Why it matters

Language identification is the quiet first step in almost every multilingual text pipeline. Before you can translate, moderate, route, or index text, you have to know what language it's in — and you often have to do it on a single sentence, a search query, or a chat message. Running it on-device means you can branch instantly with zero latency and without sending user text to a server.

  • Route to the right model — pick the correct translator, spell-checker, or moderation model per message.
  • Pre-filter a dataset — keep only the languages you support, or split a mixed corpus by language before processing.
  • Set the UI language — infer a user's language from what they type and adapt on the fly.
  • Tag user-generated content — label reviews, comments, or support tickets so the right team or workflow picks them up.

How the API works

It's a plain text-classification pipeline; ask for all labels to get the full distribution:

import { pipeline } from "@huggingface/transformers";

const detect = await pipeline(
  "text-classification",
  "onnx-community/language_detection-ONNX",
  { dtype: "q8" },
);

const out = await detect("Bonjour, comment allez-vous ?", { top_k: 201 });
// → [{ label: "fra_Latn", score: 0.998 }, { label: "bam_Latn", score: 0.0003 }, … ]
// out[0].label is the detected FLORES-200 code; the rest is the confidence distribution.

The model outputs a fixed label set (the 201 FLORES-200 codes). Passing top_k: 201 returns every language's probability, sorted — that's what powers the "see inside" bars. All inference runs in a Web Worker so detection never janks the page.
