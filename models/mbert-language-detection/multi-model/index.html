Multi-model: detect the language, then translate

Language detection is the router. Here it detects the language of your text, then chooses the matching MarianMT model to translate it into English — French text loads the French→English model, German loads the German model, and so on. Only the translator you actually need is downloaded. Both stages run locally, so you can build "understand any message" flows with nothing sent to a server.

Bundled translators cover French, German, Spanish, Italian, Russian, Dutch and Chinese into English; English text is passed through, and any other detected language reports that no translator is bundled for it — an honest "I know the language but don't have that model" rather than a wrong guess. In production you'd map every detected language to its own model the same way.
