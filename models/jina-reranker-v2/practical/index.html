Practical: two-stage retrieval for multilingual RAG

The pattern that powers production search and RAG: a cheap first stage returns a shortlist, then the cross-encoder reranks it for real precision. Here the knowledge base is mixed-language — English, Chinese, Spanish, French, German — and jina-reranker-v2 finds the passage that actually answers your English query, whatever language it's written in. In a real system your first stage (a keyword index or a multilingual embedder) would return dozens of candidates like these; the reranker is the precision filter that decides which one the LLM actually reads. Because v2 is multilingual, you keep one reranker for every locale — the top result here is often not in English.
