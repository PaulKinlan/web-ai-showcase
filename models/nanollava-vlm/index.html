nanoLLaVA — a pocket-sized LLaVA that talks about images

nanoLLaVA is a ~1B-parameter LLaVA-style vision-language model: a SigLIP vision encoder bolted onto a small Qwen1.5 language model. Give it an image, a system persona, and a question — and it answers in a chat format, streaming token by token on your GPU with nothing uploaded.

Why it matters

nanoLLaVA brings the well-known LLaVA architecture down to a browser-friendly size. The system persona makes it steerable — the same image and question can produce a terse label, a marketing caption, or a careful accessibility description, just by changing the persona.

  • Steerable alt-text — a persona like "You write concise, factual alt-text" yields ready-to-ship image descriptions with no server cost.
  • Product tagging — ask a persona of "a retail cataloguer" to name the object, colour, and material for a listing.
  • Screenshot Q&A — let users ask questions about a chart or UI screenshot right in the app.
  • Creative captions — a "playful copywriter" persona turns a photo into social-ready copy on-device.

How the API works

nanoLLaVA uses the LLaVA model class. You build a ChatML message list with a system line and a user turn that embeds the <image> placeholder, template it, tokenize, run the image through the processor, then generate — streaming tokens as they come: 

import { AutoProcessor, AutoTokenizer, LlavaForConditionalGeneration, RawImage, TextStreamer }
  from "@huggingface/transformers";

const model_id = "Xenova/nanoLLaVA";
const tokenizer = await AutoTokenizer.from_pretrained(model_id);
const processor = await AutoProcessor.from_pretrained(model_id);
const model = await LlavaForConditionalGeneration.from_pretrained(model_id, {
  dtype: { embed_tokens: "fp16", vision_encoder: "fp16", decoder_model_merged: "q4f16" },
  device: "webgpu",                        // WebGPU required
});

const messages = [
  { role: "system", content: "Answer the question." },
  { role: "user", content: `<image>\n${prompt}` },
];
const text = tokenizer.apply_chat_template(messages, { tokenize: false, add_generation_prompt: true });
const text_inputs = tokenizer(text);
const image = await RawImage.fromURL(imageURL);
const vision_inputs = await processor(image);

const streamer = new TextStreamer(tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });
await model.generate({ ...text_inputs, ...vision_inputs, do_sample: false, max_new_tokens: 150, streamer });

This page runs exactly that, in a Web Worker, and gates on a real WebGPU adapter first — if none exists it shows a labelled needs-WebGPU state instead of a fake answer.
