ModernBERT — BERT, rebuilt for 2024

Same familiar job as the original BERT — hide a word behind [MASK] and rank what belongs there — but a genuinely new encoder underneath: rotary position embeddings, GeGLU feed-forwards, and alternating global/local attention that lets it read an 8,192-token context (16× BERT's 512). Type a sentence with a mask and see its guesses, plus the exact logits. Everything runs on your device.

See inside — logits, and what changed vs classic BERT

Like any masked language model, ModernBERT doesn't output words — it outputs a logit (a raw, unbounded score) for every one of its 50,368 vocabulary tokens at the masked position. We softmax those into probabilities. The bars above show the probability; the logit beside each is the actual number the network produced. ModernBERT vs the original BERT

Both are masked-language-model encoders — the difference is everything under the hood.
 BERT-base (2018)ModernBERT-base (2024)
position encodinglearned absolute (capped at 512)rotary (RoPE) — extrapolates
attentionfull attention, every layeralternating global / 128-token local
context length512 tokens8,192 tokens
feed-forwardGeLUGeGLU (gated)
normalizationpost-LayerNormpre-LayerNorm, bias-free
tokenizerWordPiece, 30,522 vocabbyte-level BPE, 50,368 vocab
training data~3.3B tokens (books + wiki)~2T tokens, incl. code

The alternating attention is the trick that unlocks long context cheaply: most layers only look 128 tokens either side (local), and every third layer looks at everything (global). You get long-range reach without the quadratic cost of full attention on 8,192 tokens.

Why it matters

Encoders never went away — they're the quiet workhorses behind search, retrieval, classification and extraction, where you need to understand text fast, not generate it. ModernBERT is a drop-in upgrade to that whole layer: same masked-LM interface, but faster, longer-context, and trained on far more (and more modern, code-aware) data. Because it scores every word in context, you can ask it what fits, how strongly, and compare alternatives — all locally, with no prompt engineering and no API bill.

  • Long-document autocomplete & cloze — predict a missing word using thousands of tokens of surrounding context, not just the last sentence.
  • Knowledge & code probing — "The capital of France is ___" or "return a ___ b" surfaces what a code-aware encoder learned.
  • Bias auditing — compare predictions across parallel templates to expose the associations baked into training data.
  • A modern backbone — the same weights fine-tune into best-in-class retrieval, classification and NER encoders.

How the API works

The one-liner uses the fill-mask pipeline:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "answerdotai/ModernBERT-base");
const out = await unmask("The capital of France is [MASK].");
// → [{ token_str: "Paris", score: 0.34, sequence: "…" }, { token_str: "Lyon", … }]

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 50368]), slice the row at the [MASK] position, and softmax it ourselves. That's the only way to show the real logit values, not just the pipeline's rounded scores. ModernBERT uses a byte-level BPE tokenizer, so a word-initial token carries a leading space — we trim it for display. All of it runs in a Web Worker so the page never janks, even on a long document.
