I-JEPA — image features learned by predicting, not reconstructing

I-JEPA turns any image into a general-purpose descriptor with no labels — but it got there by a route no other vision model here takes. Instead of reconstructing masked pixels (MAE) or pulling two augmented views together (DINO/CLIP), it predicts the encoder's own representations of masked target regions from a visible context block — learning in representation space. The backbone is a plain ViT with no [CLS] token and no register tokens: the whole output is the patch grid, and the image descriptor is their mean. Add a few images and watch it build a live similarity matrix; then look inside at the patch features predictive pretraining produces. Nothing leaves your device.

See inside — the patch grid & the global descriptor

I-JEPA splits the image into a grid of 14×14-pixel patches and produces one feature vector per patch — and that's all it produces (no [CLS], no registers). Two heatmaps read the grid. Cosine-to-descriptor shows which patches most define "what this image is" (the foreground the mean descriptor is dominated by). Patch norm shows the raw feature magnitude across the grid. Because I-JEPA is trained to predict masked-region representations from context, its patch features are smooth and semantically organised — no [CLS]-token bottleneck to route global information through.

Why it matters

A general-purpose image descriptor is the vision equivalent of a text embedding — and because I-JEPA is trained without labels, it isn't boxed into a fixed list of classes. Its predictive objective is interesting precisely because it skips the expensive parts of other self-supervised recipes: no pixel decoder to reconstruct fine detail, and no bank of augmentations/negatives to hand-tune. That makes it a clean, efficient backbone whose features you can index, compare, and line up between images — here, entirely in a browser tab, without uploading a single photo.

  • Visual deduplication — find near-identical photos, re-uploads, or slightly-edited copies in a library.
  • Reverse image search — "find more like this" over a private set, entirely client-side.
  • Dense correspondence — match the same regions across two photos (registration, tracking, part alignment) using the patch features.
  • Few-shot classification — embed a handful of examples per class, then label new images by nearest match — no fine-tuning.

How the API works

The image-feature-extraction pipeline is the whole recipe; drop to the raw model for patch features:

import { pipeline, RawImage } from "@huggingface/transformers";

const extractor = await pipeline(
  "image-feature-extraction",
  "onnx-community/ijepa_vith14_1k",
); // downloads + caches ~636 MB once

// Simple path: one pooled 1280-d descriptor per image (mean of patch tokens).
const out = await extractor("cat.jpg", { pool: true });
const vec = out.tolist()[0]; // 1280 numbers

// unit-normalize, then cosine similarity is just the dot product:
const norm = (v) => { const n = Math.hypot(...v); return v.map((x) => x / n); };
const sim = (a, b) => a.reduce((s, x, i) => s + x * b[i], 0);

// Deeper path: the raw model gives the per-PATCH sequence — no [CLS], no registers.
const image = await RawImage.read("cat.jpg");
const inputs = await extractor.processor(image);
const { last_hidden_state } = await extractor.model(inputs);
// last_hidden_state: [1, 256, 1280] — 256 patch tokens on a 16×16 grid.
// the image descriptor = mean over the 256 tokens.

This page computes the mean descriptor plus each patch token's norm and cosine-to-descriptor, so "See inside" can show the real magnitudes and the patch map. Inference runs in a Web Worker so building the matrix never janks the page.
