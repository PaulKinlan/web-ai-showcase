RT-DETR — real-time detection with no NMS clean-up

RT-DETR is DETR made fast enough for live video. It keeps the transformer, set-prediction idea — one forward pass, no anchor boxes — but redesigns the encoder and query selection so it emits one clean box per object directly, with no non-maximum-suppression pass. Same 80 COCO classes as classic DETR, a tenth of the model size, and quick enough to run your webcam. All on your device.

Speed contrast — RT-DETR vs classic DETR

Both find the same objects; the difference is speed and cleanliness. RT-DETR (ResNet-18, ~22 MB, NMS-free) is built for real time; the original DETR (ResNet-50, ~160 MB) is the accurate-but-heavy baseline. Run both on the current image and compare the real latencies measured on your device. Loading DETR downloads ~160 MB the first time.

Why it matters

Detection you can run every frame changes what you can build. RT-DETR turns "what's in this video, right now?" into a local function call — private, offline, free per call — without the accuracy tax of the tiny anchor detectors or the latency of a full DETR.

  • Real-time counting — count people at a door, cars at a junction, or stock on a shelf from a live feed, nothing uploaded.
  • Cleaner outputs, less glue — NMS-free means no suppression thresholds to tune; one box per object out of the box.
  • Camera-aware UX at frame rate — react to what the webcam sees (a person? a phone? a cup?) fast enough to feel live.
  • Edge & offline — a ~22 MB detector that ships with your web app and runs on modest hardware.

How the API works

Detection is a one-liner with Transformers.js — RT-DETR uses the same object-detection pipeline as DETR:

import { pipeline } from "@huggingface/transformers";

const detect = await pipeline(
  "object-detection",
  "onnx-community/rtdetr_r18vd",
  { dtype: "q8" },        // ~22 MB, cached after first load
);

const output = await detect(imageURL, { threshold: 0.5, percentage: false });
// → [{ label: "cat", score: 0.97, box: { xmin, ymin, xmax, ymax } }, …]
//   NMS-free: the boxes are already de-duplicated — no suppression pass needed.
//   percentage:false gives pixel coordinates; true gives 0–1 fractions

This page runs the model once at a low floor threshold (0.05) inside a Web Worker, caches every box, and filters the cached list as you drag the slider — so the slider is instant and never re-runs inference. The webcam loop grabs a frame, detects it, and only schedules the next after the last finishes, so it never piles up work or blocks the page.
