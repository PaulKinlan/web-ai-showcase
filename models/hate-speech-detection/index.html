twitter-roberta-hate — spot protected-group hate, privately

Cardiff NLP's RoBERTa-large fine-tuned to detect hate speech aimed at a protected characteristic. It reads any text and picks one of eight classes — seven kinds of targeted hate (gender, race, sexuality, religion, origin, disability, age) plus not_hate — as a single softmax. This is a strictly narrower task than toxicity or offensiveness: not everything rude is hate. It is an imperfect moderation signal, not a verdict — it reads surface words, has no speaker model, and carries its training biases. Every score is computed on your device; no comment is ever sent to a server.

Run it

Type or paste text; it re-scores automatically a moment after you stop typing. The hate threshold decides the verdict: if the combined hate probability (1 − not_hate) lands at or above it, the text is flagged and the model names the targeted group — otherwise it's allowed. See inside — one softmax that also names the target

The final layer produces eight raw logits, and a single softmax turns them into competing probabilities that sum to 100%. Seven of those classes are hate keyed to a protected characteristic; the eighth is not_hate. So the model does two things at once: it decides whether the text is hateful (1 − P(not_hate)) and, if so, which group is targeted — something a plain toxicity score can't tell you. Because the classes compete, a comment that is nasty but not group-targeted lands on not_hate — the model is deliberately conservative about calling something hate. That is a feature for precision and a limitation for recall: coded or novel hate can slip through.

Why it matters

Hate speech is a distinct, high-stakes category: platforms, regulators, and trust-and-safety teams treat protected-group attacks differently from ordinary rudeness. A model that not only flags hate but names the targeted characteristic lets you route, prioritise, and report it correctly — and doing it on-device means sensitive text never leaves the tab.

  • Targeted-hate routing — send a race or religion hit to the specialist queue, not the generic spam bin.
  • Separate hate from mere rudeness — pair with a toxicity or offensiveness model so "you're an idiot" and "[slur], go home" get different responses.
  • Private-by-default — score reports, DMs, or support tickets for hate without a server round-trip or third-party API.
  • Human-in-the-loop triage — surface likely hate with its target for a reviewer, instead of an automatic ban on a fallible signal.

Honesty note: this model reflects its training data (English tweets). It is conservative — reliable on explicit, surface-level attacks, but it lets a lot of coded, "polite", or subtle hate through as not_hate (a recall gap), and it can encode demographic bias. Treat every score as one signal that routes a human — never a verdict, and never read not_hate as proof that text is safe.

How the API works

The convenience path is one call; ask for every class with topk: null:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline("text-classification", "onnx-community/twitter-roberta-large-hate-latest-ONNX");
const out = await classify("go back to your own country", { topk: null });
// → [{ label: "hate_origin", score: 0.94 }, { label: "not_hate", score: 0.03 }, …]  (softmax over 8)

For "See inside" this page runs the tokenizer then the model() directly, reads the raw eight-value logits, and applies the softmax itself — so it can show both the logit and the probability, compute the aggregate hate score (1 − not_hate), and apply your threshold. All of it runs in a Web Worker so typing never janks.
