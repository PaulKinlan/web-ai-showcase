SmolVLM2 — reason across frames, not just one picture

SmolVLM v1 talks to a single image. SmolVLM2 takes several images — or frames sampled from a video — in one turn, and reasons across them: what changed, how two scenes compare, what happened over time. Same tiny 256M footprint, all streaming token by token on your GPU, with nothing uploaded.

Why it matters

Single-image VLMs answer "what's in this picture?". A multi-frame VLM answers "what happened?" — and it does it on-device, privately, for free. Once a model can hold several frames in one context, a new class of browser features opens up:

  • Video moment triage — sample frames from a clip and ask "did anything change?" without uploading the video or paying per call.
  • Before/after checks — compare two product photos, two document scans, or two UI states and get a plain-language diff.
  • Visual step-by-step help — feed a sequence of screenshots and ask "what did the user do between these steps?".
  • Private camera copilots — hold the last few camera frames and answer "what just happened?" entirely on the device.

How the API works

SmolVLM2 uses the vision-to-text model classes. You build a chat message whose content mixes multiple image placeholders and text, template it, run the processor over the array of images, then generate — streaming tokens as they come: 

import { AutoProcessor, AutoModelForVision2Seq, load_image, TextStreamer }
  from "@huggingface/transformers";

const processor = await AutoProcessor.from_pretrained("HuggingFaceTB/SmolVLM2-256M-Video-Instruct");
const model = await AutoModelForVision2Seq.from_pretrained(
  "HuggingFaceTB/SmolVLM2-256M-Video-Instruct",
  { dtype: "q4f16", device: "webgpu" },   // WebGPU required
);

const images = await Promise.all(frameURLs.map(load_image));      // several frames
const content = images.map(() => ({ type: "image" }));            // one placeholder per frame
content.push({ type: "text", text: "What changed between these frames?" });
const messages = [{ role: "user", content }];

const text = processor.apply_chat_template(messages, { add_generation_prompt: true });
const inputs = await processor(text, images, { do_image_splitting: false });
// inputs.pixel_values.dims === [1, numFrames, 3, H, W]  ← the frames, batched

const streamer = new TextStreamer(processor.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });
await model.generate({ ...inputs, max_new_tokens: 160, streamer });

This page runs exactly that, in a Web Worker, and gates on a real WebGPU adapter first — if none exists it shows a labelled needs-WebGPU state instead of a fake answer.
