Multi-model: SmolVLM2 reads, an LLM narrates

Two on-device models, one pipeline. SmolVLM2 captions each frame; then a small text LLM — Qwen2.5-0.5B — weaves those captions into a single flowing narration. Vision → language → composition, all local, all on WebGPU, nothing uploaded.

Models

1 · Vision model (SmolVLM2)

2 · Narrator LLM (Qwen2.5-0.5B)

How the pipeline is wired

Each model runs in its own Web Worker. SmolVLM2 is called once per frame for a one-sentence caption; the captions become the LLM's prompt. The two stages are honest and inspectable — you see the captions before the narration is composed from them. 

// Stage 1 — SmolVLM2 captions each frame (vision → text)
const captions = [];
for (const frame of frames) {
  captions.push(await vlm.generate([frame], "Describe this frame in one short sentence."));
}
// Stage 2 — the LLM composes them (text → text)
await llm.narrate(captions, tone);   // Qwen2.5-0.5B-Instruct, streamed
