SmolLM2-135M — the smallest useful chat LLM, in your browser

SmolLM2-135M-Instruct is the smallest member of Hugging Face's SmolLM2 instruct family — only ~135M parameters. That's the interesting bet: how capable can a chat model be when it's tiny enough to download in seconds and run on any device? This page runs it entirely in your browser via Transformers.js, streaming a real chat completion on the universal WebAssembly/CPU path (no GPU required). Nothing is uploaded; there is no API key and no server.

A distinct model from the 360M sibling already in this showcase (SmolLM2-360M) — this is the extreme-low-end checkpoint. Its ONNX build exports as the standard llama decoder architecture, so it loads through the ordinary text-generation pipeline. Being this small, it's fast but simple: great for quick replies, rewriting, and structured little tasks — not deep reasoning.

Why it matters

A 135M-parameter chat model that fits in ~140 MB and runs on plain CPU makes an on-device assistant reachable on almost any device — an old phone, a locked-down laptop, a kiosk — with no GPU, no key, and nothing sent to a server. It won't out-argue a frontier model, but for fast, private, "good enough" text tasks it's a genuinely useful primitive.

  • Instant, private replies — draft, rewrite, and summarise short text without a round-trip to a server.
  • Runs anywhere — ~140 MB q8 on the WebAssembly/CPU path; no adapter or driver flags required.
  • An honest baseline — feel exactly where a tiny model helps and where it doesn't, so you can pick the right size for a job.
  • A component, not a brain — pair it with retrieval or tools (see the multi-model demo) to ground its answers.

How the API works

It's the standard Transformers.js text-generation pipeline — pass a messages array (the pipeline applies SmolLM2's ChatML template), a TextStreamer for token-by-token output, and your sampling settings: 

import { pipeline, TextStreamer } from "@huggingface/transformers";

// q8 runs on the universal WASM/CPU path (WebGPU as an optional accelerator).
const generator = await pipeline(
  "text-generation", "HuggingFaceTB/SmolLM2-135M-Instruct", { device: "wasm", dtype: "q8" },
);

const messages = [
  { role: "system", content: "You are a helpful, concise assistant." },
  { role: "user", content: "Give three tips for staying focused." },
];

const streamer = new TextStreamer(generator.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });

await generator(messages, {
  max_new_tokens: 200, do_sample: true, temperature: 0.7, top_p: 0.9, streamer,
});

The "see inside" panel runs one raw forward pass — generator.model(generator.tokenizer(prompt)) — reads the last position's logits, softmaxes them, and shows the real top-k. Everything runs in a Web Worker so generation never blocks the page; the loader auto-inits an already-cached copy and never silently re-downloads.
