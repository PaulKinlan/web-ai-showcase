wav2vec2 — Romanian speech, one letter per frame

The English wav2vec2 demo speaks one language, the Russian XLSR demo another, and the Polish and Finnish VoxPopuli demos two more. This one recognises Romanian. It is wav2vec2-base pre-trained self-supervised on 10,000 hours of unlabelled European-Parliament audio (VoxPopuli) and then fine-tuned on Romanian — a compact ~95M-param CTC recogniser whose output alphabet carries the Romanian diacritics (ă â î ș ț). Same CTC mechanics as the other ASR demos: label every ~20 ms frame, then collapse the frames into text. Pick a clip and watch the frames collapse into Romanian.

Why it matters

Self-supervised pre-training on huge piles of unlabelled speech — here 10,000 hours of European-Parliament audio — is how you get a good recogniser for a language like Romanian from a modest amount of labelled data. The model learned what human speech sounds like before it ever saw a Romanian transcript. That is what puts a usable Romanian recogniser inside a browser tab — private, offline once cached, no server, no key. And because it's CTC (frames, not generated tokens) it can't hallucinate an unspoken sentence, and every character is pinned to a moment in time.

  • On-device Romanian dictation and captions — for apps that must keep audio on the device (health, legal, journalism) in the Romanian market.
  • Runs anywhere WASM does — no GPU required; the model runs on ONNX Runtime Web's WebAssembly backend, the same engine transformers.js uses underneath.
  • Romanian forced alignment — real per-word timings for karaoke, subtitles, and pronunciation feedback, straight from the frames.
  • Privacy-first field work — transcribe sensitive Romanian audio with nothing leaving the device.

How the API works

The Romanian ONNX export is packaged for the onnx-asr toolkit, not transformers.js — its config.json declares model_type: "wav2vec2-ctc", so transformers.js's AutoModelForCTC refuses it (Unsupported model type). Instead we drive the raw graph directly on ONNX Runtime Web — feature normalisation is baked in, so we feed the raw 16 kHz waveform and CTC-decode the log-probs ourselves: 

import * as ort from "onnxruntime-web";

const base = "https://huggingface.co/OpenVoiceOS/wav2vec2-base-10k-voxpopuli-ft-ro-onnx/resolve/main/";
const graph = new Uint8Array(await (await fetch(base + "model.onnx")).arrayBuffer());
const data  = new Uint8Array(await (await fetch(base + "model.onnx.data")).arrayBuffer()); // ~360 MB fp32

const session = await ort.InferenceSession.create(graph, {
  executionProviders: ["wasm"],
  externalData: [{ path: "model.onnx.data", data }],   // register the external weights
});

// audio = raw 16 kHz mono Float32Array (normalisation is inside the graph)
const { logprobs } = await session.run({
  input_values:  new ort.Tensor("float32", audio, [1, audio.length]),
  input_lengths: new ort.Tensor("int64", BigInt64Array.from([BigInt(audio.length)]), [1]),
});
// logprobs shape [1, frames, 37]  ← 37 Latin-Romanian symbols, log-softmax

// greedy CTC decode: argmax per frame, merge duplicates, drop the blank (id 0), '▁' → space
const ids  = argmaxPerFrame(logprobs);
const text = collapse(ids);            // Romanian text — no decoder involved

Inference runs in a Web Worker so the UI never janks. The worker returns the raw per-frame argmax and real per-word timings (from the frame each character was emitted at) — nothing is faked or pre-computed.
