SCUNet — clean the noise from a photo

Give this model a noisy or grainy photo — high-ISO shots, low-light grain, sensor noise — and it removes the noise while keeping real detail: clean flats, crisp edges, no smeared texture. SCUNet (Swin-Conv-UNet) is a strong blind image denoiser. It's not a blur — it learned to tell noise apart from structure and strip only the noise. Runs entirely on your device via ONNX Runtime Web; the photo never leaves the tab.

How to read it

Bright green = big change (noise removed); dark = untouched. Notice the glow fills the flat, noisy areas but eases off along strong edges — SCUNet removes the grain and keeps the structure, which is exactly what separates a denoiser from a blur.

Why it matters

Noise is the price of low light and small sensors. On-device denoising rescues those shots privately, with no upload and no per-image cloud cost:

  • Phone & photo apps — a "clean up this shot" button that runs client-side, taming high-ISO and night-mode grain.
  • Scanning & archiving — de-noise old scans and film grain in a static web tool with zero backend.
  • Video calls & webcams — clean up a noisy low-light feed frame-by-frame on the device.
  • Pre-processing for other models — feed a cleaner image into detection, OCR or super-resolution so they see signal, not noise.

SCUNet removes plausible noise; extreme noise cleans only partially, and the blind variant stays faithful rather than inventing texture (the GAN variant, not shipped here, does the opposite).

How the API works

transformers.js has no SCUNet class and no denoising task, so we run the ONNX graph directly with ONNX Runtime Web. In goes the noisy image, out comes the cleaned one:

import * as ort from "onnxruntime-web";

// the .onnx graph references its weights in an external-data sidecar (.onnx.data)
const session = await ort.InferenceSession.create(graphBytes, {
  executionProviders: ["wasm"],
  externalData: [{ path: "scunet_color_real_psnr.onnx.data", data: weightBytes }],
});

// noisy image → [1,3,H,W] float32 in [0,1] (plain RGB, no normalisation)
const input = new ort.Tensor("float32", toNCHW(noisy, W, H), [1, 3, H, W]);
const { denoised } = await session.run({ image: input });

// denoised is [1,3,H,W] in [0,1] → clamp to RGBA and resize back to the original aspect
const clean = tensorToImage(denoised.data, W, H);

Inference and the whole output composite run in a Web Worker so the UI never blocks, and the ~75 MB model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
