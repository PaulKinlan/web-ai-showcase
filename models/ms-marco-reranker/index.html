MS MARCO MiniLM — rank passages by how well they answer

This is a cross-encoder: instead of turning a query and a passage into separate vectors and comparing them, it reads both together and outputs a single relevance score. Attention flows across the query and the passage, so it catches relevance that word-matching and even embeddings miss. Give it a query and some candidate passages and watch it sort them — then look inside at the raw logit it assigns each pair. Nothing leaves your device.

Why it matters

Rerankers are the second stage of modern search. A fast retriever (keyword index or embeddings) casts a wide net and returns, say, the top 50 candidates; a cross-encoder then reads each candidate against the query and reorders them with far higher precision. That two-stage pattern powers RAG, site search, and recommendations — and at 90 MB the reranker runs in a tab, so the precise, expensive step happens on-device with nothing sent anywhere.

  • Two-stage search — retrieve fast, then rerank the shortlist for real precision.
  • Better RAG — feed the LLM the genuinely most relevant chunk, not the closest vector.
  • Answer selection — pick the best reply from a set of candidate answers.
  • Deduping intent — score which FAQ entry actually matches a user's phrasing.

How the API works

A cross-encoder scores query–passage pairs; we read the raw logit off the model directly:

import { AutoTokenizer, AutoModelForSequenceClassification } from "@huggingface/transformers";

const model_id = "Xenova/ms-marco-MiniLM-L-6-v2";
const tokenizer = await AutoTokenizer.from_pretrained(model_id);
const model = await AutoModelForSequenceClassification.from_pretrained(model_id);

// Encode the query with each passage AS A PAIR — [CLS] query [SEP] passage [SEP]:
const inputs = tokenizer(
  passages.map(() => query),
  { text_pair: passages, padding: true, truncation: true },
);
const { logits } = await model(inputs); // one relevance logit per pair — higher = more relevant
// sort the passages by logits.data — that's your reranking

This page loads the model through the shared question-answering-style loader as a text-classification pipeline, then reuses pipe.model + pipe.tokenizer to get the raw logits (the pipeline would squash them through a sigmoid). All inference runs in a Web Worker so ranking never janks the page.
