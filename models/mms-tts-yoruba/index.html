MMS-TTS Yoruba — end-to-end Yoruba speech, one network

This is the Yoruba checkpoint of Meta's MMS-TTS family — a distinct model from the English MMS-TTS demo and the Hindi one: its own native Yoruba vocabulary (43 symbols, is_uroman:false). Yoruba's orthography is Latin-based but carries the sub-dot letters ẹ/ọ/ṣ and the tone marks (à/á, è/é) that encode Yoruba's three tones — glyphs no other checkpoint's vocab includes — so it speaks tone-marked Yoruba that the Latin-alphabet checkpoints cannot. Yoruba is a major West African tonal language. VITS is a conditional VAE with normalizing flows and a stochastic duration predictor: one end-to-end network from text straight to a waveform, no separate vocoder, and the same Yoruba sentence comes out with slightly different rhythm every time.

Why it matters

A Yoruba voice that runs entirely on-device: no per-character bill, no text leaving the tab, and a tiny ~38 MB footprint you can ship inside a PWA or kiosk. Yoruba is spoken by tens of millions across Nigeria, Benin, Togo and the diaspora; on-device TTS for African languages is scarce, so a private, offline Yoruba voice that works everywhere is genuinely useful.

  • Offline Yoruba read-aloud — narrate Yoruba articles, forms or messages with no service to sign up for and nothing sent anywhere.
  • Accessibility — a built-in Yoruba screen-voice for content, working without a network.
  • Tiny, embeddable Yoruba voice — ~38 MB makes it practical inside an app or device UI.
  • Natural variation — the stochastic duration predictor keeps repeated Yoruba prompts from sounding mechanically identical.

How the API works

MMS-TTS Yoruba is a one-liner with the text-to-speech pipeline — the Yoruba checkpoint is picked by its id:

import { pipeline } from "@huggingface/transformers";

const tts = await pipeline("text-to-speech", "Xenova/mms-tts-yor", { dtype: "q8" });

const out = await tts("Nẹ́tíwọ́ọ̀kì kan ń yí àwọn lẹ́tà padà sí ìgbì ohùn tààrà.");
// out.audio         → Float32Array waveform (mono)
// out.sampling_rate → 16000
// Run it again with the same text → a slightly different waveform (stochastic duration).

This page runs the pipeline inside a Web Worker so synthesis never blocks the interface, encodes the returned Float32 samples to a WAV blob for the <audio> element, and draws the same samples as the waveform under "See inside". The English demo is the same call with a different checkpoint (mms-tts-eng). The Yoruba tokenizer is is_uroman:false, so the tone-marked Yoruba text (with ẹ/ọ/ṣ and the tone accents) is tokenised directly — no romanisation step.
