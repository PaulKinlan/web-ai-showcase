Granite Embedding multilingual — a cross-lingual embedder from IBM

IBM's Granite Embedding r2 is an Apache-2.0 retrieval embedder trained for enterprise search — and it is genuinely multilingual: a question in English lands right next to its French, Spanish, Chinese or Japanese equivalent in the same 384-dimensional space, so you can search one language and match another. The r2 generation sits on a ModernBERT backbone (rotary positions, local/global attention, a long context) rather than the classic 512-token BERT encoder. Paste passages in any mix of languages to build a live similarity matrix, search by meaning across languages, and inspect the vector. Nothing leaves your device.

See inside — the vector, and cross-lingual alignment

ModernBERT emits one 384-number vector per token. Granite Embedding mean-pools them into a whole-passage vector and this page L2-normalizes it — so cosine similarity is a plain dot product. Pick a passage to see its real vector, its pre-normalization magnitude, and how many tokens it spanned.

Why it matters

Embeddings power search, recommendations, deduplication, clustering and retrieval-augmented generation. Granite Embedding's pitch is cross-lingual retrieval under a permissive licence: index a knowledge base once and let users query it in their own language; deduplicate content that says the same thing in different languages; cluster a multilingual support inbox by theme — all in a tab, all private, all Apache-2.0.

  • Cross-lingual search — one index, many query languages; an English query finds the Spanish or Japanese answer.
  • Multilingual RAG — retrieve the right passages regardless of the document's language, then answer with an LLM (see the multi-model demo).
  • Dedup & clustering — group "same thing, different language" across a mixed corpus.
  • Retrieve → rerank — a fast multilingual first pass, then a cross-encoder for precision (see the multi-model demo).

Use cases

Basics

Do two sentences mean the same thing — even in different languages? One cosine number.

Practical

Cross-lingual semantic search and near-duplicate detection over your own passages.

Wild

Cluster a pile of mixed-language passages into themes with k-means over the 384-d vectors.

Multi-model

Granite retrieves a shortlist across languages, then a cross-encoder reranks the top hits.

How the API works

Feature-extraction with mean pooling is the whole recipe — no instruction prefix, works across languages:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "onnx-community/granite-embedding-97m-multilingual-r2-ONNX",
); // downloads + caches ~98 MB once

// Granite Embedding mean-pools the token vectors — pass texts in any language:
const out = await extract([
  "How do I reset my password?",
  "Comment réinitialiser mon mot de passe ?",
], { pooling: "mean", normalize: true });
const [en, fr] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = en.reduce((s, x, i) => s + x * fr[i], 0); // high — same meaning, different language

This page asks the worker for the un-normalized vectors so "See inside" can show the real pre-normalization magnitude, then normalizes them in JS. Inference runs in a Web Worker, so the matrix, projection and search never jank the page.
