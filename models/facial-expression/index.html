Facial expression recognition — read the emotion, on your device

A Vision Transformer, fine-tuned on faces, takes a single frame and scores it across 7 emotions — happy, sad, angry, surprise, fear, disgust, neutral. Point your camera or drop a photo; the read happens entirely in this tab. No frame is ever uploaded or stored.

See inside

The ViT's final layer emits one raw score — a logit — per emotion. A softmax turns the 7 logits into probabilities that sum to 100%. Two numbers describe the shape of that distribution: the top-1 margin (how far the winning emotion leads the runner-up) and the normalised entropy (0 = the model is certain, 1 = it's torn between emotions). A plain smile is peaky and low-entropy; a subtle or blended expression is flat and high-entropy — which is exactly when you should NOT trust a single label. Affective reads are noisy: treat them as a soft signal, never ground truth about how someone actually feels.

Why it matters

Expression is a signal most interfaces throw away, because "read the face" used to mean a camera upload to a cloud API — a privacy non-starter. Running the model locally flips that: the frame never leaves the device, so affective UX becomes something you can actually ship.

  • Reaction capture — let people react to content with their face instead of a star rating, entirely client-side.
  • Sentiment-aware UX — a tutorial or support flow that notices visible confusion/frustration and offers help, without sending video anywhere.
  • Accessible, consented affect — opt-in emotion cues for research, wellbeing check-ins, or expressive avatars — with the raw frame staying on-device.
  • Calibration-gated decisions — use the entropy/margin read-out to act only on confident reads and ignore ambiguous ones.

A note on ethics: facial-expression models infer a visible expression, not a person's true inner state, and they carry demographic bias. Use them for consented, low-stakes, on-device UX — never surveillance or high-stakes judgements about people.

How the API works

The convenience path is two lines with Transformers.js:

import { pipeline } from "@huggingface/transformers";

const readEmotion = await pipeline(
  "image-classification",
  "Xenova/facial_emotions_image_detection",
); // downloads + caches ~88 MB once

const out = await readEmotion(imageOrCanvasURL, { top_k: 7 });
// → [{ label: "happy", score: 0.94 }, { label: "neutral", score: 0.03 }, … ]

This page goes one level deeper for "See inside": instead of the pipeline it runs the model's own processor then model() in a single forward pass, so it can read the raw logits tensor and compute the softmax, entropy, and margin itself. Webcam frames are captured to a <canvas> and classified as a data URL — nothing is uploaded. Inference runs in a Web Worker so the camera preview never stutters.
