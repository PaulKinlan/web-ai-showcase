mBART-50 — many-to-many translation from one denoising-pretrained model

mBART-50 translates directly between any pair of 50 languages — no English pivot — with a single multilingual encoder–decoder. Its twist is its upbringing: mBART was pretrained by denoising (corrupting then reconstructing monolingual text in all 50 languages, the BART objective scaled up) before it was ever taught to translate. Pick a source and target, type, and watch it translate on-device. The output language is chosen by a single forced beginning-of-sequence token, shown below.

See inside — language codes, forced BOS token & tokens

mBART is a sequence-to-sequence translator: it encodes the source, then decodes the target one token at a time. What selects the output language is a single trick — the decoder is forced to begin with the target language's special token (forced_bos_token_id). Change the target and only that first token changes; the same input then decodes into a different language. Why it matters

One model, 50 languages, any direction, entirely on-device — no per-call fees, no data leaving the tab. mBART-50 covers the widely-used languages of Europe and Asia with solid quality, and because it shares one representation across all of them you can translate between two languages that were never explicitly paired in training.

  • Private & offline — translate messages, notes, or documents where nothing can leave the device, with no signal needed.
  • Any-to-any pairs — go Japanese→German or Spanish→Hindi directly, no English hop.
  • Relay translation — chain languages A→B→C locally and watch meaning travel and drift.
  • Compose with other models — translate first, then speak, summarise, or classify the result — all in the browser.

How the API works

Translation is one pipeline call; the language pair is set per call:

import { pipeline } from "@huggingface/transformers";

const translator = await pipeline(
  "translation",
  "Xenova/mbart-large-50-many-to-many-mmt",
); // downloads + caches ~850 MB once

const out = await translator("The weather is nice today.", {
  src_lang: "en_XX",   // mBART-50 language code
  tgt_lang: "fr_XX",   // French
});
// → [{ translation_text: "Le temps est agréable aujourd'hui." }]

// Under the hood, tgt_lang selects a forced BOS token id that the decoder
// must emit first — that single token is what picks the output language.

mBART-50 codes are a two-letter language plus a region tag (en_XX, zh_CN, pt_XX) — the same scheme as the original fairseq model. This page streams tokens via a TextStreamer inside a Web Worker, so decoding never blocks the interface.
