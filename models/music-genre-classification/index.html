Music genre classification — what genre is this?

Drop in an audio file — or record a few seconds — and the model listens and names its musical genre across ten classes (blues, classical, country, disco, hip-hop, jazz, metal, pop, reggae, rock), showing the full confidence distribution. It hears timbre and rhythm, not metadata. Everything runs on your device — the audio never leaves the tab.

Why it matters

Genre is the classic music-understanding task — the model learns what disco or metal sounds like from the raw waveform, no tags or filenames. Doing it in the browser means a library of audio can be organised or filtered without uploading a single track.

  • Auto-tagging — label a personal music library or a sample pack by genre, on-device.
  • Playlisting & discovery — group tracks by feel, or seed a "more like this" queue.
  • Creative tools — a DAW plugin that suggests a genre for a loop as you record it.
  • Moderation / routing — sort user-uploaded audio by genre before it hits a server.

It's a best-effort classifier trained on the small GTZAN set: it's strongest on clear, full productions and can be swayed by a track's dominant texture — a chord progression reads as jazz, a drum-heavy loop as hip-hop — and non-music audio (speech) just gets the nearest-looking genre. The built-in samples are short procedural clips (a chord progression, a drum beat); upload a real song for the full ten-genre range.

How the API works

Genre classification is one pipeline call on a 16 kHz mono waveform:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline("audio-classification",
  "onnx-community/Musical-genres-Classification-Hubert-V1-ONNX");
const out = await classify(audioFloat32, { top_k: 10 });
// → [{ label: "classical", score: 0.97 }, { label: "jazz", score: 0.01 }, … ]

The browser decodes/resamples the file to 16 kHz mono; the DistilHuBERT model turns the waveform into features and scores the ten genres. It all runs in a Web Worker so the UI stays responsive.
