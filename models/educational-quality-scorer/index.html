Educational-quality scorer — how much does this text teach?

Paste a passage and the model rates its educational value on a 0–5 scale — not whether it's positive or on-topic, but how much it teaches. It's the FineWeb-Edu classifier HuggingFace built to filter a trillion-token pre-training corpus down to the genuinely educational parts. It all runs on your device — the text never leaves the tab.

Why it matters

The quality of an LLM depends on the quality of its training text, and "educational value" turned out to be a strong filter: HuggingFace scored the web with this model and kept the high-scoring pages to build FineWeb-Edu, which trains markedly better models. Running the same scorer on-device lets you judge or rank text without sending it anywhere.

  • Data curation — filter or rank a corpus by educational value before training or fine-tuning.
  • Content triage — surface the genuinely informative pages in a crawl or a knowledge base.
  • Writing feedback — see whether a draft reads as substantive teaching material or filler.
  • EdTech — rank or recommend reading material by how much it actually teaches.

The score is a regression estimate, not a fact: it rewards clear, on-topic, informative prose and marks down promotional, chatty, or incoherent text — but it judges style of usefulness, not correctness, and it was trained mostly on English web text. Treat it as a fast signal, not a verdict. The raw value can dip slightly below 0 or above 5; it's clamped to the 0–5 band.

How the API works

It's a regression head — one score per passage — so we read the raw output directly:

import { AutoModel, AutoTokenizer } from "@huggingface/transformers";

const tok = await AutoTokenizer.from_pretrained("onnx-community/fineweb-edu-classifier-ONNX");
const model = await AutoModel.from_pretrained("onnx-community/fineweb-edu-classifier-ONNX");
const { logits } = await model(await tok(text, { truncation: true }));
const score = Math.max(0, Math.min(5, logits.data[0])); // 0–5 educational value

It runs the encoder + regression head in a Web Worker so typing stays smooth. We use AutoModel rather than the text-classification pipeline because the head is a regressor, not a classifier.
