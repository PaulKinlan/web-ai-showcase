RoBERTa GoEmotions — read 28 emotions at once

Binary sentiment tells you positive or negative. This RoBERTa model, fine-tuned on Google's GoEmotions, is multi-label: it scores text across 28 distinct emotions — gratitude, curiosity, annoyance, grief, pride… — each with its own independent confidence. Several can fire at once. Type below and watch the whole emotional profile light up, all on-device.

See inside — all 28 sigmoid scores

This is the model's raw output: one independent probability per class, from a sigmoid — not a softmax. That's the whole point of multi-label. A softmax would force the 28 scores to compete and sum to 1 (pick one winner); a sigmoid lets "gratitude" and "relief" both be 0.9 because they aren't mutually exclusive. Drag the threshold to see which emotions the model would report — the line between "present" and "absent" is a choice you make, not the model.

Why it matters

Sentiment flattens feeling into a single axis. Emotion detection keeps the nuance — and doing it in the browser means you can read how someone feels as they type, privately, with no round-trip.

  • Support triage with intent — route on the emotion, not just the star rating: anger → escalate, confusion → docs, gratitude → testimonial.
  • Content mood tagging — label posts, reviews, or journal entries by their emotional palette for search and analytics.
  • Tone-aware writing aids — warn a draft that reads as annoyed when you meant it as neutral.
  • Private wellbeing tools — track the emotional arc of a diary entirely on-device, nothing uploaded.

How the API works

Multi-label classification is one call; ask for all 28 scores with top_k:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "text-classification",
  "SamLowe/roberta-base-go_emotions-onnx",
); // downloads + caches ~125 MB once

const out = await classify("thank you, I was so worried!", { top_k: 28 });
// → [{ label: "gratitude", score: 0.92 }, { label: "relief", score: 0.78 },
//    { label: "nervousness", score: 0.41 }, … all 28, each independent ]

The model's config declares problem_type: "multi_label_classification", so Transformers.js applies a sigmoid per class instead of a softmax — that's why the scores don't sum to 1. All of it runs in a Web Worker so typing never janks.
