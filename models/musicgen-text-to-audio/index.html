MusicGen — turn a text prompt into music, on your device

MusicGen composes short, original music from a text description — "lo-fi hip hop, chill" or "90s rock, loud guitars" — right in your browser. This is not text-to-speech: the model predicts a stream of audio tokens and decodes them into a 32 kHz waveform. No server, no upload, no per-clip bill. It is also genuinely compute-heavy, so keep clips short and expect a real wait.

Run it

Describe some music, choose a length and a couple of sampling controls, then hit Generate. Generation runs in a Web Worker with a real token-level progress bar — the model is decoding audio frames one step at a time, and the bar reflects that, not a spinner.

Heads up — this is compute-heavy. MusicGen runs on the CPU (WASM). Even a few seconds of music can take a while to generate, and generation time grows with the length. Start with a short clip (3–5 s). The wait and the timing readout below are real; nothing here is pre-baked.

Why it matters

Generative music used to mean a cloud API, a licence, and your prompt leaving the building. MusicGen makes short original music a local capability of your page — the prompt never leaves the tab, and there's no per-clip bill. It's slow on CPU today, but it proves the shape of what's coming to the browser.

  • Royalty-free background loops — generate a bespoke ambient bed for a video, game, or presentation, client-side, with no stock-music licence to clear.
  • Generative UI & game audio — mood-driven stingers and beds composed from a text description at runtime, unique per session.
  • Prototyping & sketching — musicians and designers audition ideas ("what would 80s synthwave here sound like?") without opening a DAW.
  • Accessible, private creativity — anyone can make music from words, offline, with nothing uploaded and no account.

How the API works

MusicGen is a multi-part model, so we use the low-level classes (that's also the only way to get token-level progress). Load once, then generate: 

import {
  MusicgenForConditionalGeneration,
  AutoTokenizer,
} from "@huggingface/transformers";

const MODEL = "Xenova/musicgen-small";
const model = await MusicgenForConditionalGeneration.from_pretrained(MODEL, {
  dtype: {                         // the known-good config
    text_encoder: "q8",
    decoder_model_merged: "q8",
    encodec_decode: "fp32",        // MUST stay fp32 — q8 here garbles the audio
  },
  device: "wasm",                  // no reliable WebGPU path for this model
});
const tokenizer = await AutoTokenizer.from_pretrained(MODEL);

const inputs = tokenizer("lo-fi hip hop, chill");
const out = await model.generate({
  ...inputs,
  max_length: 5 * 50 + 4,          // ~50 audio tokens per second of music
  guidance_scale: 3,
  temperature: 1,
  do_sample: true,
  // streamer: a BaseStreamer whose put() fires once per decoded token → real progress
});
// out.data → Float32Array waveform
// model.config.audio_encoder.sampling_rate → 32000

This page runs all of that inside a Web Worker so the long decode never blocks the interface, attaches a BaseStreamer for the real progress bar, encodes the Float32 samples to a WAV blob for the <audio> element, and draws the same samples as the waveform.
