Signature detection — where are the signatures?

Drop in a scanned document and the model draws a box around every handwritten signature it finds. It's a detector trained for one job — locating signatures on forms, contracts and letters — the way another detector finds tables. Everything runs on your device; the document never leaves the tab.

Why it matters

Finding where a signature is — not reading it — is a common first step in document automation: you can check a form was signed, crop or redact the signature, or route the page for review. Doing it in the browser means sensitive contracts never have to be uploaded.

  • Form & contract processing — verify a document was signed before accepting it.
  • Redaction — locate and blur signatures before sharing a document.
  • Data extraction — crop the signature region for an e-signature or audit trail.
  • Triage — flag unsigned pages in a batch of scanned documents, on-device.

It detects the presence and location of a signature — it does not read the name, verify who signed, or check authenticity. It's trained on scanned-document signatures, so it expects real pen-stroke marks (the built-in samples use drawn signatures); very faint marks or unusual layouts can be missed, and the confidence is shown so you can judge.

How the API works

It's one object-detection call; boxes come back in image pixel coordinates:

import { pipeline } from "@huggingface/transformers";

const detect = await pipeline("object-detection",
  "onnx-community/yolos-base-signature-detection-ONNX");
const out = await detect(documentImage, { threshold: 0.3, percentage: false });
// → [{ label: "signature", score: 0.95, box: { xmin, ymin, xmax, ymax } }]

One forward pass returns every signature above the threshold. It runs in a Web Worker so the page stays responsive, and the boxes are scaled from original pixels to the displayed image.
