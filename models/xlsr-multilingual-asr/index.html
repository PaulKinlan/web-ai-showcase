wav2vec2-XLSR-53 — Russian speech, one Cyrillic character per frame

The English wav2vec2 demo speaks one language. This is its cross-lingual cousin: XLSR-53 was pre-trained on 53 languages (56,000 hours of multilingual audio) and then fine-tuned on Russian. That shared multilingual foundation is what makes a solid Russian recogniser trainable from a modest amount of labelled speech — the model already learned what human speech sounds like before it ever saw a Russian transcript. Same CTC mechanics as the English model, but the output alphabet is Cyrillic. Pick a clip and watch the frames collapse into Russian.

Why it matters

Most of the world's ~7,000 languages have little labelled speech data. Cross-lingual self-supervised pre-training is how the field reaches them: XLSR-53 learns speech structure from 53 languages at once, so fine-tuning to a new language needs far fewer transcripts. That is what puts a usable Russian recogniser inside a browser tab — private, offline once cached, no server, no key. And because it's CTC (frames, not generated tokens) it can't hallucinate an unspoken sentence, and every character is pinned to a moment in time.

  • Non-English on-device ASR — dictation and captions for languages that never get a cloud API.
  • Low-resource language reach — the XLSR recipe extends to dozens of fine-tunes from the same backbone.
  • Cyrillic forced alignment — real per-word timings for Russian karaoke, subtitles, and pronunciation feedback.
  • Privacy-first field work — transcribe sensitive Russian audio with nothing leaving the device.

How the API works

The one-liner is the ASR pipeline — but to see the frames we load the model directly and do the CTC collapse ourselves (which is exactly what the pipeline does internally): 

import { AutoProcessor, AutoModelForCTC } from "@huggingface/transformers";

const model_id = "onnx-community/wav2vec2-large-xlsr-53-russian-ONNX";
const processor = await AutoProcessor.from_pretrained(model_id);
const model = await AutoModelForCTC.from_pretrained(model_id, {
  device: "wasm", dtype: "q4",   // q4 is the runnable export; q8/int8 are degenerate here
});

// audio = 16 kHz mono Float32Array
const inputs = await processor(audio);
const { logits } = await model(inputs); // shape [1, frames, 39]  ← 39 Cyrillic symbols

// greedy CTC decode: argmax per frame, merge duplicates, drop the blank (id 0)
const ids  = argmaxPerFrame(logits);    // e.g. [0,0,22,22,0,23,15,...]
const text = collapse(ids);             // "при…" in Cyrillic — no decoder involved

Inference runs in a Web Worker so the UI never janks. The worker returns the raw per-frame argmax and real per-word timings (from the frame each character was emitted at) — nothing is faked or pre-computed.
