BERTimbau — predict the missing Portuguese word

BERTimbau is a BERT trained on brWaC — 2.7 billion tokens of Brazilian Portuguese and nothing else. Its one job here is masked language modelling: hide a word behind [MASK] and it ranks what belongs there — with a Portuguese-native vocabulary, accents, the cedilla and everyday Brazilian words baked in. Type a sentence with a mask and see its guesses, the exact logits it computed, and how it chops Portuguese words into subword pieces. Everything runs on your device.

See inside — logits + how Portuguese splits into subwords

A language model doesn't output words — it outputs a logit (a raw, unbounded score) for every one of its 29,794 Portuguese vocabulary tokens at the masked position. We softmax those into probabilities. The bars above show the probability; the logit beside each is the actual number the network produced. The – below is the entropy of the top-k — low means BERTimbau is sure, high means it's hedging.

A Portuguese-only vocabulary keeps common words whole and breaks longer ones (eletroencefalografista, otorrinolaringologista) into a handful of Portuguese subwords instead of spelling them out letter by letter.

Why it matters

A model trained only on Portuguese reads Portuguese the way a multilingual model can't afford to — every token in its vocabulary is a Portuguese subword, so accented vowels, the cedilla (ç), nasal diphthongs (ão, õe) and everyday Brazilian words (ônibus, feijoada, saudade) come out cleaner. Because it scores every word in context, you can ask it what fits, how strongly, and compare alternatives — all locally, with no prompt engineering and no API bill.

  • Portuguese autocomplete — suggest the next or missing word in a compose box, offline, with Portuguese-native suggestions.
  • Cloze & language drills — generate fill-in-the-blank Portuguese exercises with graded difficulty from the entropy.
  • Register & word-choice checks — compare candidate words at a slot (formal vs colloquial, você vs tu) using the logits.
  • Brazilian vs European insight — see which variant's word the model prefers in a slot, useful for localisation and learner tools.

How the API works

The one-liner uses the fill-mask pipeline:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "luiseduardobrito/bert-base-portuguese-cased-onnx");
const out = await unmask("A capital do Brasil é [MASK].");
// → [{ token_str: "Brasília", score: 0.72, sequence: "…" }, { token_str: "Salvador", … }]

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 29794]), slice the row at the [MASK] position, and softmax it ourselves. That's the only way to show the real logit values, not just the pipeline's rounded scores. All of it runs in a Web Worker so the page never janks. This repo ships only the full-precision ONNX graph, so we load it with dtype: "fp32".
