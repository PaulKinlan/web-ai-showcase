Baguettotron — Pleias's tiny reasoning LLM in your browser

Baguettotron is a small "encyclopedic" reasoning model from Pleias, trained entirely on the synthetic SYNTH dataset. It is unusually shaped: only ~321M parameters, but 80 layers deep — the bet being that depth buys reasoning at a tiny size. It always thinks out loud inside a <think> block, then gives its answer. This page runs it entirely in your browser via Transformers.js: it streams tokens on the universal WebAssembly/CPU path (no GPU required). Nothing is uploaded; there is no API key and no server.

A different family from the other small chat models here (Qwen, Llama, Gemma, Falcon, SmolLM). Its ONNX build exports as the standard llama decoder architecture, so it loads through the ordinary text-generation pipeline — but the model, data, and training recipe are Pleias's own.

Run it

Heads up: the first load transfers the weights (~310 MB), then it's cached. Because it's a deep 80-layer model on the WASM/CPU path, tokens stream at a modest pace — and it spends its first tokens thinking before it answers. That's the honest cost of a GPU-free reasoning model.

Why it matters

Baguettotron is a bet that a tiny, deep model can reason — and that you can run that reasoning locally. Because the weights are fully synthetic (SYNTH) and Apache-2.0, and the model is small enough to cache in a browser, it makes private, offline, auditable reasoning a client-side primitive: it works on a locked-down laptop or an older phone, with no GPU, no key, and nothing sent to a server.

  • Visible chain-of-thought — the <think> block lets you see the model's reasoning, which is useful for teaching, debugging prompts, and trust.
  • On-device reasoning — step-through explanations and small logic puzzles, entirely in the page.
  • Universal reach — ~310 MB q8 runs on the WebAssembly/CPU path; no adapter or driver flags required.
  • Reproducible & open — synthetic training data + open weights make it a clean base for experiments (see the multi-model demo for grounded reasoning).

How the API works

It's the standard Transformers.js text-generation pipeline — pass a messages array (the pipeline applies Baguettotron's ChatML template, which appends the opening <think>), a TextStreamer for token-by-token output, and your sampling settings: 

import { pipeline, TextStreamer } from "@huggingface/transformers";

// q8 runs on the universal WASM/CPU path. (The q4 block-quant build aborts on the
// WASM backend during ORT-Web session creation, so we use q8 everywhere.)
const generator = await pipeline(
  "text-generation", "onnx-community/Baguettotron-ONNX", { device: "wasm", dtype: "q8" },
);

const messages = [
  { role: "system", content: "You are a helpful, concise assistant." },
  { role: "user", content: "What is the capital of France?" },
];

const streamer = new TextStreamer(generator.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });

// The model streams a <think> reasoning trace, then its answer.
await generator(messages, {
  max_new_tokens: 220, do_sample: true, temperature: 0.7, top_p: 0.9, streamer,
});

The "see inside" panel runs one raw forward pass — generator.model(generator.tokenizer(prompt)) — reads the last position's logits, softmaxes them, and shows the real top-k. Everything runs in a Web Worker so generation never blocks the page; the loader auto-inits an already-cached copy and never silently re-downloads.
