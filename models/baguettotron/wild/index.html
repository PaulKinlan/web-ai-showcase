Wild: reasoning race — greedy vs sampled

Sampling doesn't just change wording — for a reasoning model it can change the whole thought process. Give Baguettotron one puzzle and run it twice: once greedy (deterministic, always the top token) and once with sampling (temperature 0.9). Watch the two chains of thought diverge — and see whether they still reach the same answer. Both runs are real, on your device.

What to look for

The greedy run is reproducible — same input, same chain of thought, every time. The sampled run explores different intermediate steps, which can help it escape a bad line of reasoning — or lead it astray. Comparing the two is a hands-on way to feel what temperature actually does to a model that shows its working. Both runs happen sequentially in the same Web Worker, so the page stays responsive.
