text-classification · text · WASM · ~260 MB

Formality detection — read the register, not the sentiment

A RoBERTa fine-tuned to judge register — is this written formally (a report, a cover letter) or informally (a text to a friend)? This is stylistic classification, orthogonal to what the text is about or how positive it feels. Type below and watch the formality meter move, then see which words carry the tone. Nothing leaves your device.

Because there's no quantised build, this model is a ~260 MB fp32 download and needs a few hundred MB of memory to run — comfortable on desktop and modern phones, but heavier than most demos here. It's cached after the first load and never re-downloaded silently.

See inside — which words set the register

This is occlusion attribution: we re-run the model with each word removed and see how much the formal score shifts (in log-odds). Remove a word that was pulling the text formal and the score drops — that word gets a blue weight. Remove a casual word and the score rises — amber. No gradients, just the model's own predictions, N+1 of them. This is what "register cues" look like: "grateful", "convenience" pull formal; "hey", "u", "lol" pull informal.

Why it matters

Register is a different axis from sentiment or topic: the same message can be warm or cold, positive or negative, and still be phrased formally or casually. Scoring it in the browser means a writing tool can react to tone as the user types — privately, with no round-trip.

  • Tone check for professional writing — warn when an email to a client reads too casual (or a note to a teammate reads stiff).
  • Brand-voice guardrails — keep support replies or marketing copy inside a target register.
  • Style-consistent editing — flag the one informal sentence in an otherwise formal document.
  • Dataset curation — filter a corpus by register, entirely client-side.

How the API works

Classification is one call. This repo uses a non-standard file layout, so we point at the exact file:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "text-classification",
  "Deepchecks/roberta_base_formality_ranker_onnx",
  { subfolder: "", model_file_name: "model_optimized", dtype: "fp32" },
); // the ONNX sits at the repo root, not under onnx/

const out = await classify("hey can u take a look?", { top_k: 2 });
// → [{ label: "informal", score: 0.994 }, { label: "formal", score: 0.006 }]

"See inside" re-uses that same call: it classifies the full text plus one copy per word with that word deleted (batched in a single pass), then subtracts the formal log-odds to attribute the shift to each word. All of it runs in a Web Worker so typing never janks.
