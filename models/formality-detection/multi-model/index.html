Multi-model: detect the register, then shift it

A tone-rewrite loop, entirely on-device. First this formality detector reads a message's register. Then the T5 paraphraser rewrites it several ways, and the detector re-scores each rewrite so we can surface the one that best flips it into the register you asked for. Two models passing work back and forth — no text leaves the tab.

Loads two models: the ~260 MB formality detector and the ~235 MB T5 paraphraser (cached after first use). Heavy — best on desktop or a capable phone.

This is a real composition: the paraphraser doesn't "know" about formality, so we generate several rewrites and let the detector judge which one actually moved the register. Both models run in a single Web Worker, sequentially, so the UI stays responsive and each stage lands as it completes.
