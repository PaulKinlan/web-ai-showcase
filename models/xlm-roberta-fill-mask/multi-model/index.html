Multi-model: fill the mask, then name the language

A two-stage on-device pipeline. First XLM-RoBERTa fills your <mask>. Then a second model — itself an XLM-RoBERTa, fine-tuned for language ID — reads the completed sentence and tells you which of 20 languages it's in, with confidence. Same architecture, two jobs, chained locally with nothing leaving the tab.

Models

This demo loads TWO ~280 MB models. The fill-mask model is the same one the other demos use (cached if you've run them); the language detector is separate.

1 · Fill-mask model

2 · Language-ID model

fill –lang-id –backend –

The detector runs on the filled-in sentence, so the chain is honest end-to-end: the first model's output is literally the second model's input.
