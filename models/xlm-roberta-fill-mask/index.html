XLM-RoBERTa — fill the blank in 100 languages

This is one model trained on 2.5 TB of text across 100 languages, sharing a single 250,002-token vocabulary. Its job here is masked language modelling: hide a word behind <mask> — in English, French, Hindi, Swahili, or Japanese — and the same weights rank what belongs there. No language flag, no switch. Type a sentence with a mask and see its guesses plus the exact logits. Everything runs on your device.

See inside — one shared vocabulary, 100 languages

A language model doesn't output words — it outputs a logit (a raw, unbounded score) for every one of its 250,002 vocabulary tokens at the masked position, then we softmax those into probabilities. What makes XLM-RoBERTa special is that this ONE vocabulary is shared across every language: a French sentence and a Hindi sentence draw candidates from the same 250k-token space, and the model has learned where each language's words live in it. The bars above show the probability; the logit beside each is the actual number the network produced. The – below is the entropy of the top-k — low means it's sure, high means it's hedging (common when a slot is grammatical rather than factual).

Why it matters

Most on-device NLP models are English-only. XLM-RoBERTa is a single set of weights that reads ~100 languages — so you ship one model and serve the whole world, offline, with no per-call API bill and nothing leaving the device. Because it scores every word in context, you can ask it what fits, how strongly, and compare alternatives — in any of those languages at once.

  • Multilingual autocomplete — suggest the missing/next word in a compose box that works the same in Portuguese, Polish, or Persian, with no language switch.
  • Cross-lingual knowledge probing — ask "The capital of X is ___" in five languages and see where the model's facts agree (and where they don't).
  • Low-resource language tooling — a cloze/grammar drill generator for languages that have no dedicated on-device model.
  • Multilingual bias auditing — compare predictions across parallel templates in different languages to expose associations baked into web-scale training data.

How the API works

The one-liner uses the fill-mask pipeline — note the RoBERTa-style <mask> token:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "Xenova/xlm-roberta-base");
const out = await unmask("Die Hauptstadt von Deutschland ist <mask>.");
// → [{ token_str: "Berlin", score: 0.22, sequence: "…" }, { token_str: "München", … }]
//   the SAME call works for French, Spanish, Japanese… no language argument

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 250002]), slice the row at the <mask> position, and softmax it ourselves. XLM-RoBERTa uses a SentencePiece tokenizer, so word-initial subwords carry a leading ▁ marker we clean up for display. All of it runs in a Web Worker so the page never janks.
