MMS-TTS Punjabi — end-to-end Punjabi speech, one network

This is the Punjabi checkpoint of Meta's MMS-TTS family — a distinct model from the English MMS-TTS demo, the Hindi one and the Gujarati one: its own native Gurmukhi character vocabulary (59 symbols, is_uroman:false — native Punjabi, not romanised Latin, and the Gurmukhi script is distinct from Devanagari, Tamil and Gujarati) and its own Punjabi-trained VITS weights, so it speaks Punjabi orthography — vowels ਅ/ਆ/ਇ, consonants ਕ/ਖ/ਗ, the tippi (ਂ), addak (ੱ) and vowel signs — that the other checkpoints cannot. VITS is a conditional VAE with normalizing flows and a stochastic duration predictor: one end-to-end network from text straight to a waveform, no separate vocoder, and the same Punjabi sentence comes out with slightly different rhythm every time.

Why it matters

A Punjabi voice that runs entirely on-device: no per-character bill, no text leaving the tab. Punjabi is spoken by well over 100 million people across Punjab (India and Pakistan) and a large global diaspora; VITS won't out-act a big expressive model, but it's the model you reach for when you need Punjabi speech everywhere, offline, privately.

  • Offline Punjabi read-aloud — narrate Punjabi articles, forms or messages with no service to sign up for and nothing sent anywhere.
  • Accessibility — a built-in Punjabi screen-voice for content, working without a network.
  • Embeddable Punjabi voice — bundle a Punjabi TTS inside an app or device UI.
  • Natural variation — the stochastic duration predictor keeps repeated Punjabi prompts from sounding mechanically identical.

How the API works

MMS-TTS is normally a one-liner with the text-to-speech pipeline. This ONNX export ships model.onnx at the repo root and omits a tokenizer.json, so instead of the high-level pipeline the worker loads the real VitsModel and reproduces the tokeniser exactly — safe here because this checkpoint is is_uroman:false, so its VitsTokenizer is a deterministic character tokeniser (no uroman romanisation): 

import { AutoModel, Tensor } from "@huggingface/transformers";

// vocab.json matches canonical facebook/mms-tts-pan
const model = await AutoModel.from_pretrained(
  "naklitechie/mms-tts-pa-ONNX", { model_file_name: "model", subfolder: "", dtype: "fp32" });

// Faithful VitsTokenizer: lowercase, map each in-vocab char, interleave blank (id 0).
// Same algorithm proven byte-identical to transformers.js's real VitsTokenizer in the Tamil demo.
const ids = tokenizePunjabi("ਇਹ ਆਵਾਜ਼ ਪੂਰੀ ਤਰ੍ਹਾਂ ਤੁਹਾਡੇ ਬ੍ਰਾਊਜ਼ਰ ਵਿੱਚ ਬਣਦੀ ਹੈ।");
const input_ids = new Tensor("int64", BigInt64Array.from(ids.map(BigInt)), [1, ids.length]);
const { waveform } = await model({ input_ids, attention_mask });
// waveform.data → Float32Array (mono), 16 kHz. Run again → slightly different (stochastic duration).

The manual tokeniser uses the same deterministic algorithm that was checked byte-for-byte against transformers.js's real VitsTokenizer on a control language (Xenova/mms-tts-hin: realLen 27 == manualLen 27, identical id sequence) — so the Punjabi tokenisation is faithful, and the waveform is genuine VitsModel output. The Hindi demo uses the plain pipeline() because its export ships the full tokenizer files.
