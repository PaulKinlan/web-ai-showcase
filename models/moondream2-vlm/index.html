Moondream2 — ask a small model about a picture

Moondream2 is a compact vision-language model with a reputation punching well above its size: give it an image and a question and it answers in plain language. Describe a scene, read a receipt, count objects, reason about what's happening — streaming token by token on your GPU, with nothing uploaded.

Moondream doesn't use a chat template like larger VLMs — it frames the task in plain text. Your image is turned into a block of image tokens that fill the <image> placeholder, and your words are wrapped in a simple Question: … Answer: frame. Why it matters

Moondream2 shows how far a small VLM has come: a couple of gigabytes of weights, running in a browser tab, that can read text in a photo and answer open questions about it — privately, offline-capable, and free to run.

  • On-device document triage — read a photographed receipt, label, or form and pull out the numbers, without the image ever leaving the device.
  • Screenshot Q&A — let users ask questions about a chart, dashboard, or UI screenshot right inside your app.
  • Automatic alt-text — generate a description for every image on a page for screen-reader users, with no per-image API cost.
  • Camera copilots — point-and-ask experiences ("what does this sign say?", "is this ripe?") with zero server round-trips.

How the API works

Moondream in Transformers.js uses a dedicated model class. You tokenize a simple Question/Answer prompt, run the image through the processor, merge the two, then generate — streaming tokens as they come: 

import { AutoProcessor, AutoTokenizer, Moondream1ForConditionalGeneration, RawImage, TextStreamer }
  from "@huggingface/transformers";

const model_id = "Xenova/moondream2";
const tokenizer = await AutoTokenizer.from_pretrained(model_id);
const processor = await AutoProcessor.from_pretrained(model_id);
const model = await Moondream1ForConditionalGeneration.from_pretrained(model_id, {
  dtype: { embed_tokens: "fp16", vision_encoder: "fp16", decoder_model_merged: "q4f16" },
  device: "webgpu",                        // WebGPU required
});

const text = `<image>\n\nQuestion: ${prompt}\n\nAnswer:`;
const text_inputs = tokenizer(text);
const image = await RawImage.fromURL(imageURL);
const vision_inputs = await processor(image);

const streamer = new TextStreamer(tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });
await model.generate({ ...text_inputs, ...vision_inputs, do_sample: false, max_new_tokens: 150, streamer });

This page runs exactly that, in a Web Worker, and gates on a real WebGPU adapter first — if none exists it shows a labelled needs-WebGPU state instead of a fake answer.
