FinBERT — sentiment tuned for financial text

A BERT further pre-trained on a financial corpus and fine-tuned on the Financial PhraseBank, it scores a headline or sentence as positive, negative or neutral — from a market participant's point of view. "Shares tumbled after the firm slashed its outlook" reads clearly negative to FinBERT, where a movie-review sentiment model often shrugs and calls it neutral. Paste some finance text and watch the three-way verdict, then look inside to see which words moved it. Nothing leaves your device.

Signal, not advice. FinBERT reports the tone of a sentence, not a buy/sell recommendation or a price prediction. It knows nothing after its training data, and sarcasm, hedging and forward-looking spin can all fool it — treat every score as one noisy input, never a trade.

Which words moved the needle

This is occlusion attribution: we re-run the model with each word removed and measure how much the net sentiment (positive minus negative, in log-odds) shifts. Remove a word that was pushing bullish and the signal drops — green. Remove a bearish word and it rises — red. No gradients: just N+1 forward passes, batched in a Web Worker so typing never janks.

The domain caveat. FinBERT was tuned on short, formal financial statements. It shines on headlines and earnings language; it is weaker on tweets, slang, heavy sarcasm, and long multi-clause paragraphs — and it has no notion of magnitude or causation, only tone. The head-to-head demo makes the domain gap vivid against a general model.

Why it matters

General sentiment models learn tone from product and movie reviews, so they miss the vocabulary of markets: "impairment", "guidance cut", "beat", "headwind", "buyback". A domain-tuned model reads that language the way an analyst does — and doing it in the browser means you can triage a feed as it streams, privately, with no data leaving the tab.

  • News & earnings triage — colour a headline feed by tone to surface what needs a human's eyes first (signal, not advice).
  • Private analyst desks — score confidential notes and drafts client-side; nothing goes to a server.
  • Tone drift dashboards — track how sentiment around a name shifts across a batch of filings or transcripts.
  • Labelling & dataset triage — pre-sort a finance text corpus into positive / negative / neutral before human review.

How the API works

Classification is one call; top_k: 3 returns every class score:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "text-classification",
  "Xenova/finbert",
); // downloads + caches ~111 MB once

const out = await classify("Shares tumbled after the firm cut its outlook.", { top_k: 3 });
// → [ { label: "negative", score: 0.76 },
//     { label: "neutral",  score: 0.14 },
//     { label: "positive", score: 0.10 } ]

Pass top_k: 3 (or null) to get all three classes — the default returns only the winner. "See inside" reuses the same call: it classifies the full text plus one copy per word with that word deleted (batched in a single pass), then attributes the net-sentiment shift to each word. It all runs in a Web Worker so the control UI stays responsive.
