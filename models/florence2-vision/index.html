Florence-2 — one model, every vision task

Florence-2 is a vision foundation model steered entirely by a task prompt. The same 230M-parameter network captions an image, detects and boxes objects, reads text with OCR, and segments a thing you name — you just change the task token. Pick an image, pick a task, and watch it run on your GPU with nothing uploaded.

Why it matters

Florence-2 collapses a whole computer-vision toolbox into one small download. Instead of shipping a captioner, a detector, an OCR engine, and a segmenter — each with its own model and server — you ship one file and switch tasks with a token, all on-device.

  • Alt-text & captioning — describe every image on a page for accessibility, privately.
  • On-device OCR — read receipts, signs, and screenshots without an OCR API.
  • Detection & counting — box and count objects for moderation, inventory, or analytics.
  • Grounded selection — "segment the red car" for smart cut-out and annotation tools.

How the API works

Florence-2 uses the dedicated vision-seq2seq class. You build the prompt from a task token, run the processor over the image and prompt, generate, decode, then post-process into the shape that task produces: 

import { Florence2ForConditionalGeneration, AutoProcessor, load_image }
  from "@huggingface/transformers";

const model = await Florence2ForConditionalGeneration.from_pretrained(
  "onnx-community/Florence-2-base-ft", { dtype: "fp16", device: "webgpu" });   // WebGPU required
const processor = await AutoProcessor.from_pretrained("onnx-community/Florence-2-base-ft");

const image = await load_image(imageURL);
const task = "<OD>";                              // or <CAPTION>, <OCR>, <REFERRING_EXPRESSION_SEGMENTATION>…
const prompts = processor.construct_prompts(task);
const inputs = await processor(image, prompts);

const ids = await model.generate({ ...inputs, max_new_tokens: 512 });
const raw = processor.batch_decode(ids, { skip_special_tokens: false })[0];
const result = processor.post_process_generation(raw, task, image.size);
// result["<OD>"] = { bboxes: [[x1,y1,x2,y2], …], labels: [ … ] }

This page runs exactly that in a Web Worker, and gates on a real WebGPU adapter first — with none it shows a labelled needs-WebGPU state instead of a fake result.
