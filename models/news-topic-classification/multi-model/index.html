Multi-model: route an article, then summarise it

A newsroom assistant in two hops, both on-device. First the dedicated topic head routes an article to its desk. Then DistilBART-CNN writes a short desk-ready TL;DR. Paste an article and watch the pipeline run — classification then summarisation — with no text leaving the tab.

Loads two models: the ~13 MB topic head and the ~250 MB DistilBART summariser (cached after first use).

This is a real composition: the classifier's output (the desk) could pick which summariser prompt, length, or downstream editor a story goes to. Here both models run in a single Web Worker, sequentially, so the UI stays responsive and you see each stage land as it completes.
