wav2vec2-XLSR-53 — Thai speech, one character per frame

The English wav2vec2 demo speaks one language, the Russian XLSR demo another, the Chinese and Japanese demos two more. This is another specialisation of the same cross-lingual cousin: XLSR-53 was pre-trained on 53 languages (56,000 hours of multilingual audio) and then fine-tuned on Thai. That shared foundation is what makes a solid Thai recogniser trainable from a modest amount of labelled speech. Same CTC mechanics as the other models, but the output alphabet is the Thai script — consonants, vowels and tone marks. Thai is written without spaces between words, so the model also learned a | word-boundary symbol, which we render as a space. Pick a clip and watch the frames collapse into Thai.

Why it matters

Most of the world's ~7,000 languages have little labelled speech data. Cross-lingual self-supervised pre-training is how the field reaches them: XLSR-53 learns speech structure from 53 languages at once, so fine-tuning to a new language needs far fewer transcripts. That is what puts a usable Thai recogniser inside a browser tab — private, offline once cached, no server, no key. And because it's CTC (frames, not generated tokens) it can't hallucinate an unspoken sentence, and every character is pinned to a moment in time.

  • On-device Thai dictation and captions — for apps that must keep audio on the device (health, legal, journalism) in Thai-speaking markets.
  • Word segmentation for free — Thai has no spaces; this model's | boundary output is a first pass at splitting the transcript into words.
  • Thai forced alignment — real per-word timings for karaoke, subtitles, and pronunciation feedback, straight from the frames.
  • Privacy-first field work — transcribe sensitive Thai audio with nothing leaving the device.

How the API works

Normally the one-liner is the ASR pipeline — but this repo ships no config.json or vocab and keeps its ONNX at the repo root under a non-standard filename, so we load the model with a config borrowed from the base repo, normalise the audio ourselves, and do the CTC collapse (which is exactly what the pipeline does internally): 

import { AutoConfig, AutoModelForCTC, Tensor } from "@huggingface/transformers";

const model_id = "BlackHand013/Wav2Vec2-large-xlsr-53-th-onnx";
// the export repo has no config.json → borrow the architecture config from the base repo
const config = await AutoConfig.from_pretrained("airesearch/wav2vec2-large-xlsr-53-th");
const model = await AutoModelForCTC.from_pretrained(model_id, {
  device: "wasm", dtype: "fp32",              // the only export is a single fp32 file…
  subfolder: "",                              // …at the repo ROOT…
  model_file_name: "wav2vec2-large-xlsr-53-th", // …under this non-standard name
  config,
});

// audio = 16 kHz mono Float32Array → normalise to zero mean / unit variance (wav2vec2 feature step)
const x = normalize(audio);
// NOTE: this graph's input node is named `input`, not the usual `input_values`
const { logits } = await model({ input: new Tensor("float32", x, [1, x.length]) });
// logits shape [1, frames, 70]  ← 70 Thai symbols

// greedy CTC decode: argmax per frame, merge duplicates, drop the blank ([PAD]=69); | → space
const ids  = argmaxPerFrame(logits);
const text = collapse(ids);             // Thai words, space-separated at | boundaries

Inference runs in a Web Worker so the UI never janks. The worker returns the raw per-frame argmax and real per-word timings (from the frame each character was emitted at) — nothing is faked or pre-computed.
