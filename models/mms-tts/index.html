MMS-TTS — end-to-end speech, one network, many languages

MMS-TTS is a VITS model: a single, end-to-end network that turns text straight into a waveform — no separate vocoder stage. It's a different beast from Kokoro (a StyleTTS2 decoder) and SpeechT5 (encoder-decoder + HiFi-GAN): VITS is a conditional VAE with normalizing flows and a stochastic duration predictor, so the same sentence comes out with slightly different rhythm every time. This page runs the English checkpoint — one of 1000+ languages Meta released in the same family.

Why it matters

VITS is the pragmatic workhorse of open TTS: one self-contained network, small, CPU-friendly, and — via MMS — available in more than a thousand languages from a single recipe. It won't out-act a big expressive model, but it's the model you reach for when you need speech everywhere, offline, with no per-character bill and nothing leaving the device.

  • Speech in the long tail of languages — the same code path serves 1000+ languages, including many with no commercial TTS at all.
  • Offline, private read-aloud — narrate articles, forms, or messages with no service to sign up for and no text sent anywhere.
  • Tiny, embeddable voice — ~38 MB per language makes VITS practical to ship inside a PWA or kiosk.
  • Natural variation — the stochastic duration predictor keeps repeated prompts from sounding mechanically identical.

How the API works

MMS-TTS is a one-liner with the text-to-speech pipeline — pick the language by its checkpoint:

import { pipeline } from "@huggingface/transformers";

const tts = await pipeline("text-to-speech", "Xenova/mms-tts-eng", { dtype: "q8" });

const out = await tts("One network turns letters directly into a sound wave.");
// out.audio         → Float32Array waveform (mono)
// out.sampling_rate → 16000
// Run it again with the same text → a slightly different waveform (stochastic duration).

This page runs the pipeline inside a Web Worker so synthesis never blocks the interface, encodes the returned Float32 samples to a WAV blob for the <audio> element, and draws the same samples as the waveform under "See inside". Switch language by loading a different mms-tts-<iso> checkpoint — same call, same shape.
