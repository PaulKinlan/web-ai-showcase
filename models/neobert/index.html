NeoBERT — a next-generation BERT encoder

BERT is from 2018. NeoBERT (2025) rebuilds the encoder from scratch with the modern transformer stack: rotary position embeddings, SwiGLU feed-forwards, a depth-over-width shape (28 layers, ~250M params), and a 4,096-token context — 8× classic BERT's 512. It's the backbone you'd fine-tune into the next retriever or classifier. Here it turns text into 768-number vectors so you can measure meaning: build a similarity matrix, search a corpus, and see how a 2025 encoder differs from a 2024 and a 2020 one. Everything runs on your device.

See inside — the vector, and the architecture behind it

The encoder emits one 768-number vector per token; we mean-pool them into a single sentence vector and L2-normalize it, so every sentence becomes a unit-length arrow and cosine similarity is a plain dot product. Pick a passage to see its real vector and its pre-normalization magnitude.

What makes it "next-gen"

Positions
RoPE

Rotary embeddings — relative, extrapolates past training length. Classic BERT used learned absolute positions capped at 512.

Feed-forward
SwiGLU

Gated activation in pre-norm blocks — the same stack modern LLMs use, replacing BERT's GELU MLP.

Shape
28 × 768

Depth over width — 28 layers, ~250M params, a compute-optimal aspect ratio.

Context
4,096

8× classic BERT's 512-token window, trained on the RefinedWeb corpus.

Three generations of encoder on the same pairs

NeoBERT (2025, RoPE/SwiGLU base encoder), GTE-ModernBERT (2024, contrastively fine-tuned) and MiniLM (2020 classic BERT, distilled). Each embeds the same query→passage pairs on your device. Read the GAP between "relevant" and "unrelated", not the absolute number: a base encoder like NeoBERT compresses cosines into a high, narrow band, while a fine-tuned embedder spreads them — different jobs, both real. Loads GTE-ModernBERT (~150 MB) and MiniLM (~25 MB) on demand.

Why it matters

Encoders are the quiet workhorses of NLP: retrieval, reranking, classification, clustering and RAG all start by turning text into a vector. Most production embedders still ride a 2018 BERT backbone. NeoBERT shows what a modern rebuild buys — a longer context, a better-scaling shape, and the same architectural tricks that powered the LLM era — in a package small enough to run in a tab. As a base model it's the foundation you fine-tune; seeing it embed zero-shot makes that foundation legible.

  • Long-document embedding — its 4k context takes whole articles or transcripts without the 512-token truncation classic BERT forces.
  • A modern backbone to fine-tune — start retrieval / classification / reranking projects on a 2025 encoder instead of a 2018 one.
  • On-device semantic search & dedup — cluster and shortlist text client-side, private because nothing leaves the device.
  • Encoder A/B research — measure how architecture (RoPE, SwiGLU, depth) changes the geometry versus the models you already use.

Basics

Practical

Semantic search over your own corpus, plus near-duplicate detection at a threshold.

Multi-model

NeoBERT vs GTE-ModernBERT vs MiniLM — three encoder generations ranking the same corpus.

How the API works

Feature-extraction with mean pooling — the base encoder gives token vectors, we pool + normalize:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "onnx-community/NeoBERT-ONNX",
); // downloads + caches ~223 MB once

const out = await extract(
  ["how do I sign in?", "the login page rejects my password"],
  { pooling: "mean", normalize: true },
);
const [a, b] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = a.reduce((s, x, i) => s + x * b[i], 0);

This page asks the worker for the un-normalized vectors so "See inside" can show the real pre-normalization magnitude, then normalizes them in JS. Inference runs in a Web Worker, so the matrix, projection and comparison never jank the page.
