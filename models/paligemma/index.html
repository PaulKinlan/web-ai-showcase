PaliGemma 2 — one model, many vision tasks via prefixes

PaliGemma is Google's task-steerable VLM: instead of a chat, you give it a short prefix — caption en, ocr, detect car, segment dog, answer en … — and it captions, reads text, locates objects (with box coordinates), or answers a visual question. All streaming, on your GPU, nothing uploaded.

Made by Paul Kinlan · Model runs locally in your browser by 🤗 Transformers.js · Source on GitHub

Why it matters

Most VLMs do one thing (chat about an image). PaliGemma is a Swiss-army vision model: the same weights caption, read, locate, and answer — you just change the prefix. Running that in a browser tab turns a stack of separate vision services into one on-device primitive.

  • Object detection without a detector — detect {thing} returns box coordinates you can draw, no bespoke detection model or labels.
  • Captioning + OCR + VQA in one download — swap the prefix instead of shipping three models.
  • Open-vocabulary localisation — detect or segment things a fixed-class detector was never trained on, by just naming them.
  • Private, offline visual tooling — receipts, product photos, screenshots analysed with zero server round-trips or per-call cost.

How the API works

PaliGemma in Transformers.js uses a dedicated model class. You pass the image and a task-prefix prompt to the processor, then generate — streaming tokens as they come: 

import { AutoProcessor, PaliGemmaForConditionalGeneration, load_image, TextStreamer }
  from "@huggingface/transformers";

const model_id = "onnx-community/paligemma2-3b-pt-224";
const processor = await AutoProcessor.from_pretrained(model_id);
const model = await PaliGemmaForConditionalGeneration.from_pretrained(model_id, {
  dtype: { embed_tokens: "q8", vision_encoder: "fp16", decoder_model_merged: "q4f16" },
  device: "webgpu",                        // WebGPU required
});

const image = await load_image(imageURL);
const prompt = "<image>caption en";  // or "<image>ocr", "<image>detect car", "<image>answer en …"
const inputs = await processor(image, prompt);

const streamer = new TextStreamer(processor.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });
await model.generate({ ...inputs, max_new_tokens: 100, do_sample: false, streamer });

This page runs exactly that, in a Web Worker, and gates on a real WebGPU adapter first — if none exists it shows a labelled needs-WebGPU state instead of a fake answer.
