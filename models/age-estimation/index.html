Age estimation — a soft signal, read on your device

A Vision Transformer, fine-tuned on faces, takes a single frame and scores it across 9 age buckets — 0-2, 3-9, 10-19, 20-29, 30-39, 40-49, 50-59, 60-69, 70+ — then folds the distribution into one expected-age number. Drop a photo or use your camera; the read happens entirely in this tab. No frame is ever uploaded or stored.

Be honest about what this is: a guess at apparent age from pixels. It is imperfect, it carries demographic bias (accuracy varies across skin tone, lighting, expression, and makeup), and it is not identity, verification, or a birth date. Treat it as a soft signal for low-stakes, consented UX — never a verdict about a person. The page shows the real output, right or wrong — never a canned answer.

See inside

The ViT's final layer emits one raw score — a logit — per bucket. A softmax turns the 9 logits into probabilities that sum to 100%. The expected age above is Σ (probability × bucket midpoint) — it respects the whole distribution, not just the top bucket, which is why it can land between the bucket edges. Two numbers describe the shape of that distribution: the top-1 margin (how far the winning bucket leads the runner-up) and the normalised entropy (0 = certain, 1 = torn between buckets). A clear, well-lit face is peaky and low-entropy; an occluded, angled, or blended one is flat and high-entropy — exactly when you should NOT trust a single number.

Why it matters

"Estimate age from a face" used to mean uploading the photo to a cloud API — a privacy non-starter for something this sensitive. Running the model locally flips that: the frame never leaves the device, so age-aware UX becomes something you can actually ship without collecting a single image.

  • Age-appropriate UX (as a signal) — nudge toward a simpler or safer default when the estimate skews young/old, then let the user confirm — never a hard gate on the number alone.
  • Analytics without faces — approximate audience age distribution for a kiosk or display entirely on-device, storing only the bucket, never the image.
  • Consented, private demos — "guess my age" games, avatar age-matching, or research tools where the raw frame stays local.
  • Calibration-gated decisions — use the entropy/margin read-out to act only on confident reads and defer ambiguous ones to a human or an explicit prompt.

A note on ethics + the law: apparent-age models are imprecise and biased, and real age-verification (for restricted goods, adult content, or legal thresholds) has specific regulatory requirements this cannot meet. Use it for consented, low-stakes UX — never surveillance, discrimination, or as the sole basis for a legal age decision.

How the API works

The convenience path is two lines with Transformers.js:

import { pipeline } from "@huggingface/transformers";

const estimateAge = await pipeline(
  "image-classification",
  "jdp8/vit-age-classifier",
); // downloads + caches ~88 MB once

const out = await estimateAge(imageOrCanvasURL, { top_k: 9 });
// → [{ label: "20-29", score: 0.41 }, { label: "30-39", score: 0.22 }, … ]

This page goes one level deeper for "See inside": instead of the pipeline it runs the model's own processor then model() in a single forward pass, so it can read the raw logits tensor and compute the softmax, the probability-weighted expected age, and the entropy/margin itself. Webcam frames are captured to a <canvas> and classified as a data URL — nothing is uploaded. Inference runs in a Web Worker so the camera preview never stutters.
