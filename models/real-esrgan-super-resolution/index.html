Real-ESRGAN — enlarge a small image 4×

Give this model a small or compressed image and it reconstructs a 4× larger one with real detail — sharp edges, recovered texture, cleaned-up JPEG blocking. Real-ESRGAN (ICCVW 2021) is a GAN trained on real-world degradation, so it does far more than a bicubic enlargement: it synthesises plausible high-frequency detail. It's a different architecture from the built Swin2SR (a GAN, not a transformer, and 4× not 2×). Runs entirely on your device via ONNX Runtime Web; the image never leaves the tab.

How to read it

Bright indigo = big gain (an edge or texture the GAN rebuilt); dark = the two upscales agree. A simple sharpen filter would just amplify existing contrast; Real-ESRGAN invents structurally-plausible detail, which is why the map traces objects and textures, not uniform noise.

Why it matters

Low-resolution is usually a dead end — you can't re-shoot the source. On-device super-resolution means you can enlarge it privately, with no upload and no per-image cloud cost:

  • Photo apps — a client-side "enhance / enlarge" button that runs on the user's phone or laptop, no server.
  • E-commerce & media — upscale tiny product thumbnails or legacy assets to modern display sizes in a static web tool.
  • Archives & scanning — batch-enlarge small scans and rescue over-compressed images with zero backend.
  • Pre-processing for other models — enlarge a thumbnail so a detector, OCR, or classifier has enough pixels to work with.

Real-ESRGAN reconstructs plausible detail; it can't recover information the downscaling erased entirely. On faces it may over-smooth — pair it with a face-restoration model (see the multi-model demo). It's an enhancement, not a source of ground truth.

Shrink a photo to a thumbnail, then watch Real-ESRGAN enlarge it 4× — wipe between bicubic and Real-ESRGAN and see the sharpness jump.

Multi-model

Enlarge → colourise: upscale a tiny old photo with Real-ESRGAN, then run DDColor on the result.

How the API works

transformers.js has no Real-ESRGAN (RRDBNet) class, so we run the ONNX graph directly with ONNX Runtime Web. In goes the small image, out comes a 4× larger one:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(esrganBytes, { executionProviders: ["wasm"] });

// small image → [1,3,h,w] float32 in [0,1] (plain RGB, no normalisation)
const input = new ort.Tensor("float32", toNCHW(small, h, w), [1, 3, h, w]);
const { [session.outputNames[0]]: big } = await session.run({ [session.inputNames[0]]: input });

// big is [1,3,4h,4w] in [0,1] → clamp to RGBA (a 4× larger image)
const upscaled = tensorToImage(big.data, 4 * w, 4 * h);

Inference and the whole output composite (clamp, bicubic baseline, detail-gain map) run in a Web Worker so the UI never blocks, and the 67 MB model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
