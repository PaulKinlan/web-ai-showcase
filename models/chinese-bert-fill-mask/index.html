Chinese BERT — predict the missing character

Google's BERT, pre-trained on Chinese Wikipedia. Its one job here is masked language modelling: hide a character behind [MASK] and it ranks which Hànzì belongs there. Chinese BERT is character-level — every character is its own token, so it needs no word-segmentation step. Type a sentence with a mask and see its guesses, plus the exact logits it computed. Everything runs on your device.

See inside — character tokens & logits

First, how the text is tokenised: Chinese BERT splits your sentence into one chip per character — no word segmentation, no sub-word merging. The ▢ chip is the position being predicted.

And the prediction itself: the model outputs a logit (a raw, unbounded score) for every one of its 21,128 vocabulary characters at the masked position. We softmax those into probabilities. The bars above show the probability; the logit beside each is the actual number the network produced. The – below is the entropy of the top-k — low means BERT is sure which character fits, high means it's hedging.

Why it matters

Chinese has no spaces between words, so most NLP starts with the hard problem of segmentation. A character-level masked model sidesteps that entirely: it reasons one Hànzì at a time, in context. Because it scores every character in context, you can ask it what fits, how strongly, and compare alternatives — all locally, with no prompt engineering and no API bill.

  • Input-method assist — rank the most likely next character for a Chinese IME or compose box, offline.
  • Typo & OCR correction — mask a suspicious character and see what the model expects instead.
  • Cloze & HSK drills — generate fill-in-the-blank Chinese exercises graded by the model's confidence.
  • Chengyu knowledge probing — test whether the model has memorised four-character idioms.

How the API works

The one-liner uses the fill-mask pipeline:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "Xenova/bert-base-chinese");
const out = await unmask("北京是中国的[MASK]都。");
// → [{ token_str: "首", score: 0.82, sequence: "…" }, { token_str: "古", … }]  → 首都 = "capital"

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 21128]), slice the row at the [MASK] position, and softmax it ourselves. That's the only way to show the real logit values and the character-by-character tokenisation, not just the pipeline's rounded scores. All of it runs in a Web Worker so the page never janks.
