wav2vec2-XLSR-53 — Spanish speech, one accented character per frame

The English wav2vec2 demo speaks one language, the Russian XLSR demo another, and the French XLSR demo a third. This is a fourth specialisation of the same cross-lingual cousin: XLSR-53 was pre-trained on 53 languages (56,000 hours of multilingual audio) and then fine-tuned on Spanish. That shared foundation is what makes a solid Spanish recogniser trainable from a modest amount of labelled speech — the model already learned what human speech sounds like before it ever saw a Spanish transcript. Same CTC mechanics as the English, Russian and French models, but the output alphabet carries the Spanish diacritics (á é í ó ú ü) and ñ. Pick a clip and watch the frames collapse into Spanish.

Why it matters

Most of the world's ~7,000 languages have little labelled speech data. Cross-lingual self-supervised pre-training is how the field reaches them: XLSR-53 learns speech structure from 53 languages at once, so fine-tuning to a new language needs far fewer transcripts. That is what puts a usable Spanish recogniser inside a browser tab — private, offline once cached, no server, no key. And because it's CTC (frames, not generated tokens) it can't hallucinate an unspoken sentence, and every character is pinned to a moment in time.

  • On-device Spanish dictation and captions — for apps that must keep audio on the device (health, legal, journalism) across Spanish-speaking markets.
  • One backbone, many languages — the same XLSR-53 recipe produces the English, Russian, French and Spanish recognisers in this showcase, plus dozens more fine-tunes.
  • Spanish forced alignment — real per-word timings for karaoke, subtitles, and pronunciation feedback, straight from the frames.
  • Privacy-first field work — transcribe sensitive Spanish audio with nothing leaving the device.

How the API works

Normally the one-liner is the ASR pipeline — but this repo ships no config.json or preprocessor_config.json and keeps its ONNX at the repo root, so we load the model with a config borrowed from the base repo, normalise the audio ourselves, and do the CTC collapse (which is exactly what the pipeline does internally): 

import { AutoConfig, AutoModelForCTC, Tensor } from "@huggingface/transformers";

const model_id = "FinDIT-Studio/wav2vec2-large-xlsr-53-spanish-onnx";
// the export repo has no config.json → borrow the architecture config from the base repo
const config = await AutoConfig.from_pretrained("jonatasgrosman/wav2vec2-large-xlsr-53-spanish");
const model = await AutoModelForCTC.from_pretrained(model_id, {
  device: "wasm", dtype: "fp32",   // the only export is a single fp32 model.onnx…
  subfolder: "",                   // …at the repo ROOT, not the usual onnx/ subfolder
  config,
});

// audio = 16 kHz mono Float32Array → normalise to zero mean / unit variance (wav2vec2 feature step)
const x = normalize(audio);
const { logits } = await model({ input_values: new Tensor("float32", x, [1, x.length]) });
// logits shape [1, frames, 41]  ← 41 Latin-Spanish symbols

// greedy CTC decode: argmax per frame, merge duplicates, drop the blank (id 0)
const ids  = argmaxPerFrame(logits);    // e.g. [0,0,19,19,0,7,20,21,...]
const text = collapse(ids);             // "mano…" in Spanish — no decoder involved

Inference runs in a Web Worker so the UI never janks. The worker returns the raw per-frame argmax and real per-word timings (from the frame each character was emitted at) — nothing is faked or pre-computed.
