GTE-small — a tiny embedder that ranks with the big ones

GTE (General Text Embeddings, from Alibaba) turns a sentence into 384 numbers — a point in "meaning space" — and does it so well that this 33-million-parameter model outscores far larger ones on the MTEB retrieval benchmark. Unlike BGE or E5, it needs no query prefix: you embed a question and a document the exact same way. Type a few lines to build a live similarity matrix, search by meaning, and watch three different embedders disagree on the same pairs. Nothing leaves your device.

See inside — the vector, and why the model choice matters

The encoder emits one 384-number vector per token. To get a single vector per sentence GTE mean-pools those token vectors (mask-aware) and L2-normalizes, so every sentence becomes a unit-length arrow and cosine similarity is a plain dot product. Pick a sentence to see its real vector and its pre-normalization magnitude.

MiniLM vs BGE vs GTE on the same pairs

Three embedders, three training recipes: MiniLM (mean pool), BGE (CLS pool, prefix-tuned for retrieval) and GTE (mean pool, no prefix). They do not agree on how similar two sentences are — that disagreement is exactly why the model you pick changes your search results. Why it matters

Embeddings are the quiet workhorse behind search, recommendations, deduplication, clustering, and retrieval-augmented generation. GTE's pitch is efficiency: near-top MTEB retrieval quality from a 33M-param model with no prefix ritual — small enough to run in a tab, good enough to trust for real ranking, and private because the text never leaves the device.

  • Semantic search — rank a corpus by meaning, not keyword overlap, entirely client-side.
  • Nearest-label classification — embed your labels once, then tag new text by closest label with zero training.
  • Clustering & dedup — group "same thing, different words" and collapse near-duplicates.
  • Private RAG retrieval — fetch the right snippet to feed a local LLM, all on-device.

How the API works

Feature-extraction plus the pooling discipline is the whole recipe — and note there is no prefix:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Xenova/gte-small",
); // downloads + caches ~34 MB once

// GTE uses MEAN pooling and needs NO instruction prefix — embed query and doc identically:
const out = await extract(["how do I sign in?", "the login page rejects my password"], {
  pooling: "mean",
  normalize: true,
});
const [q, d] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = q.reduce((s, x, i) => s + x * d[i], 0); // high — same meaning

This page asks the worker for the un-normalized pooled vectors so "See inside" can show the real pre-normalization magnitude, then normalizes them in JS. Inference runs in a Web Worker, so the matrix, projection and comparison never jank the page.
