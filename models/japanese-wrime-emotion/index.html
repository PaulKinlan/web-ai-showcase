Japanese WRIME emotion — the feeling behind Japanese text

Most emotion models only read English, and most Japanese BERTs can't even run in a browser — they need a MeCab morphological analyser the browser doesn't have. This one is different on both counts: it's a Japanese-native model (Ruri v3, a Japanese ModernBERT) fine-tuned on WRIME, the Japanese emotion corpus, and its tokenizer is SentencePiece — so it runs fully on your device. Type Japanese below and it scores the text across Plutchik's eight emotions: 喜び joy, 悲しみ sadness, 期待 anticipation, 驚き surprise, 怒り anger, 恐れ fear, 嫌悪 disgust, 信頼 trust. Watch the real probabilities, and look inside to see how it reads a spaceless sentence.

See inside — subword pieces & which of them moved the emotion

Japanese is written without spaces, so the first thing the model does is split your sentence into SentencePiece pieces — the dashed chips below marked ▁ are where the tokenizer decided a new unit begins. This is the step a MeCab-based Japanese BERT would need a native analyser for; SentencePiece does it in pure WebAssembly, which is why this model runs in a tab.

To make the model's reasoning legible we run occlusion attribution: re-score the text with each piece removed and measure how far the winning emotion's confidence drops (in log-odds). A piece tinted in the emotion's colour was pushing that emotion; a grey piece was pulling against it. Because the tokenizer — not a word list — decides the units, it works on Japanese with no segmentation heuristics.

Why it matters

Sentiment as "positive / negative" throws away most of what a message carries. The difference between an angry complaint, a fearful one and a disgusted one changes how you'd route or answer it — and in Japanese, where politeness and indirection hide a lot of the signal, a model trained on Japanese social text reads cues a translated English model misses. That this one runs entirely on-device means emotion analysis of private Japanese text — reviews, DMs, support threads — never has to leave the browser.

  • Fine-grained feedback triage — separate angry from anxious from disappointed Japanese reviews, not just "negative".
  • Private journalling & wellbeing — chart the emotional tone of a Japanese diary locally, nothing uploaded.
  • Character & dialogue tooling — score the emotion of lines in a Japanese script or game, offline.
  • Community health signals — flag a fearful or hostile turn in a Japanese thread, on the client.

How the API works

The one-liner uses the text-classification pipeline:

import { pipeline } from "@huggingface/transformers";

const clf = await pipeline(
  "text-classification",
  "arajun/ruri-v3-30m-wrime-onnx",
  { dtype: "fp32" },
); // downloads + caches ~156 MB once

const out = await clf("今日はとても嬉しい気分です。", { top_k: 8 });
// → [{ label: "喜び", score: 0.93 }, { label: "信頼", score: 0.04 }, … ]

For the "see inside" surface we load one level lower — AutoModelForSequenceClassification — so we can read the exact logits tensor ([1, 8]), softmax it ourselves to show every emotion's probability plus its raw logit, and reuse the loaded instance to re-score occluded variants for per-piece attribution. All of it runs in a Web Worker so the page never janks. The model loads with dtype: "fp32" because the WebAssembly backend has no fp16 compute kernel.
