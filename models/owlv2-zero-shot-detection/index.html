OWLv2 — detect anything you can describe in an image

Ordinary detectors only know the classes they were trained on. OWLv2 is open-vocabulary: you type the objects you want as free text — a cat, a remote control, a person wearing red — and it embeds each phrase and each image region into a shared space, then boxes the regions that match. New label set, no retraining, all on your device.

Why it matters

Every fixed-class detector forces the same question: "is your object one of the 80 it knows?" If not, you collect a dataset and fine-tune. OWLv2 removes that step — the class list is an input, typed at runtime, in the browser, for free. That turns bespoke detection from a training project into a text box.

  • Custom on-device inventory / inspection — find "a forklift", "a safety helmet", or "an empty shelf" without training a model per site, and with nothing uploaded.
  • Zero-setup visual search — let users describe what to find in their own photos ("my passport", "a red mug") and box it locally.
  • Content triage & moderation — box regions matching policy phrases before an image is shared, all client-side.
  • Region proposals for other models — OWLv2 finds "where"; hand each region to a segmenter or captioner for "what exactly", chaining tiny local models.

How the API works

Open-vocabulary detection is a one-liner with Transformers.js — the query list is just an argument:

import { pipeline } from "@huggingface/transformers";

const detect = await pipeline(
  "zero-shot-object-detection",
  "Xenova/owlv2-base-patch16-ensemble",
); // downloads + caches ~400 MB once

const queries = ["a cat", "a remote control", "a person wearing red"];
const output = await detect(imageURL, queries, { threshold: 0.1, percentage: false });
// → [{ label: "a cat", score: 0.29, box: { xmin, ymin, xmax, ymax } }, …]
//   label is the matched query; percentage:false gives pixel coordinates

This page runs the model once at a low floor threshold inside a Web Worker, caches every box, and filters the cached list as you drag the slider — so the slider is instant and never re-runs inference. Changing the query text re-runs the model (new text embeddings). The canvas is drawn at the image's natural resolution and scaled by CSS, so boxes stay pixel-accurate.
