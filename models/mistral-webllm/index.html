Mistral 7B Instruct v0.3 — the catalogue's most capable model on your GPU

The strongest model in this showcase, running entirely in your browser. Mistral 7B Instruct v0.3 (7.2B parameters) is compiled to WebGPU by MLC's WebLLM engine — you get streaming tokens, sampling controls, a system prompt and full conversation history, with nothing uploaded and no API bill. Seven billion parameters is a lot to move: it's a ~4 GB download and needs a capable GPU, with no WASM fallback. In return you get noticeably stronger reasoning and instruction-following than the smaller chat models here.

How it differs from the other chat pages: it's the largest and most capable model in the catalogue and a Mistral architecture (Apache-2.0), distinct from the Llama-family (Llama 3.2 1B), Gemma-family (Gemma 2 2B) and StableLM (StableLM 2 Zephyr 1.6B) models. The trade-off is weight: at ~4 GB it asks for a real GPU and a serious download, where the smaller models reach more devices.

Why it matters

A 7B model was, until recently, firmly server-side: an API key, a GPU instance, and a per-token bill. Mistral 7B Instruct on WebLLM puts a genuinely capable reasoning model on the client — private by construction, free to run, and available offline once the weights are cached. The cost is honest: a ~4 GB download and a real GPU. When your users have that, you get server-grade quality with no server.

  • Serious private assistants — multi-step reasoning and long instructions handled locally; ideal where data can't leave the device.
  • Reasoning & tool-style prompting — the extra capability shows up on planning, structured output, and "think then act" prompts the small models fumble.
  • Zero-cost premium features — ship your best "summarise", "draft", or "analyse" features with no inference bill and no rate limits, for users whose hardware can run it.
  • On-device RAG — pair it with a local embedding model for grounded answers over a user's own documents, all in the tab (see the multi-model demo).

How the API works

WebLLM gives you an OpenAI-shaped chat API backed by a WebGPU engine. You create the engine (which downloads and compiles the weights), then stream a completion. This page uses the shared helpers in lib/webllm.js, running inside a Web Worker so the UI stays responsive. Because Mistral's [INST] template has no system role, we fold the system prompt into the first instruction: 

import { webGPUAdapterAvailable, createEngine, streamChat }
  from "/web-ai-showcase/lib/webllm.js";

// WebLLM is WebGPU-only — gate before doing anything (headless requestAdapter() → null).
if (!(await webGPUAdapterAvailable())) {
  showNeedsWebGPU();          // never a faked reply
} else {
  const engine = await createEngine({
    model: "Mistral-7B-Instruct-v0.3-q4f16_1-MLC",
    onProgress: (p) => setProgress(p.text, p.progress),   // ~4 GB, cached after first load
  });

  // Mistral Instruct has no system role — fold it into the first user message.
  const messages = [
    { role: "user", content: "You are a concise assistant.\n\nExplain WebGPU in one sentence." },
  ];

  const full = await streamChat(
    engine,
    { messages, temperature: 0.7, top_p: 0.95, max_tokens: 256 },
    (delta) => appendToken(delta),          // streams token-by-token on your GPU
  );
  // engine.interruptGenerate() stops decoding early; engine.runtimeStatsText() reports tok/s.
}

Everything here runs exactly that path in worker.js, and gates on a real WebGPU adapter first. No adapter → a labelled needs-WebGPU state, never a canned answer.
