Metric3D — depth in metres from one photo

Most monocular depth models give you a relative map — a nearer/farther ordering in arbitrary units. Metric3D is different: it predicts absolute depth in metres against a canonical camera, so the numbers are real distances you can compare across the scene. Give it one photo and read how far every pixel is. It's small enough to run on WebGPU or plain WASM, and the photo never leaves the tab.

Depth distribution (near → far)

Camera intrinsics. Metric3D calibrates to a canonical focal length, so absolute metres are exact for that reference and an estimate for a photo with different intrinsics. Ratios and relative distances across the scene are the robust part — which is exactly what the "measure distances" demo relies on.

Metric vs relative depth

Depth Anything and DPT give a relative map — great for parallax and background blur, but the range is arbitrary and re-normalized per image, so "3" in one photo isn't "3" in another. Metric3D commits to metres: the same scene reads the same absolute distances, which is what lets you measure. (Apple's DepthPro is also metric but a much larger, WebGPU-only model tuned for razor-sharp boundaries — Metric3D is the small, WASM-friendly metric option.)

Why it matters

Absolute depth in metres from a single frame — no stereo rig, no LiDAR — turns a photo into measurable geometry:

  • Real distances — read how far a point is, and compare two points' distances in metres, from one image.
  • Consistent scale — the same scene gives the same numbers, so measurements are comparable across photos.
  • Robotics & AR scale — metric depth gives virtual objects the right real-world size and placement.
  • Scene reconstruction — a metric depth field per pixel is the starting point for a 3D point cloud.

Use cases

Basics

One photo in, a colourised metric depth map out — read distance as colour, with a metre legend.

Practical

Tap any point to read its distance in metres, and tap two to get the depth gap.

Wild

3D parallax — tilt a flat photo with your pointer, driven by real metric depth.

Multi-model

Metric depth + segmentation → the mean distance in metres of each labelled region.

How the API works

Depth is a one-liner with Transformers.js. The pipeline returns both a ready-to-display image and the raw metric tensor:

import { pipeline } from "@huggingface/transformers";

const estimator = await pipeline(
  "depth-estimation",
  "onnx-community/metric3d-vit-small",
  { device: "webgpu", dtype: "fp16" },   // or device:"wasm", dtype:"fp32"
);

const { depth, predicted_depth } = await estimator(imageURL);
// depth          → RawImage: grayscale, min–max normalized to 0–255 (for display)
// predicted_depth → Tensor:  the raw per-pixel depth in METRES (dims = [H, W])

This page colourises depth and reads predicted_depth for the real metre range + distribution, in "See inside". Both the inference AND the per-pixel colour composite run in a Web Worker (the worker transfers a ready ImageBitmap), so estimating a large photo never freezes the page. Metric3D runs on WebGPU when available and falls back to WASM otherwise.
