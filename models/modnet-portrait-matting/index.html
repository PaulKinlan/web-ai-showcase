MODNet — matting that keeps the hair

Removing a background is easy until you reach the hair. MODNet is a matting network: instead of a hard in/out decision it predicts a soft alpha value for every pixel, so flyaway strands, fur and soft edges survive the cut. Give it a portrait and it returns a matte you can drop onto transparency, a colour, or a new scene — entirely on your device.

See inside

MODNet never sees "a person" — it outputs an alpha matte: a number from 0 (background) to 255 (subject) for every pixel. What makes a matting model different from a plain background remover is the middle of that range: the grey, half-transparent band along hair and fur where it hedges. Those in-between values are exactly what a hard segmentation throws away — and what makes a composite look real. Why it matters

There are two jobs hiding inside "remove the background". RMBG-1.4 answers where is the subject with a confident, general-purpose mask. MODNet answers the harder question — exactly how opaque is each pixel — which is what you need whenever the edge is soft:

  • Believable portraits — profile pictures and headshots where cut hair looks fake and matted hair looks real.
  • Virtual backgrounds — the matte is the first frame of a green-screen-free video call, on-device and private.
  • Compositing & try-on — soft alpha blends the subject into a new scene instead of pasting a hard sticker.
  • Pick the right tool — run both on the same photo and choose per image: hard mask for logos/products, matte for people and fur.

How the API works

MODNet uses a custom architecture, so the model card's path loads the model and processor directly and reads the matte tensor:

import { AutoModel, AutoProcessor, RawImage } from "@huggingface/transformers";

const model = await AutoModel.from_pretrained("Xenova/modnet", {
  device: "webgpu", dtype: "fp32", config: { model_type: "custom" },
});
const processor = await AutoProcessor.from_pretrained("Xenova/modnet");

const image = await RawImage.read(imageURL);
const { pixel_values } = await processor(image);
const { output } = await model({ input: pixel_values });   // the ONNX input is named "input"

// output is a [1,1,H,W] soft alpha map in 0–1. Scale to 0–255 and resize to the original photo:
const matte = (await RawImage.fromTensor(output[0].mul(255).to("uint8")))
  .resize(image.width, image.height);
// then copy `matte` into the image's alpha channel → the cut-out

This page runs that in a Web Worker and stamps the matte into the photo's alpha on a canvas, so the interface stays responsive and the composite (transparency, colour, or a new scene) is all client-side. The input tensor's name (input) is read off the live ONNX session — no invented signature.
