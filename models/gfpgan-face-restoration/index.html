GFPGAN — restore an old or degraded face

Give this model an old, low-resolution or degraded portrait and it reconstructs a clean, sharp face: defined eyes, smooth skin, recovered hair and expression. GFPGAN (CVPR 2021) is a blind face-restoration network built on a StyleGAN2 face prior — it learned what real faces look like and rebuilds the detail degradation destroyed. It works on a single, roughly front-facing, aligned face. Runs entirely on your device via ONNX Runtime Web; the photo never leaves the tab.

How to read it

Bright indigo = big change (eyes, teeth, skin texture the model reconstructed); dark = untouched. The delta concentrates on facial features because the GAN prior only knows faces — feed it something that isn't a single aligned face and it will try (wrongly) to make one, which is why the input matters.

Why it matters

Old and low-quality portraits are everywhere — family archives, scanned documents, low-res avatars, video-call stills. On-device face restoration means you can rescue them privately, with no upload and no per-image cloud cost:

  • Photo & memory apps — a client-side "restore this old photo" button that runs on the user's device, no server.
  • Archives & genealogy — sharpen faces in scanned family photos so features are legible again, entirely offline.
  • Avatars & profiles — clean up a small or compressed profile picture without shipping it to a backend.
  • Pipelines — restore a face before colourising it or enlarging the whole photo (see the multi-model demo).

GFPGAN synthesises a plausible face from a learned prior; it can invent detail that wasn't there and can subtly change identity, so it's an enhancement, not forensic evidence. It expects a single, roughly aligned, front-facing face — group shots or profiles need a face detector/aligner first.

Use cases

Basics

Restore a genuinely old portrait as-is — wipe between the degraded original and the reconstructed face.

Practical

Rescue a small, compressed avatar or scan so the face is clean again, then download the result.

Wild

Push it: crank the degradation up and find where honest restoration turns into invented identity.

Multi-model

Restore → upscale: reconstruct the face with GFPGAN, then enlarge the whole photo 4× with Real-ESRGAN.

How the API works

transformers.js has no GFPGAN class and no face-restoration task, so we run the ONNX graph directly with ONNX Runtime Web. In goes an aligned face, out comes the restored one:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(gfpganBytes, { executionProviders: ["wasm"] });

// aligned face → [1,3,512,512] float32 normalised to [-1,1]:  (x/255 - 0.5) / 0.5
const input = new ort.Tensor("float32", toNCHW(face, 512, 512), [1, 3, 512, 512]);
const { [session.outputNames[0]]: out } = await session.run({ [session.inputNames[0]]: input });

// out is [1,3,512,512] in [-1,1] → map back to RGB:  (v + 1) / 2
const restored = tensorToImage(out.data, 512, 512);

Inference and the whole output composite run in a Web Worker so the UI never blocks, and the 340 MB model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
