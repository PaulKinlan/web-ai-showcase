Arctic-embed-m-v2.0 — one multilingual meaning space, with Matryoshka vectors

Snowflake's second-generation retrieval embedder. Where the built arctic-embed-s is a small English BERT (384-d), arctic-embed-m-v2.0 is built on the GTE-multilingual backbone: one 768-dimensional space across roughly 50 languages, up to 8192 tokens of context, and Matryoshka training so you can truncate the vector (e.g. to 256 dims) and it still works. Paste passages in any mix of languages, build a cross-language cosine matrix, and search long documents by meaning. It's Apache-2.0. Nothing leaves your device.

Why it matters

Arctic-embed-m-v2.0 is a modern, permissively-licensed multilingual retriever that fits in a browser. You embed questions, documents, tickets and reviews — long or short, in ~50 languages — into one 768-d space, and nearest-neighbour search works across the whole corpus. Matryoshka means you can store the full 768-d vectors for accuracy or truncated 256-d vectors to cut your index size and memory by two thirds, from the same model. It runs here with no server and no data leaving the tab.

  • Multilingual RAG — one retriever for a corpus written in dozens of languages; a query in any language finds the right passage in any other.
  • Cheaper vector indexes — Matryoshka lets you drop to 256 dims for a 3× smaller index with a small accuracy cost, decided at query time, no re-embedding.
  • Long-document search — embed a whole paragraph or section (up to 8192 tokens) instead of shredding it into sentences that lose context.
  • Ship it commercially — Apache-2.0, so it can go straight into a product.

Use cases

Basics

Do two passages mean the same thing across languages, or across a paraphrase? One cosine number.

Practical

Long-document, cross-language semantic search over a mixed-language knowledge base.

Wild

Matryoshka in action: shrink the vectors dim by dim and watch how far you can go before search breaks.

Multi-model

Retrieve cross-lingually with arctic-embed, then translate the winning passage to English with M2M100.

Contrast with the built arctic-embed-s (v1, English, 384-d), the BGE-M3 (1024-d, 100+ langs, larger) and the lighter multilingual MiniLM (384-d): this is the Apache-2.0, GTE-multilingual, Matryoshka option in the middle.

How the API works

Same feature-extraction recipe as any sentence embedder — CLS pooling, and a "query: " prefix on retrieval queries:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Snowflake/snowflake-arctic-embed-m-v2.0",
); // downloads + caches ~310 MB once

// documents: no prefix. query: prefix with "query: ". CLS pooling, L2-normalized:
const docs = await extract(["このバッテリーは一日中もちます", "The refund policy allows returns."], { pooling: "cls", normalize: true });
const q = await extract(["query: How long does the battery last?"], { pooling: "cls", normalize: true });

// unit-length vectors → cosine similarity is just the dot product:
const [qv] = q.tolist(), [d0] = docs.tolist();
const sim = qv.reduce((s, x, i) => s + x * d0[i], 0); // high — same meaning, different languages

// Matryoshka: truncate + re-normalize to shrink the vector (e.g. first 256 dims) — still usable.

This page asks the worker for the un-normalized CLS vectors so "See inside" can show the real pre-normalization magnitude (and demonstrate Matryoshka truncation), then normalizes them in JS. Inference runs in a Web Worker so the matrix, projection and search never jank the page.
