HandLandmarker — 21 joints on every hand

MediaPipe's HandLandmarker finds hands in an image or video and returns 21 landmarks for each — every knuckle and fingertip — as normalized x/y/z coordinates, plus whether it's a left or right hand. It's a real downloadable .task model that runs on-device via WebAssembly (with a GPU delegate when available). Drop a photo or turn on your webcam.

Why it matters

Hand tracking used to need a depth camera or a native SDK. HandLandmarker turns "where are the fingers?" into a local function call in any browser — private, offline, and fast enough for a live camera on a phone.

  • Touchless UI — control a page with gestures: pinch to zoom, swipe, or point, no controller.
  • Sign-language & learning tools — build fingerspelling practice or gesture games that work on-device.
  • Creative & AR — draw in the air, place virtual rings/props, or drive a puppet with your hand.
  • Accessibility — map custom hand poses to actions for people who can't use a mouse or keyboard.

How the API works

MediaPipe Tasks loads a .task bundle and gives you a landmarker you call per image or per video frame:

import { FilesetResolver, HandLandmarker } from "@mediapipe/tasks-vision";

const vision = await FilesetResolver.forVisionTasks(".../wasm");
const hand = await HandLandmarker.createFromOptions(vision, {
  baseOptions: {
    modelAssetPath: ".../hand_landmarker.task",  // ~8 MB
    delegate: "GPU",                              // or "CPU"
  },
  numHands: 2,
  runningMode: "IMAGE",                           // or "VIDEO"
});

const result = hand.detect(imageElement);
// result.landmarks  → [ [ {x,y,z} × 21 ], … ]  (normalized 0–1)
// result.handedness → [ [ { categoryName:"Right", score } ], … ]

This page loads the model through the showcase's shared auto-init loader and draws the landmarks on a canvas at the image's natural resolution; for the webcam it switches the task to VIDEO mode and calls detectForVideo each frame.
