YOLO-World — open-vocabulary detection, in real time

A normal detector only knows the classes it was trained on — YOLOv10 boxes 80 fixed COCO categories and nothing else. YOLO-World detects anything you can name. Type your own class list — forklift, traffic cone, swan — and a CLIP text encoder turns those words into vectors that steer a fast YOLOv8 head. No fixed vocabulary, no retraining, all on your device. It is far quicker than the phrase-grounding Grounding DINO, so it is the open-vocab detector you can actually run on a live camera.

See inside — how words become classes

This is what "open-vocabulary" actually means. Your class words go through a CLIP text encoder into 512-dim vectors; the detector treats those vectors as the class list, so its output tensor grows a channel per word you typed — [1, 4+N, 8400] for N classes. Every surviving box below is a real anchor from that tensor: the class it matched (the argmax over your N words), its probability, and the box in the image's own pixels. Change a word and the whole class space shifts — no retraining.

Why it matters

Fixed-class detectors force a choice you make once, at training time. Open-vocab detection moves that choice to runtime: the same tiny model detects "shopping cart" today and "fire extinguisher" tomorrow, just by changing a string. And unlike Grounding DINO — which grounds a whole caption and takes seconds per image — YOLO-World runs a single fast YOLO head, so it is practical on a live camera on-device.

  • Detect what you care about, not what a dataset chose — inventory, safety-gear, or niche-object detection with a class list you edit in a text box.
  • No labelling, no retraining — ship a detector for a new object by typing its name; nothing uploaded, no vision API bill.
  • Fast enough for video — one YOLO forward pass per frame after the classes are encoded once, so it keeps up where phrase-grounding can't.
  • Prompt-tunable precision — narrow "dog" to "golden retriever" or broaden to "animal" and watch the boxes follow the words.

Use cases

Basics

Type a class list, detect, and feel open-vocab first-hand — swap a word, watch the boxes change.

Wild

Rare-class hunt — detect objects no COCO model knows, and A/B two phrasings of the same thing.

Multi-model

YOLO-World finds it → crop → a ViT classifier names it more finely.

How the API works

Two on-device models compose. CLIP text-embeds your class names; the YOLO graph takes those embeddings as a dynamic input and returns per-class boxes:

import { AutoTokenizer, CLIPTextModelWithProjection } from "@huggingface/transformers";
import * as ort from "onnxruntime-web";

// 1) class names → L2-normalized CLIP ViT-B/32 text embeddings  [N, 512]
const tok  = await AutoTokenizer.from_pretrained("Xenova/clip-vit-base-patch32");
const clip = await CLIPTextModelWithProjection.from_pretrained("Xenova/clip-vit-base-patch32", { dtype: "fp16" });
const { text_embeds } = await clip(tok(["forklift", "pallet", "hard hat"], { padding: true }));

// 2) image + txt_feats → per-class boxes (48.8 MB YOLOv8-World-v2 ONNX)
const out = (await session.run({ images, txt_feats })).output0; // [1, 4+N, 8400]
// channels 0-3 = box cx,cy,w,h; channels 4..4+N = class probabilities (sigmoid baked in) → NMS

Both models run inside a Web Worker (CLIP text encode + letterbox + session.run + decode + NMS), off the main thread. The class embeddings are cached, so dragging the score slider or re-detecting the same classes never re-encodes text; the detector weights are fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
