D-FINE — real-time detection that refines a distribution

D-FINE is a state-of-the-art real-time detector. It keeps RT-DETR's fast, NMS-free, one-pass design — but changes how a box is located: instead of regressing four raw coordinates, it predicts a probability distribution for each box edge and sharpens it layer by layer, then distils that refined localisation back into the earlier layers. The payoff is higher COCO accuracy than RT-DETR at the same speed — small enough (~10 MB) to run your webcam. All on your device.

How D-FINE locates a box: FDR + GO-LSD

A normal detector head regresses four numbers per box — x, y, w, h — as point estimates. D-FINE instead predicts, for each of the four edges, a probability distribution over a set of discretized offsets, and the decoder layers iteratively refine those distributions (Fine-grained Distribution Refinement). A sharp, single-peaked distribution means a confident edge; a spread-out one means the model is unsure where the boundary is. Then GO-LSD (Global Optimal Localization Self-Distillation) distils the final, best-refined distributions back into the earlier layers during training — so the network localises better with zero extra cost at inference. That is the accuracy edge over RT-DETR, at the same real-time latency.

Contrast — D-FINE vs RT-DETR (same family, sharper localisation)

D-FINE is built on the RT-DETR decoder, so this is the fairest head-to-head: same NMS-free, real-time design, but D-FINE's distribution heads push COCO accuracy up. Published COCO AP: D-FINE-S ≈ 48.5 vs RT-DETR-R18 ≈ 46.5 (from the D-FINE paper). Latency below is measured live on your device — loading RT-DETR downloads ~22 MB the first time.

Why it matters

D-FINE is the current sweet spot for detection you can run every frame: RT-DETR's speed with a measurable accuracy bump and no NMS to tune. It turns "what's in this video, right now?" into a local function call — private, offline, free per call — at a ~10 MB download.

  • More accurate real-time counting — count people, cars, or shelf stock from a live feed, with tighter boxes than the older real-time detectors and nothing uploaded.
  • Cleaner outputs, less glue — NMS-free means no suppression thresholds; one box per object out of the box.
  • Uncertainty is legible — the distribution formulation means the box itself carries a notion of how sure the model is about each edge.
  • Edge & offline — a tiny SOTA detector that ships with your web app and runs on modest hardware.

How the API works

Detection is a one-liner with Transformers.js — D-FINE uses the same object-detection pipeline as DETR and RT-DETR:

import { pipeline } from "@huggingface/transformers";

const detect = await pipeline(
  "object-detection",
  "onnx-community/dfine_s_coco-ONNX",
  { dtype: "q8" },        // ~10 MB, cached after first load
);

const output = await detect(imageURL, { threshold: 0.5, percentage: false });
// → [{ label: "cat", score: 0.98, box: { xmin, ymin, xmax, ymax } }, …]
//   NMS-free: the boxes are already de-duplicated — no suppression pass needed.
//   percentage:false gives pixel coordinates; true gives 0–1 fractions

This page runs the model once at a low floor threshold (0.05) inside a Web Worker, caches every box, and filters the cached list as you drag the slider — so the slider is instant and never re-runs inference. The webcam loop grabs a frame, detects it, and only schedules the next after the last finishes, so it never piles up work or blocks the page.
