XLM-RoBERTa NER — name the entities, even in languages it never trained on

Named-entity recognition labels each span: who (PersonPER), what organization (OrgORG), where (LocationLOC), and when (DateDATE). This model runs on an XLM-RoBERTa backbone — pretrained on 2.5 TB of web text in 100 languages with a SentencePiece vocabulary. It's fine-tuned for NER on ten high-resource languages, but XLM-R's cross-lingual alignment is strong enough that it also tags entities in languages it never saw labelled — Turkish, Swedish, Japanese and more. Try the dashed "transfer" samples below. All on your device.

See inside — SentencePiece tokens & BIO tags

Where multilingual BERT splits into WordPiece (##-marked) fragments, XLM-RoBERTa uses SentencePiece: it segments text into sub-word pieces with no continuation marker (a rare name like "Schäuble" becomes Schä · u · ble). Each piece is tagged with a BIO label — B-PER (begin person), I-PER (inside person) — and we stitch the B-/I- runs into whole entities. Why it matters

A single NER model that structures text in ten languages — and degrades gracefully into dozens more it was never trained on — means you can index, redact, or link entities across a genuinely global corpus without a per-language pipeline and without a translation step. XLM-RoBERTa is the go-to backbone for cross-lingual transfer, and here it runs entirely in the browser, so not a byte of the source text leaves the device.

  • Long-tail language coverage — reach markets with no dedicated NER model by leaning on XLM-R's transfer, client-side.
  • Cross-border PII redaction — mask people, places and dates in documents in any supported language before they're shared or logged.
  • Multilingual data extraction — pull the who-where-when out of emails, notes or transcripts regardless of language.
  • One tagging pipeline, many locales — retire a rack of single-language NER services for one transferable model.

How the API works

Token classification is one pipeline call:

import { pipeline } from "@huggingface/transformers";

const ner = await pipeline(
  "token-classification",
  "tjruesch/xlm-roberta-base-ner-hrl-onnx", // ONNX export of Davlan/xlm-roberta-base-ner-hrl
);
const out = await ner("Stefan Löfven träffade Angela Merkel i Stockholm."); // Swedish — untrained
// → [{ entity: "B-PER", word: "Stefan",  score: 1.0, index: 1 },
//    { entity: "I-PER", word: "Lö",      score: 1.0, index: 2 },   // SentencePiece pieces
//    { entity: "I-PER", word: "f",        score: 1.0, index: 3 },
//    { entity: "I-PER", word: "ven",      score: 1.0, index: 4 },
//    { entity: "B-PER", word: "Angela",   score: 1.0, index: 6 },
//    { entity: "I-PER", word: "Merkel",   score: 1.0, index: 7 },
//    { entity: "B-LOC", word: "Stockholm",score: 0.99,index: 9 }]

The pipeline returns one BIO-tagged row per SentencePiece token, but no character offsets. We map each piece back to its span in the original text (a cursor-walk) so highlights land exactly — across Latin, CJK and RTL scripts — then merge B-/I- runs into whole entities with a pooled confidence. It all runs in a Web Worker so typing stays smooth.
