Why it matters

A capable instruct model that fits in a browser tab turns language into a client-side primitive: private, offline-capable, and free to run at any scale. No inference bill, no data leaving the device, no cold-start latency once it's cached.

  • Private chat & drafting — an assistant that never sends a keystroke to a server.
  • Structured extraction — turn messy text into clean JSON for a form or API, on-device.
  • On-device agents — classify, route, and summarise inside the page with zero per-call cost.
  • Offline & edge — kiosks, field apps, and privacy-first tools that work with no network.

How the API works

Qwen uses the standard text-generation pipeline. Pass a messages array (the pipeline applies the chat template for you), a TextStreamer for token-by-token output, and your sampling settings: 

import { pipeline, TextStreamer } from "@huggingface/transformers";

const generator = await pipeline(
  "text-generation",
  "onnx-community/Qwen2.5-0.5B-Instruct",
  { dtype: "q4f16", device: "webgpu" },   // WASM + q4 is the fallback
);

const messages = [
  { role: "system", content: "You are a helpful, concise assistant." },
  { role: "user", content: "Explain WebGPU in one sentence." },
];

const streamer = new TextStreamer(generator.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });

await generator(messages, {
  max_new_tokens: 256, do_sample: true, temperature: 0.7, top_p: 0.9, streamer,
});

The "see inside" panel runs one raw forward pass — generator.model(generator.tokenizer(prompt)) — reads the last position's logits, softmaxes them, and shows the real top-k. Everything runs in a Web Worker and gates on a real WebGPU adapter first; with none, it offers a labelled WASM path instead of faking tokens.
