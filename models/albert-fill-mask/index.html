ALBERT — the tiny BERT that reuses one layer

ALBERT does the same job as BERT — hide a word behind [MASK] and it ranks what belongs there — but it carries only ~11.8M parameters where BERT-base carries ~110M, at the same 12-layer depth. Its trick: it reuses one transformer block over and over and factorises its embedding table. Type a sentence, watch the guesses and the raw logits, then open "See inside" to watch the parameter sharing that shrinks it. All on your device.

See inside — the shared layer that makes ALBERT tiny

BERT and ALBERT have the same shape — 12 transformer layers, 768 hidden units. The difference is what's stored. BERT keeps 12 independent blocks; ALBERT keeps one and runs your text through it 12 times. Why it matters

ALBERT is the answer to a very practical question: how small can a real BERT get without changing its shape? By sharing one block across all layers and factorising the embedding, it fits language understanding into a fraction of the memory — which is exactly what you want when the model has to live inside a browser tab, a phone, or a cache budget.

  • Small-footprint understanding — masked-LM smarts at ~40 MB, cacheable and offline, for autocomplete and suggestion boxes that never phone home.
  • Plausibility scoring — because it scores every word in context, you can ask "does this word fit here?" and compare alternatives on device.
  • Knowledge & bias probing — templated [MASK] prompts surface what a compact model did (and didn't) learn, and the associations baked into its training.
  • A teaching model — parameter sharing and factorised embeddings are two of the clearest "make a transformer smaller" ideas; ALBERT makes them visible and runnable.

How the API works

The one-liner uses the fill-mask pipeline — identical call, tiny model:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "Xenova/albert-base-v2");
const out = await unmask("The capital of France is [MASK].");
// → [{ token_str: "paris", score: 0.05, sequence: "…" }, { token_str: "reims", … }]

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 30000]), slice the row at the [MASK] position, and softmax it ourselves. That's the only way to show the real logit values, not just the pipeline's rounded scores. All of it runs in a Web Worker so the page never janks — and because ALBERT reuses one block, the on-disk weights are small even though the compute still runs the full 12-layer depth.
