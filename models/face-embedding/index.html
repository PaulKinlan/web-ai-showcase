FaceNet — how alike are two faces?

FaceNet turns a face crop into a 128-number embedding — a point in "face space" where the same person's photos land close together and different people land far apart. Compare two faces and you get a similarity signal: likely the same person, or likely not. It runs entirely on your device — the photos and the embeddings never leave the tab. This is face similarity, not identification, lookup, or surveillance.

Why it matters

A face embedding is the building block behind "is this the same person?" — and doing it on-device means the biometric never touches a server. Once a face is a 128-number point you can compare, cluster, and threshold, all client-side:

  • Private photo organising — group a local gallery by who's in each shot without uploading a single face.
  • Consent-first re-auth signal — a "does this look like the enrolled user?" hint that stays a signal, with the real decision made by a proper, liveness-checked system.
  • De-duplication — spot the same face across a set of images (avatars, ID photos) locally.
  • Look-alike & similarity search — rank a small gallery by how close each face is to a reference.

These are similarity tools. Real identity, verification, or surveillance needs consent, liveness detection, bias auditing, and calibration this demo deliberately does not provide.

Use cases

Basics

Two faces in, one similarity number out — see same vs different in "face space".

Practical

On-device "unlock" concept — enrol a reference face, then check a probe against a threshold (a signal, never a verdict).

Wild

Look-alike finder — rank a whole gallery of faces by how close each is to your reference.

Multi-model

Detect → crop → embed: find the face in a full photo first, then compare only the aligned crop.

How the API works

transformers.js has no FaceNet class, so we run the ONNX graph directly with ONNX Runtime Web. The two things a pipeline would normally hide — prewhiten preprocessing and L2-normalizing the output — are a few lines:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(facenetBytes, { executionProviders: ["wasm"] });

// face → [1,160,160,3] NHWC, prewhitened: (px - mean) / max(std, 1/sqrt(N))
const input = new ort.Tensor("float32", prewhiten(faceCrop), [1, 160, 160, 3]);
const { [session.outputNames[0]]: out } = await session.run({ [session.inputNames[0]]: input });

const emb = l2normalize(out.data);         // 128-D unit vector
const similarity = dot(embA, embB);        // cosine similarity ∈ [-1, 1]; higher = more alike

Both embeddings are computed in a Web Worker so the UI never blocks, and the model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
