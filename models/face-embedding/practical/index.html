An on-device "unlock" signal

A common shape: enrol one reference face, store just its 128-number embedding, then later verify a probe face against it. Move the threshold and see the accept/reject flip. This is a signal, not a security decision — real re-auth needs liveness detection and calibration this demo deliberately omits.
