Practical · text-generation · on-device

Multilingual free text → structured JSON

The most useful thing a small on-device LLM does for a product: read messy human text — in whatever language it arrives — and emit clean, schema-shaped JSON your form or API can consume, with nothing leaving the device. Paste a review, email, or note in English, French or Spanish and extract it. The page then parses the JSON to prove it's real, machine-readable output. Falcon3's multilingual training is the point here: the input language can vary, the JSON shape stays fixed.

Why this pattern works

A system prompt fixes the schema and demands "JSON only, no prose, keys in English"; greedy decoding keeps the output deterministic. The page strips any stray code fences and runs JSON.parse() — the honest test of whether the model produced machine-readable output, not just plausible text. A multilingual model lets you accept input in the user's language while keeping a single, stable output schema. Small models occasionally miss a field; validating in-page and re-prompting is the shippable pattern.
