Falcon3 1B Instruct — TII's compact multilingual LLM in your browser

Falcon3 is the Technology Innovation Institute's 2024 open family, and the 1B Instruct is its smallest chat model — deliberately built to be capable at a size that runs anywhere. This page runs it entirely in your browser via Transformers.js: it streams tokens on the universal WebAssembly/CPU path (no GPU required) and automatically takes the WebGPU fast path when your device has one. Chat with it, steer it with a system prompt and sampling controls, and watch the real probability distribution behind every token. Nothing is uploaded; there is no API key and no server.

A different family from the other small chat models here (Qwen2, Llama, Gemma, Mistral): Falcon3 is a TII Falcon architecture, notably strong across English, French, Spanish, Portuguese and more at its size.

Why it matters

Falcon3-1B is the "runs anywhere" tier of a serious open family. Because it is small and multilingual, it makes client-side language a universal primitive: it works on the locked-down work laptop, the older phone, the kiosk with the GPU blocklisted — and it can read and reply across several languages without a translation API. Private, offline-capable once cached, free to run, no key.

  • Multilingual on-device — draft, summarise, and answer across languages with nothing sent to a server.
  • Structured extraction — turn messy text into clean JSON for a form or API, entirely in the page.
  • Universal reach — no GPU adapter or driver flags required; if the browser runs WebAssembly, it runs.
  • On-device RAG — pair it with a local embedding model for grounded answers over a user's own documents (see the multi-model demo).

How the API works

It's the standard Transformers.js text-generation pipeline — pass a messages array (the pipeline applies Falcon3's chat template), a TextStreamer for token-by-token output, and your sampling settings. The only device-specific choice is device + dtype, which this page picks by probing for a real WebGPU adapter: 

import { pipeline, TextStreamer } from "@huggingface/transformers";

// Probe for a real adapter — WebGPU (q4f16) fast path, else universal WASM (q8) on the CPU.
const adapter = navigator.gpu && await navigator.gpu.requestAdapter().catch(() => null);
const [device, dtype] = adapter ? ["webgpu", "q4f16"] : ["wasm", "q8"];

const generator = await pipeline(
  "text-generation", "onnx-community/Falcon3-1B-Instruct", { device, dtype },
);

const messages = [
  { role: "system", content: "You are a helpful, concise assistant." },
  { role: "user", content: "Traduis 'hello' en français." },
];

const streamer = new TextStreamer(generator.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });

await generator(messages, {
  max_new_tokens: 200, do_sample: true, temperature: 0.7, top_p: 0.9, streamer,
});

The "see inside" panel runs one raw forward pass — generator.model(generator.tokenizer(prompt)) — reads the last position's logits, softmaxes them, and shows the real top-k. Everything runs in a Web Worker so generation never blocks the page; the loader auto-inits an already-cached copy and never silently re-downloads.
