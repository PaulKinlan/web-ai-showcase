mMARCO mMiniLMv2 — rank passages by relevance, across 14 languages

The multilingual sibling of the classic English MS MARCO MiniLM reranker. Where that one is an English BERT trained on the English MS MARCO passage-ranking data, mmarco-mMiniLMv2-L12-H384-v1 is a 12-layer XLM-RoBERTa MiniLMv2 (distilled from XLM-R Large) trained on mMARCO — MS MARCO machine-translated into 13 more languages. Instead of turning a query and a passage into separate vectors and comparing them, it reads both together and outputs one relevance score, so it ranks a Spanish, Japanese or Russian passage against an English query as easily as an English one — and it's Apache-2.0, so you can actually ship it. Give it a query and candidates and watch it re-sort them; then look inside at the raw logit per pair. Nothing leaves your device.

Run it

Type a query and one candidate passage per line. The model scores each passage against the query and re-sorts them by relevance logit (higher = more relevant). The default set mixes English, Spanish, Japanese and German so you can see cross-language ranking.

Why it matters

Rerankers are the precision stage of modern search. A fast retriever (a keyword index or embeddings) casts a wide net and returns, say, the top 50 candidates; a cross-encoder then reads each candidate against the query and reorders them with far higher accuracy. mMARCO mMiniLMv2 brings that precision to 14 languages from one 118M-parameter model, and — unlike the CC-BY-NC jina-reranker-v2 — under a permissive Apache-2.0 licence, so it can go into a commercial product. At ~470 MB it runs in a tab, so the precise, expensive step happens on-device with nothing sent anywhere.

  • Global multilingual RAG — retrieve across a mixed-language corpus, then rerank so the LLM reads the genuinely best chunk whatever language it's in.
  • Cross-language site / support search — one reranker serves every locale instead of a model per language, and a query in one language surfaces answers written in another.
  • Ship it commercially — Apache-2.0 means no per-deployment licence, the practical reason to reach for this over jina-reranker-v2.
  • Distilled + small — a 12-layer MiniLMv2 (distilled from XLM-R Large) is a fraction of a full XLM-R reranker while keeping most of the ranking quality.

Use cases

Basics

Rank a handful of passages against a query by cross-encoder relevance.

Practical

Cross-lingual FAQ / support search: an English question over a multilingual answer bank.

Wild

Multi-model

Multilingual E5 embeddings retrieve, mMARCO mMiniLMv2 reorders — the classic retrieve-then-rerank.

Contrast with the built rerankers: the English MS MARCO MiniLM (same recipe, English only), the jina-reranker-v2 (multilingual + code, CC-BY-NC) and BGE-reranker-v2-m3 (multilingual, larger) — this one is the Apache-2.0, MS-MARCO-lineage multilingual option.

How the API works

A cross-encoder scores query–passage pairs; we read the raw logit off the model directly:

import { AutoTokenizer, AutoModelForSequenceClassification } from "@huggingface/transformers";

const model_id = "cross-encoder/mmarco-mMiniLMv2-L12-H384-v1";
const tokenizer = await AutoTokenizer.from_pretrained(model_id);
// config.json already carries model_type:"xlm-roberta", so a plain load routes correctly:
const model = await AutoModelForSequenceClassification.from_pretrained(model_id, { dtype: "fp32" });

// Encode the query with each passage AS A PAIR — [CLS] query [SEP] passage [SEP]:
const inputs = tokenizer(passages.map(() => query), { text_pair: passages, padding: true, truncation: true });
const { logits } = await model(inputs); // one relevance logit per pair — higher = more relevant
// sort the passages by logits.data — that's your reranking (works even when query and passage are in
// different languages, because the model was trained on mMARCO across 14 languages)

All inference runs in a Web Worker so ranking never janks the page. The pipeline helper would squash the logit through a sigmoid, so this page reads model() directly to keep the raw logit.
