CLAP — name a sound with any labels you invent

CLAP is CLIP for sound. It learned one shared space for audio and words, so you can hand it a clip and a handful of labels you make up on the spot — a dog barking, rain, applause, a guitar — and it scores how well each one matches what it hears. No fixed class list, no training, nothing leaves your device.

See inside

CLAP turns your clip into one 512-number vector and each label into its own 512-number vector, then measures the cosine similarity (the angle) between the audio vector and every label vector. Those similarities are scaled into logits, and a softmax turns them into the percentages above. Same sound, different words → different winner. The audio encoder itself reads a 64-band log-mel spectrogram — the real image of your clip is below (time left→right, frequency low→high, brightness is energy).

Why it matters

Every other audio classifier is stuck with the classes it was trained on (AudioSet's 527, say). CLAP isn't: the label set is an input, decided at runtime, in the browser, for free. That turns "collect a labelled dataset and fine-tune" into "write a list of words" — for sound.

  • Custom sound events — watch for your vocabulary (glass breaking, a baby crying, a smoke alarm) with no training pipeline and no audio leaving the device.
  • Content triage & moderation — score a clip against policy labels (gunshot, screaming, music, safe) client-side before anything is uploaded.
  • Auto-tagging & search by meaning — tag a recording library against a vocabulary you control, then search it by what things sound like.
  • Vibe / mood sorting — rank a sound against abstract qualities (calm, chaotic, ominous) that no fixed classifier has a class for.

How the API works

The whole thing is three lines with Transformers.js. The label list is just an argument:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "zero-shot-audio-classification",
  "Xenova/clap-htsat-unfused",
  { dtype: "q8" },              // downloads + caches ~160 MB once
);

const output = await classify(audio, ["a dog barking", "rain", "applause"]);
// audio = Float32Array @ 48 kHz mono (decode with the Web Audio API)
// → [{ label: "a dog barking", score: 0.94 }, { label: "rain", score: 0.05 }, … ]
//   score is the softmax over CLAP's audio↔label cosine similarities

This page goes one level deeper for "See inside": it reuses the pipeline's own model, processor, and tokenizer to read the raw logits_per_audio and the audio_embeds / text_embeds tensors from a single forward pass — the same call, just keeping the intermediate numbers. Inference runs in a Web Worker so the page never janks.
