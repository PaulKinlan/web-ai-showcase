CLAP — describe a sound in words, and the model finds it

CLAP is CLIP for your ears. It embeds audio and text into the same 512-dimensional space, so "which of these sentences describes this sound?" becomes a similarity lookup — with sentences you type, invented on the spot. No fixed class list, no retraining, and here: no server. Every score on this page is computed locally in your browser.

At a glance

Modellaion/clap-htsat-unfused (ONNX build: Xenova/clap-htsat-unfused)
Task / pipelinezero-shot-audio-classification
ArchitectureTwo towers, contrastively trained: HTSAT (a Swin-style transformer reading a 64-band log-mel spectrogram) for audio + RoBERTa for text, each projected to a shared 512-d space
Training dataLAION-Audio-630K — 633,526 audio–text pairs
Input48 kHz mono audio, up to 10 s per window (shorter clips are repeat-padded) · any text labels
OutputOne L2-normalised 512-d embedding per clip and per sentence; cosine × learned scale = logit; softmax over labels = score
Quantizationq8 (8-bit) ONNX, combined text+audio graph
BackendWebAssembly (runs anywhere; WebGPU used when a real adapter exists)
Download~163 MB once, then cached
LicenseApache-2.0
Web APIsWebAssembly, Web Workers, Web Audio, MediaRecorder, Canvas — all Baseline widely available

Why it matters

Classic audio classifiers ship a frozen menu — AudioSet's 527 classes, or whatever you had labelled data for. CLAP removes the menu: any sound concept you can phrase, you can detect, rank, or search for — and running it in the browser means the microphone never leaves the device and there is no per-call bill.

  • Detectors without datasets — "glass breaking", "a forklift reversing", "my machine's bearing squealing": one sentence replaces a labelled training set and a fine-tuning run.
  • Semantic audio search — embed a sound library once, then find clips by description ("upbeat drums", "rain on a window") — a text-to-audio search engine with no backend.
  • Personalisable accessibility alerts — users type the sound they need flagged, in their own words, private by construction.
  • Tagging & moderation at zero marginal cost — auto-tag podcasts, field recordings or user uploads with a vocabulary you can change at any moment.

Basics

Feel what zero-shot means: invent labels for a sound, then watch how the wording of a label changes its score.

How the API works

One pipeline call with Transformers.js — feed 48 kHz mono audio and your labels:

import { pipeline } from "@huggingface/transformers";

const classifier = await pipeline(
  "zero-shot-audio-classification",
  "Xenova/clap-htsat-unfused",
  { dtype: "q8" },                    // ~163 MB once, then cached
);

const scores = await classifier(audio, [
  "an emergency siren",
  "birds chirping",
  "heavy rain",
], { hypothesis_template: "This is a sound of {}." });
// audio = Float32Array @ 48 kHz mono (decode with the Web Audio API)
// → [{ label: "an emergency siren", score: 0.94 }, … ]

Under the hood each label becomes a sentence via the template, RoBERTa embeds the sentences, HTSAT embeds the audio, and the softmaxed scaled cosines are your scores. This page calls ClapModel directly — the exact same mechanism — so it can also hand you the embeddings, cosines and logits it visualizes, and reuse the embeddings for search. Everything runs in a Web Worker, so the page never janks.
