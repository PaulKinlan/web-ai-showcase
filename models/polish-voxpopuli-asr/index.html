wav2vec2 — Polish speech, one letter per frame

The English wav2vec2 demo speaks one language, the Russian XLSR demo another, and the Spanish and Italian demos two more. This one recognises Polish. It is wav2vec2-base pre-trained self-supervised on 10,000 hours of unlabelled European-Parliament audio (VoxPopuli) and then fine-tuned on Polish — a compact ~95M-param CTC recogniser whose output alphabet carries the Polish diacritics (ą ć ę ł ń ó ś ź ż). Same CTC mechanics as the other ASR demos: label every ~20 ms frame, then collapse the frames into text. Pick a clip and watch the frames collapse into Polish.

Why it matters

Self-supervised pre-training on huge piles of unlabelled speech — here 10,000 hours of European-Parliament audio — is how you get a good recogniser for a language like Polish from a modest amount of labelled data. The model learned what human speech sounds like before it ever saw a Polish transcript. That is what puts a usable Polish recogniser inside a browser tab — private, offline once cached, no server, no key. And because it's CTC (frames, not generated tokens) it can't hallucinate an unspoken sentence, and every character is pinned to a moment in time.

  • On-device Polish dictation and captions — for apps that must keep audio on the device (health, legal, journalism) in the Polish market.
  • Compact + mobile-friendly — a 4-bit ~90 MB CTC model runs on WASM with no GPU, so it's realistic on a phone.
  • Polish forced alignment — real per-word timings for karaoke, subtitles, and pronunciation feedback, straight from the frames.
  • Privacy-first field work — transcribe sensitive Polish audio with nothing leaving the device.

How the API works

This repo ships the standard transformers.js layout, so the model loads directly as AutoModelForCTC. We still normalise the audio and do the CTC collapse ourselves so we can show the per-frame strip (the ASR pipeline does exactly this internally): 

import { AutoModelForCTC, Tensor } from "@huggingface/transformers";

const model_id = "onnx-community/wav2vec2-base-10k-voxpopuli-ft-pl-ONNX";
const model = await AutoModelForCTC.from_pretrained(model_id, {
  device: "wasm", dtype: "q4",   // 4-bit export, ~90 MB — cleaner Polish than q8, tiny vs 378 MB fp32
});

// audio = 16 kHz mono Float32Array → normalise to zero mean / unit variance (wav2vec2 feature step)
const x = normalize(audio);
const { logits } = await model({ input_values: new Tensor("float32", x, [1, x.length]) });
// logits shape [1, frames, 41]  ← 41 Latin-Polish symbols

// greedy CTC decode: argmax per frame, merge duplicates, drop the blank (id 0)
const ids  = argmaxPerFrame(logits);    // e.g. [0,0,18,18,0,8,20,...]
const text = collapse(ids);             // "dom…" in Polish — no decoder involved

Inference runs in a Web Worker so the UI never janks. The worker returns the raw per-frame argmax and real per-word timings (from the frame each character was emitted at) — nothing is faked or pre-computed.
