zero-shot-classification · text · WASM · ~70 MB

DeBERTa-v3 zero-shot — label text with categories you invent

Normal classifiers only know the labels they were trained on. This one takes your labels, made up on the spot, and scores how well each fits the text — no training, no fixed categories. It works by asking a natural-language-inference model "does this text imply 'This example is {your label}'?" and reading off the entailment. Type text and some labels below, then look inside at the raw entailment logits doing the deciding. Nothing leaves your device.

Why it matters

Zero-shot classification means you never have to collect a labelled dataset or train a model to sort text — you just describe the buckets you want in plain words and change them whenever you like. That's the difference between a two-week ML project and a one-line config. And at 70 MB it runs in a tab, so you can triage, route, and tag text on-device with nothing leaving the browser.

  • Intent routing — send a support message to the right team from labels you define.
  • Ticket & content triage — sort feedback, tickets, or notes into categories that fit your business.
  • Flexible moderation — flag text against policy labels you can edit without retraining.
  • Ad-hoc tagging — apply multi-label topic or mood tags to a stream of messages.

How the API works

You pass the text and your labels; the pipeline builds a hypothesis per label and ranks them:

import { pipeline } from "@huggingface/transformers";

const classifier = await pipeline(
  "zero-shot-classification",
  "Xenova/nli-deberta-v3-xsmall",
); // downloads + caches ~70 MB once

const out = await classifier(
  "My package is three days late with no update.",
  ["shipping delay", "billing problem", "product feedback"],
  { multi_label: false }, // labels compete via softmax; true = independent scores
);
// → { labels: ["shipping delay", …], scores: [0.94, …] }  (sorted)

"See inside" reuses the same loaded model (classifier.model + classifier.tokenizer) to run each (text, hypothesis) pair itself and read the raw contradiction / neutral / entailment logits. All of it runs in a Web Worker so classifying never janks the page.
