Qwen2-VL — read a document, reason about a picture

Qwen2-VL is a heavier-hitting vision-language model with a reputation for reading: it transcribes text in photos, answers questions about screenshots, charts, forms and receipts, and reasons across what it sees. Give it an image and a prompt and the answer streams out token by token on your GPU — nothing uploaded.

A VLM doesn't "see" — it reads a sequence of tokens where your image has been turned into a block of vision tokens and slotted into a chat template alongside your words. Qwen2-VL wraps the turn in its ChatML format (<|im_start|> … <|im_end|>) with an <|vision_start|>…<|vision_end|> span standing in for the image patches. Why it matters

Qwen2-VL is the "reading" VLM: it's tuned to pull structure and text out of images, not just caption them. Running that in a browser tab — privately, offline-capable, free per call — turns a lot of server-side document plumbing into a client-side primitive.

  • On-device document Q&A — read a photographed invoice, receipt, or form and answer questions about it, without the image ever leaving the device.
  • Screenshot & chart understanding — let users drop a dashboard or chart screenshot and ask "what's the trend?" right inside your app.
  • OCR with reasoning — not just transcribe text, but interpret it: total up line items, extract a field, translate a sign.
  • Accessibility copilots — describe complex images and read dense text aloud for users who need it, with no per-image API cost.

How the API works

Qwen2-VL in Transformers.js uses a dedicated model class. You build a chat message whose content mixes an image placeholder and text, template it, run the processor over the templated text plus the image, then generate — streaming tokens as they come: 

import { AutoProcessor, Qwen2VLForConditionalGeneration, RawImage, TextStreamer }
  from "@huggingface/transformers";

const model_id = "onnx-community/Qwen2-VL-2B-Instruct";
const processor = await AutoProcessor.from_pretrained(model_id);
const model = await Qwen2VLForConditionalGeneration.from_pretrained(model_id, {
  dtype: { embed_tokens: "fp16", vision_encoder: "fp16", decoder_model_merged: "q4f16" },
  device: "webgpu",                        // WebGPU required
});

const image = await (await RawImage.read(imageURL)).resize(448, 448);
const conversation = [
  { role: "user", content: [{ type: "image" }, { type: "text", text: prompt }] },
];
const text = processor.apply_chat_template(conversation, { add_generation_prompt: true });
const inputs = await processor(text, image);

const streamer = new TextStreamer(processor.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });
await model.generate({ ...inputs, max_new_tokens: 200, do_sample: false, streamer });

This page runs exactly that, in a Web Worker, and gates on a real WebGPU adapter first — if none exists it shows a labelled needs-WebGPU state instead of a fake answer.
