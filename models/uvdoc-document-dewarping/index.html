UVDoc — flatten a warped document photo

Give this model a photo of a curled, skewed or perspective-distorted page — a book held open, a receipt on a table, a folded page shot at an angle — and it flattens it into a clean, rectangular, scanner-style document: straightened text lines, corrected geometry. UVDoc is the PaddleOCR document image unwarping model. It doesn't crop or rotate — it learns the page's 3D deformation and undoes it. Runs entirely on your device via ONNX Runtime Web; the document never leaves the tab.

UVDoc doesn't just crop or rotate — it predicts the page's continuous 3D deformation and resamples every pixel to undo it. That's why straight text lines that bowed in the photo come back straight, and a page shot at an angle becomes a front-on rectangle. Watch the edges of the page as you wipe: a plain crop would leave them curved; UVDoc straightens them.

Why it matters

A photo of a page is not a scan — it's curled, skewed and lit unevenly, which wrecks OCR and looks unprofessional. On-device dewarping turns a phone photo into a clean document, privately and free:

  • Mobile document scanning — a "flatten this page" step in a scanner app that runs client-side, no server round-trip.
  • OCR pre-processing — straighten text lines before recognition so the OCR model reads far more accurately.
  • Receipts, forms & contracts — turn a hurried angled photo into a filable, readable rectangular document.
  • Archiving books & notes — de-curl photographed book pages into flat scans in a static web tool with zero backend.

UVDoc corrects geometric distortion; it doesn't invent text the photo never captured, and extreme folds or occlusions flatten only partially.

Use cases

Basics

Curl a page, then watch UVDoc flatten it — wipe between warped and flattened and see the text lines straighten.

Practical

Flatten a real photographed page and download the corrected scan-style image, ready to file or print.

Wild

Crank the curl to an extreme fold and find where geometric rectification starts to break down.

Multi-model

Dewarp → read: flatten a page, then run the on-device text detector to see how many more lines OCR can find.

How the API works

transformers.js has no UVDoc class and no document-unwarping task, so we run the ONNX graph directly with ONNX Runtime Web. In goes the warped page, out comes the flattened one:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(uvdocBytes, { executionProviders: ["wasm"] });

// warped page → [1,3,H,W] float32 in [0,1] (plain RGB, no normalisation)
const input = new ort.Tensor("float32", toNCHW(warped, W, H), [1, 3, H, W]);
const { fetch_name_0 } = await session.run({ image: input });

// fetch_name_0 is [1,3,H,W] in [0,1] → the flattened page → clamp to RGBA
const flat = tensorToImage(fetch_name_0.data, W, H);

Inference and the whole output composite run in a Web Worker so the UI never blocks, and the ~30 MB model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
