Multi-model · dewarping → text detection · WASM

Dewarp, then detect the text

A curled page skews its own text lines. Here UVDoc flattens the page first, then the on-device DBNet text detector runs on both versions — so you can watch the detected boxes go from following the curl to sitting straight and axis-aligned on the flattened page. Two models, chained, entirely local.

Run it

Two models load below (UVDoc ~30 MB, detector ~3 MB). Pick a page, curl it, then run the pipeline — we detect on the warped image and on the flattened one so you can see the gain.

1 · UVDoc (document dewarping)
2 · DBNet (text detection)

Warped → detected

Flattened → detected

dewarp –detect –text regions –

Both models run on-device; the document, the flattened result and the boxes never leave the tab.

Why chain them

Dewarping is a force-multiplier for document AI: a text detector, OCR model or layout parser all read a flat page far better than a curled photo. This is the on-device version of the "scan then OCR" pipeline every document app runs — no server, no upload, each stage a real model in your browser.
