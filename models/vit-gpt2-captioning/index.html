ViT-GPT2 — write a sentence about any picture

A Vision Transformer reads the image; a GPT-2 decoder writes about it, one word at a time. The result is a plain-English caption — a cat laying on top of a bed — generated entirely on your device, no captioning API in sight.

See inside

Captioning is autoregressive: after the ViT encoder turns the image into patch embeddings once, the GPT-2 decoder produces the caption one token at a time, each token conditioned on the image and everything written so far. Why it matters

A caption is structured meaning you can act on — search it, read it aloud, translate it, gate on it. Doing it locally makes it free, private, and offline-capable, which changes where you can put it.

  • Accessibility at scale — auto-generate alt text for user-uploaded images so screen-reader users aren't left with "image".
  • Searchable media libraries — turn a folder of untitled photos into text you can full-text search, without uploading a single file.
  • Content pipelines — draft social captions, product blurbs, or thumbnail descriptions client-side, then let a human polish.
  • Assistive & offline capture — describe what a camera sees on-device, useful for low-connectivity or privacy-sensitive settings.

How the API works

The base call is two lines with Transformers.js:

import { pipeline } from "@huggingface/transformers";

const captioner = await pipeline(
  "image-to-text",
  "Xenova/vit-gpt2-image-captioning",
); // downloads + caches ~250 MB once

const output = await captioner(imageURL, { max_new_tokens: 30 });
// → [{ generated_text: "a cat laying on top of a bed" }]

For "See inside", this page passes a TextStreamer into the same call. The streamer's callback fires once per decoded token, so the page can render the caption as it forms and record each token's arrival time — all off the main thread in a Web Worker, so the page never janks.
