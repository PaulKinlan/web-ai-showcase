Emotion — the 7 Ekman feelings, one winner

Where the 28-class GoEmotions model lets many emotions fire at once (multi-label sigmoid), this compact DistilRoBERTa classifier is single-label: it runs a softmax over the seven Ekman emotions — anger, disgust, fear, joy, neutral, sadness, surprise — so the seven scores compete and sum to 1, and exactly one wins. Type below and watch the whole distribution light up the emotion wheel, on-device.

See inside — the emotion wheel

The wheel (a 7-axis radar) is the model's full softmax distribution in one shape: each spoke is one emotion, the polygon reaches out as far as that emotion's probability. Because it's a softmax — not a sigmoid — the seven values compete and sum to 1: push one up and the rest fall. That's the difference from multi-label GoEmotions, where every emotion has its own independent score and several can be high at once.

Why it matters

A single, decisive emotion label is often exactly what a product needs — one route, one tag, one reaction. A small softmax classifier gives you that decision plus its confidence, privately, with no round-trip.

  • Emotion-aware triage — route an inbound message on its dominant feeling: anger/disgust → escalate, fear/sadness → empathy queue, joy → testimonial.
  • Content mood tags — give each review, post, or diary line a single emotion badge for search and analytics.
  • Story + lyric arcs — plot the dominant emotion line by line to see how a poem or song moves.
  • Private wellbeing checks — track the emotional shape of a journal entirely on-device.

How the API works

Single-label classification is one call; ask for all 7 scores with top_k:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "text-classification",
  "onnx-community/emotion-english-distilroberta-base-ONNX",
); // downloads + caches ~82 MB once

const out = await classify("what a wonderful surprise!", { top_k: 7 });
// → [{ label: "joy", score: 0.86 }, { label: "surprise", score: 0.11 },
//    { label: "neutral", score: 0.01 }, … all 7, and they SUM to ~1 ]

The config uses the default problem_type, so Transformers.js applies a softmax across the 7 classes — that's why exactly one wins and the scores sum to 1. All of it runs in a Web Worker so typing never janks.
