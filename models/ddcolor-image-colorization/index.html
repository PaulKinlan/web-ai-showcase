DDColor — bring black-and-white photos to life

Give this model a grayscale photo and it invents plausible colour: blue skies, green foliage, warm skin tones, painted walls. DDColor predicts chroma only — the brightness and every fine detail come straight from your original image, so nothing is blurred or hallucinated into the structure; only the colour is added. It runs entirely on your device via ONNX Runtime Web; the photo never leaves the tab.

How to read it

Warm regions (reds, oranges) are where the model expects skin, brick, wood or sunset; cool regions (blues, greens) are sky, water, foliage. Flat gray means the model is unsure and left it near-neutral. Because this is only 512×512 chroma, it is intentionally soft — the sharp detail all comes from your luminance.

Why it matters

Colourisation used to mean a cloud service (upload your family photos to who-knows-where) or a desktop tool. Running it in the browser makes it private and instant — the archive never leaves the device. And because DDColor only predicts colour, it slots into any pipeline without degrading the original:

  • Photo-restoration apps — colourise scanned black-and-white family or archive photos client-side; nothing uploaded.
  • Museums & archives — batch-preview colour for historical collections in a static web page, no backend or per-image API cost.
  • Creative tooling — a “re-colour” brush: strip colour, let the model re-imagine it, then dial the intensity or blend regions.
  • Accessibility & engagement — bring dull scans to life for storytelling, education, and social sharing without a server round-trip.

DDColor invents plausible colour, not the true historical colour — treat it as an artistic reconstruction, not a factual record.

How the API works

transformers.js has no DDColor class and no image-colorization task, so we run the ONNX graph directly with ONNX Runtime Web. The model wants luminance and returns chroma:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(ddcolorBytes, { executionProviders: ["wasm"] });

// image → grayscale RGB rebuilt from the L channel, resized to 512×512, values [0,1] (no normalise)
const input = new ort.Tensor("float32", grayFrom(image), [1, 3, 512, 512]);
const { [session.outputNames[0]]: ab } = await session.run({ [session.inputNames[0]]: input });

// ab is [1,2,512,512] chroma → upsample, recombine with the ORIGINAL full-res luminance, Lab→RGB
const colour = labToRgb(originalL, ab.data);

Inference and the whole Lab→RGB composite run in a Web Worker so the UI never blocks, and the 130 MB model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
