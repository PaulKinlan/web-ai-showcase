Multi-model · inpaint → super-resolution · WASM

Erase an object, then upscale the result

A real editing pipeline: first LaMa removes something and fills the gap, then Real-ESRGAN upscales the cleaned photo 4× — sharper, larger, and object-free. Two models, chained, entirely local.

Run it

Two models load below (LaMa ~208 MB, Real-ESRGAN ~67 MB). Pick a photo, brush over something to remove, then run the pipeline.

1 · LaMa (inpainting)
2 · Real-ESRGAN (4× super-resolution)

Inpainted

Then upscaled 4×

inpaint –upscale –final size –

Both models run on-device; the image, the mask, the inpaint, and the upscale never leave the tab.

Why chain them

Object removal and upscaling are the two edits people reach for most, and they compose naturally: clean the image first, then enlarge the clean version so the fill is upscaled along with everything else — no server round-trip between stages, each a real model running in your browser.
