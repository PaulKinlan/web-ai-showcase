LaMa — erase an object, fill the gap

Brush over something you want gone — a passer-by, a power line, a blemish — and LaMa fills the region with plausible background, reconstructing texture and structure that were never there. LaMa (WACV 2022) is a large-mask inpainting network built on Fourier convolutions, so it can fill big holes coherently, not just tiny scratches. Runs entirely on your device via ONNX Runtime Web; the image and the mask never leave the tab.

How to read it

Indigo = pixels invented by LaMa; dimmed = your original, copied through untouched. A good inpaint has the indigo region blend seamlessly into the surrounding structure when you look at the result — LaMa's Fourier convolutions let it continue periodic texture (brick, foliage, water) across the hole rather than smudging it.

Why it matters

"Remove this thing from my photo" is one of the most-wanted edits there is. On-device inpainting means you can do it privately, with no upload and no per-image cloud cost:

  • Photo apps — a "magic eraser" / object-removal brush that runs client-side on the user's phone or laptop, no server.
  • Privacy redaction — paint out a face, license plate, or bystander and fill the gap with background before sharing — the original never leaves the device.
  • Restoration — remove scratches, dust, date-stamps, or watermarks from a scan and reconstruct the underlying image.
  • Real-estate & product shots — erase clutter, cables, or reflections from a listing photo entirely in the browser.

LaMa fills big masks coherently, but it reconstructs plausible content, not the truth — it's an eraser-and-filler, not an undelete. Very large holes over complex subjects fill approximately.

How the API works

transformers.js has no LaMa class and no inpainting task, so we run the ONNX graph directly with ONNX Runtime Web. In go the image and a binary mask, out comes the filled image:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(lamaBytes, { executionProviders: ["wasm"] });

// image → [1,3,512,512] in [0,1]; mask → [1,1,512,512] with 1 = fill, 0 = keep
const feeds = {
  image: new ort.Tensor("float32", toNCHW(img, 512, 512), [1, 3, 512, 512]),
  mask:  new ort.Tensor("float32", toMask(userMask, 512, 512), [1, 1, 512, 512]),
};
const { output } = await session.run(feeds);   // [1,3,512,512] in [0,255]

// composite the filled region back over the original: final = orig·(1−mask) + output·mask
const filled = composite(img, output.data, userMask);

Inference and the whole output composite run in a Web Worker so the UI never blocks, and the 208 MB model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
