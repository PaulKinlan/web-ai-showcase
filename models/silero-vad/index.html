Silero VAD — find the speech in any audio

A voice activity detector answers one small, useful question, thirty-odd times a second: is someone speaking right now? Silero VAD is a tiny (~2 MB) recurrent network that reads audio in 512-sample frames and emits a speech probability for each one. It's the quiet workhorse in front of almost every speech system — it decides what to send to a transcriber, when to stop recording, and where the pauses are. Here it runs entirely on your device.

Why it matters

VAD is infrastructure. It's cheap, it's fast, and it makes everything downstream better — which is exactly why it belongs on-device: no audio needs to leave the browser just to decide whether it's worth processing.

  • Cut transcription cost and latency — run an expensive ASR model (Whisper) only on the frames that actually contain speech; skip the silence. Often the biggest single win in a voice pipeline.
  • Endpointing — know when a speaker has finished so a voice UI can respond, or a recorder can auto-stop, without a "press stop" button.
  • Trim & segment — auto-remove dead air from recordings, or split a long file into utterances for captioning or dataset building.
  • Push-to-talk that just listens — a talk-meter that lights up when you speak, with zero cloud round-trip and nothing recorded.

How the API works

There's no pipeline() for VAD — you drive onnxruntime-web directly. It's a stateful loop: feed 512-sample frames, carry the LSTM state forward, read the probability out.

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(
  "https://huggingface.co/onnx-community/silero-vad/resolve/main/onnx/model.onnx",
  { executionProviders: ["wasm"] },
);

let state = new ort.Tensor("float32", new Float32Array(2 * 1 * 128), [2, 1, 128]);
const sr = new ort.Tensor("int64", [16000n], []);          // scalar

for (const frame of frames512) {                            // Float32Array, 512 samples @ 16 kHz
  const input = new ort.Tensor("float32", frame, [1, 512]);
  const { output, stateN } = await session.run({ input, state, sr });
  const speechProb = output.data[0];                        // 0 … 1 for this 32 ms frame
  state = stateN;                                           // carry the recurrent state forward
}

This page decodes files / mic recordings to 16 kHz mono with an OfflineAudioContext, runs the frame loop inside a Web Worker so the UI never janks, caches the ~2 MB model in Cache Storage (so a return visit auto-initialises with no download), and turns the raw curve into segments with a threshold + hysteresis pass.
