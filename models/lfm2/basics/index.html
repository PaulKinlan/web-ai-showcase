Basics: local chat, steered by a system prompt

The fundamental move: send a message, stream a reply — all on-device. A system prompt sets the assistant's persona and rules; greedy decoding is deterministic while sampling adds variety. Change the system prompt and watch the behaviour shift.

The system prompt is prepended once, before your first turn, and stays in force for the whole conversation. Greedy decoding always picks the single most likely next token (repeatable); sampling draws from the distribution, so higher temperature = more varied, occasionally less coherent.
