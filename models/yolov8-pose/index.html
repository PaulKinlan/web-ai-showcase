YOLOv8-pose — find everyone's skeleton in one pass

YOLOv8-pose is a single-stage pose estimator: one forward pass over the whole image returns a box, a confidence, and 17 COCO keypoints for every person at once — so it poses a whole crowd without a separate detector. That's a different design from the top-down ViTPose demo (which needs a person box per crop) and from MediaPipe BlazePose. At ~13 MB it runs on the universal WebAssembly/CPU path — no WebGPU, no cloud, all on your device.

At a glance

ModelXenova/yolov8n-pose — ONNX export of Ultralytics YOLOv8n-pose
Taskkeypoint-detection — single-stage detector + pose head
ArchitectureYOLOv8n backbone/neck + a pose head → one [1, 56, 8400] tensor (4 box + 1 conf + 17×3 keypoints per anchor)
OutputPer person: a box, a confidence, and 17 COCO keypoints (x, y in image pixels + a visibility each)
ApproachBottom-up single-stage — finds and poses every person in one pass (multi-person, no separate detector)
Quantizationfp32 (onnx/model.onnx) — decoded with Non-Max-Suppression
Runtimeonnxruntime-web (raw ONNX Runtime Web) — no transformers.js YOLO class exists
BackendWebAssembly (runs anywhere; no WebGPU needed)
Download~13 MB, cached after first load
LicenseAGPL-3.0 (Ultralytics YOLOv8 — same family/license as the built YOLOv10 & YOLO11 demos)
Web APIsWebAssembly, Web Workers, OffscreenCanvas, Canvas, FileReader — all Baseline widely available

Why it matters

A single-stage pose model turns "where is everyone and how are they standing?" into one cheap, private, offline function call. Because detection and pose are fused, a crowded scene costs one pass, not one-per-person — ideal for real-time and mobile.

  • Crowd & sports analytics — pose every player on a pitch in one shot and read formations, spacing, or actions.
  • Fitness & physio — check squat depth, plank alignment, or arm angle from joint coordinates, with nothing uploaded.
  • Interactive installations — drive avatars, games, or art from live full-body motion, entirely in the browser.
  • Safety & ergonomics — flag falls, slouching, or unsafe lifting posture from shoulder/hip/knee geometry, on-device.

How the API works

There's no transformers.js YOLO class, so this page runs the ONNX graph directly with onnxruntime-web and does the two pieces a pipeline would own — letterbox in, decode + NMS out:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(modelBytes);       // ~13 MB, cached once
// letterbox the image to 640×640 (aspect-preserving pad, /255, CHW, RGB) → Float32Array
const input = new ort.Tensor("float32", chw, [1, 3, 640, 640]);
const { output0 } = await session.run({ images: input });            // [1, 56, 8400]

// per anchor: [0..3]=box cx,cy,w,h · [4]=conf · [5..55]=17×(x,y,visibility), all in 640 space
// filter by conf, run Non-Max-Suppression, then unpad/unscale back to source pixels.

This page runs exactly that in a Web Worker (with the letterbox on an OffscreenCanvas), and returns the raw objectness grid so "See inside" can show the pre-NMS detections. Multiple people come out of the single pass automatically.
