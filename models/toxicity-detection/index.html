toxic-bert — score a comment for toxicity, privately

A BERT fine-tuned on the Jigsaw toxic-comment data. It reads any text and emits six independent toxicity signals — toxic, severe toxic, obscene, threat, insult, identity hate — each a 0–100% probability. This is defensive content-safety tooling: set a threshold, and triage what to flag. Every score is computed on your device; no comment is ever sent to a server.

Run it

Type or paste a comment; it re-scores automatically a moment after you stop typing. The moderation threshold decides the verdict: if any label lands at or above it, the comment is flagged for review — otherwise it's allowed. See inside — six sigmoids, not one softmax

A sentiment model picks one winner with a softmax (the scores sum to 100%). Toxicity is multi-label: the model's final layer produces six raw logits, and each passes through its own sigmoid independently. So a single comment can be 0.99 toxic and 0.95 insult at once — the columns below don't add up to anything in particular, and that's the point. The table shows each label's raw logit next to its sigmoid probability, so you can see exactly how the threshold turns numbers into a decision.

A softmax over these would force one winner and hide the fact that this text is several kinds of bad at once. Independent sigmoids keep every signal legible — the moderation queue can route a threat differently from an obscene.

Why it matters

Moderation is a scale problem: no team can read every comment, DM, or review. A fast, private, offline toxicity score is a first line of defence — it runs the instant someone hits submit, without the text ever leaving the tab, and hands a human queue only what actually needs eyes.

  • Pre-submit guardrails — warn a user their draft reads as an insult or threat before they post it, client-side.
  • Moderation-queue triage — auto-sort a comment stream so the worst (threat, identity hate) rises to the top and the benign majority is auto-approved.
  • Private-by-default safety — score sensitive user text (support tickets, health forums) without a server round-trip or third-party API.
  • Cost control — gate an expensive human review or a big LLM behind a free on-device filter that clears the easy cases.

Use cases

Basics

Score a single comment and read all six independent toxicity labels.

Practical

A moderation queue: paste many comments, set a threshold, auto-sort into flag / allow.

Wild

Edge cases — sarcasm, reclaimed slurs, quoted abuse — where the classifier's calibration shows its limits.

Multi-model

A toxicity gate in front of a sentiment model: only clean comments pass through to analysis.

How the API works

The convenience path is one call; because the model is multi-label, ask for every label with topk: null:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline("text-classification", "Xenova/toxic-bert");
const out = await classify("you are worthless", { topk: null });
// → [{ label: "toxic", score: 0.98 }, { label: "insult", score: 0.64 }, …]  (independent sigmoids)

This page goes one level deeper for "See inside": instead of the pipeline it runs the tokenizer then the model() in a single forward pass, reads the raw six-value logits tensor, and applies the sigmoid itself — so it can show both the logit and the probability, and apply your threshold. Same weights, same result. All of it runs in a Web Worker so typing never janks.
