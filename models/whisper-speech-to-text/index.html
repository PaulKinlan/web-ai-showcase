Whisper — turn speech into text, on your device

Whisper listens to audio and writes down what was said — in your browser, with no server and no upload. Record from your microphone, drop in an audio file, or use a sample clip, and get a transcript with word-level timestamps you can see line up with the waveform.

Why it matters

Speech-to-text used to mean shipping audio to a cloud API — a bill, a privacy problem, and a network dependency. Whisper-base runs on the visitor's own device, so transcription becomes a free, private, offline capability of your page.

  • Private dictation — notes, forms, and search-by-voice where the audio never leaves the tab (medical, legal, journaling).
  • Captions & subtitles — generate timed captions for user-recorded video/audio client-side, with real word timings to align them.
  • Voice commands — a hands-free control layer for any web app, with no per-request inference cost.
  • Meeting / voice-note transcripts — turn a recording into searchable, timestamped text without uploading a thing.

How the API works

Transcription is one pipeline call. Word timestamps are a single option:

import { pipeline } from "@huggingface/transformers";

const transcribe = await pipeline(
  "automatic-speech-recognition",
  "onnx-community/whisper-base_timestamped", // whisper-base, exported for word timestamps
  { device: "webgpu", dtype: "q8" }, // WASM fallback when WebGPU is absent
);

// audio = a 16 kHz mono Float32Array (decode with Web Audio, then resample)
const out = await transcribe(audio, {
  return_timestamps: "word",   // "word" | true (segment) | false
  chunk_length_s: 30,          // long-audio chunking
  stride_length_s: 5,
});
// out.text  → the full transcript
// out.chunks → [{ text: " country", timestamp: [9.8, 10.2] }, …]

This page decodes the mic/file/sample to 16 kHz mono with the Web Audio API on the main thread, then transfers just the Float32 samples to a Web Worker where the pipeline runs — so the model never blocks the interface. Word chunks are grouped into the sentence-level segments shown under "See inside".
