Multi-model · deblur → text detection · WASM

Restore, then find the hidden text

A blurry document hides its own text from a detector. Here NAFNet restores the image first, then the on-device DBNet text detector runs on the result — and finds text regions the blur was smothering. Two models, chained, entirely local.

Run it

Two models load below (deblur ~92 MB, detector ~2.4 MB). Pick a text sample, then run the pipeline — we detect on the blurred image and on the restored one so you can see the gain.

2 · DBNet (text detection)

deblur –detect –regions –

Both models run on-device; the image, the restoration, and the boxes never leave the tab.

Why chain them

Restoration is a force-multiplier for downstream vision: a detector, OCR model or classifier all see cleaner structure after a deblur. This is the on-device version of a classic document pipeline — no server, no upload, each stage a real model running in your browser.
