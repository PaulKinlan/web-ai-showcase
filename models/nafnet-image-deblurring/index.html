NAFNet — sharpen a blurred photo

Give this model a blurred photo — camera shake, motion, a soft focus — and it reconstructs the sharp one: crisp edges, readable text, recovered texture. NAFNet is a restoration network (ECCV 2022) from the OpenCV Model Zoo, trained on real motion blur. It's not a sharpen filter — it learned what un-blurred scenes look like and rebuilds the detail. Runs entirely on your device via ONNX Runtime Web; the photo never leaves the tab.

How to read it

Bright indigo = big change (an edge or letter the model re-sharpened); dark = untouched. A pure sharpening filter would light up everywhere it found contrast; NAFNet concentrates on the structure that the blur actually smeared, which is why the delta traces objects, not noise.

Why it matters

A blurred shot is usually a lost shot — you can't re-take the moment. On-device restoration means you can rescue it privately, with no upload, no per-image cloud cost:

  • Photo apps — a "de-shake" / "un-blur" button that runs client-side on the user's phone or laptop, no server.
  • Document capture — sharpen a shaky photo of a receipt, whiteboard or page so text becomes readable (and OCR-able) again.
  • Archives & scanning — batch-restore soft scans in a static web tool with zero backend.
  • Pre-processing for other models — feed a cleaner image into detection, OCR, or classification so they see sharp structure.

NAFNet reconstructs plausible detail; it can't invent information the blur erased entirely. Extreme blur restores partially — it's a rescue, not a time machine.

Use cases

Basics

Blur a photo, then watch NAFNet rebuild it — wipe between blurred and restored and see the sharpness jump.

Practical

Rescue a shaky document photo so the text is legible again, then download the restored image.

Wild

Push it: crank the blur up and find the breaking point where restoration starts to give up.

Multi-model

Restore → detect text: deblur a photo, then run the text detector on the result to find text the blur hid.

How the API works

transformers.js has no NAFNet class and no deblurring task, so we run the ONNX graph directly with ONNX Runtime Web. In goes the blurred image, out comes the restored one:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(nafnetBytes, { executionProviders: ["wasm"] });

// blurred image → [1,3,512,512] float32 in [0,1] (fixed network size, no normalisation)
const input = new ort.Tensor("float32", toNCHW(blurred, 512, 512), [1, 3, 512, 512]);
const { [session.outputNames[0]]: restored } = await session.run({ [session.inputNames[0]]: input });

// restored is [1,3,512,512] in [0,1] → clamp to RGBA and resize back to the original aspect
const sharp = tensorToImage(restored.data, 512, 512);

Inference and the whole output composite run in a Web Worker so the UI never blocks, and the 92 MB model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
