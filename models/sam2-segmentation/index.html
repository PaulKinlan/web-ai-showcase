SAM 2.1 — click or box a point, get its mask

Segment Anything 2 is Meta's newer promptable segmenter: a hierarchical Hiera backbone that emits a multi-scale feature pyramid, driving a prompt encoder and mask decoder. Point at anything — or drag a box around it — and SAM 2.1 returns three candidate masks with predicted-IoU scores. The heavy image encoder runs once; every extra prompt is fast. The same architecture powers SAM 2 video tracking; this page runs the image path, on-device.

SAM 2 predicts three candidate masks per prompt — roughly "the part / the thing / the whole region" — and rates each with a predicted IoU, its own guess of mask quality. It also emits an object-presence score (a logit for "is there really an object here"). Why it matters

SAM 2 makes "select exactly the thing I mean" a fast, local, interactive operation — with better masks than SAM 1 and a box prompt when a click is ambiguous. It's the building block under smart selection, cut-out, measurement, and annotation tools, and its memory architecture extends the same prompt to video.

  • Precise cut-out — click or box a subject and lift it onto transparency for stickers, collages, or product shots.
  • Smart selection — a "magic select" tool in a browser image editor, no server and no upload.
  • Measurement & analytics — turn a mask into a pixel area / coverage percentage for inspection or science.
  • Annotation at scale — one prompt becomes a precise segmentation label, slashing dataset-building cost.

How the API works

SAM 2 uses Sam2Model + Sam2Processor directly. Encode the image once, then decode per prompt:

import { Sam2Model, Sam2Processor, RawImage, Tensor } from "@huggingface/transformers"; // v4+

const model = await Sam2Model.from_pretrained("onnx-community/sam2.1-hiera-tiny-ONNX", {
  dtype: { vision_encoder: "fp16", prompt_encoder_mask_decoder: "fp32" }, device: "wasm",
});
const processor = await Sam2Processor.from_pretrained("onnx-community/sam2.1-hiera-tiny-ONNX");

// 1. Encode the image ONCE — the Hiera backbone returns a multi-scale pyramid (image_embeddings.0/1/2):
const image = await RawImage.read(imageURL);
const inputs = await processor(image);
const embeddings = await model.get_image_embeddings(inputs);

// 2. Per prompt, decode masks. SAM 2 pads to 1024×1024, so scale prompts into reshaped_input_sizes:
const [rH, rW] = inputs.reshaped_input_sizes[0];
const input_points = new Tensor("float32", [nx * rW, ny * rH], [1, 1, 1, 2]);
const input_labels = new Tensor("int64", [1n], [1, 1, 1]); // 1 = foreground, 0 = background
// (or a box:) const input_boxes = new Tensor("float32", [x1*rW,y1*rH,x2*rW,y2*rH], [1, 1, 4]);
const out = await model({ ...embeddings, input_points, input_labels });

const masks = await processor.post_process_masks(
  out.pred_masks, inputs.original_sizes, inputs.reshaped_input_sizes,
); // masks[0]: [1, 3, H, W] bool — three candidates; rank by out.iou_scores

This page runs all of that in a Web Worker and caches the embeddings, so the first prompt pays for the vision encoder and every prompt after is just the lightweight mask decoder.
