Practical: structured extraction to strict JSON

A workhorse business task: pull structured fields out of unstructured text. Paste a message, list the fields you want, and OLMo-2 returns JSON — which this page runs JSON.parse() on live to prove it's machine-readable, not just a paragraph that looks like JSON. All on-device, so the source text (often sensitive) never leaves the browser.

Why this works locally

Extraction is a constrained task — the model isn't inventing, it's re-shaping text you already have. A 1B instruction model handles it well with a firm system prompt ("return only JSON with these keys"), a low temperature for determinism, and a client-side JSON.parse() as the validity gate. Because it runs on-device, you can throw genuinely sensitive documents at it — HR notes, medical intake, contracts — without any of it touching a server.
