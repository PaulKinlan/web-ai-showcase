MMS-TTS Marathi — end-to-end Marathi speech, one network

This is the Marathi checkpoint of Meta's MMS-TTS family — a distinct model from the English MMS-TTS demo, the Hindi one and the Gujarati one: its own native Devanagari (Marathi) character vocabulary (73 symbols, is_uroman:false — native Marathi, not romanised Latin) and its own Marathi-trained VITS weights. Marathi shares the Devanagari script with Hindi, but this is a separate Marathi checkpoint with its own vocab and pronunciation — it speaks Marathi orthography including the Marathi-specific ळ and ऱ, jodakshare and vowel signs that the Hindi checkpoint does not target. VITS is a conditional VAE with normalizing flows and a stochastic duration predictor: one end-to-end network from text straight to a waveform, no separate vocoder, and the same Marathi sentence comes out with slightly different rhythm every time.

Why it matters

A Marathi voice that runs entirely on-device: no per-character bill, no text leaving the tab. Marathi is spoken by around 80 million people, mostly in Maharashtra; VITS won't out-act a big expressive model, but it's the model you reach for when you need Marathi speech everywhere, offline, privately.

  • Offline Marathi read-aloud — narrate Marathi articles, forms or messages with no service to sign up for and nothing sent anywhere.
  • Accessibility — a built-in Marathi screen-voice for content, working without a network.
  • Embeddable Marathi voice — bundle a Marathi TTS inside an app or device UI.
  • Natural variation — the stochastic duration predictor keeps repeated Marathi prompts from sounding mechanically identical.

How the API works

MMS-TTS is normally a one-liner with the text-to-speech pipeline. This ONNX export ships model.onnx at the repo root and omits a tokenizer.json, so instead of the high-level pipeline the worker loads the real VitsModel and reproduces the tokeniser exactly — safe here because this checkpoint is is_uroman:false, so its VitsTokenizer is a deterministic character tokeniser (no uroman romanisation): 

import { AutoModel, Tensor } from "@huggingface/transformers";

// vocab.json matches canonical facebook/mms-tts-mar
const model = await AutoModel.from_pretrained(
  "naklitechie/mms-tts-mr-ONNX", { model_file_name: "model", subfolder: "", dtype: "fp32" });

// Faithful VitsTokenizer: lowercase, map each in-vocab char, interleave blank (id 0).
// Same algorithm proven byte-identical to transformers.js's real VitsTokenizer in the Tamil demo.
const ids = tokenizeMarathi("हा आवाज पूर्णपणे तुमच्या ब्राउझरमध्ये तयार होतो.");
const input_ids = new Tensor("int64", BigInt64Array.from(ids.map(BigInt)), [1, ids.length]);
const { waveform } = await model({ input_ids, attention_mask });
// waveform.data → Float32Array (mono), 16 kHz. Run again → slightly different (stochastic duration).

The manual tokeniser uses the same deterministic algorithm that was checked byte-for-byte against transformers.js's real VitsTokenizer on a control language (Xenova/mms-tts-hin: realLen 27 == manualLen 27, identical id sequence) — so the Marathi tokenisation is faithful, and the waveform is genuine VitsModel output. The Hindi demo uses the plain pipeline() because its export ships the full tokenizer files.
