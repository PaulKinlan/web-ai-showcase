AnimeGANv2 — turn a photo into anime

Give this model any photo and it repaints it in a learned anime style — smoothed shading, bold outlines, painterly skies. AnimeGANv2 is a GAN trained to convert real photographs into the look of specific animators: Hayao (Ghibli-ish), Shinkai (Makoto Shinkai's luminous scenery), and Paprika (Satoshi Kon). It's not a colour filter — it re-draws the scene. Runs entirely on your device via ONNX Runtime Web; the photo never leaves the tab.

How to read it

Notice the model doesn't just recolour — it smooths flat regions, thickens edges into "line art", and repaints skies and foliage with painterly texture. That structural redraw is what makes it read as anime rather than a filtered photo.

Why it matters

Stylizing a photo on-device turns any picture into shareable art, privately and free per image:

  • Creative & social apps — a one-tap "anime-fy my photo" that runs client-side, no server, no per-image cost.
  • Avatars & profile art — turn a selfie or pet photo into an anime avatar entirely in the browser.
  • Storyboarding & concept art — quickly rough out a scene in a chosen animator's look from a reference photo.
  • Backgrounds & assets — restyle landscape photos into anime backdrops for games or videos.

AnimeGANv2 renders a stylistic interpretation — it's an artistic redraw, not a faithful reproduction, and fine text or faces may be simplified.

How the API works

transformers.js has no AnimeGAN class and no stylization task, so we run the ONNX graph directly with ONNX Runtime Web. Note the NHWC (channels-last) layout and the −1…1 tanh range:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(styleBytes, { executionProviders: ["wasm"] });

// photo → [1,H,W,3] float32 in [-1,1] (NHWC, channels-last — it's a TensorFlow export)
const input = new ort.Tensor("float32", toNHWC(photo, H, W), [1, H, W, 3]);
const { ["…/out_layer/Tanh:0"]: out } = await session.run({ ["generator_input:0"]: input });

// out is [1,H,W,3] in [-1,1] → map (v+1)/2 back to RGB
const anime = tanhToImage(out.data, H, W);

Inference and the whole output composite run in a Web Worker so the UI never blocks; each ~8 MB style is fetched through Cache Storage the first time you pick it, so a returning visit auto-initialises with no re-download.
