DistilBERT SQuAD — find the answer inside a passage

This is extractive question answering: give it a passage and a question, and instead of writing a new sentence it points at the exact span of the passage that answers you — and tells you how confident it is. Type a question below and watch it highlight the answer, then look inside to see the real start/end token probabilities it used to pick that span. Nothing leaves your device.

See inside — the start/end token probabilities

The model never "reads" an answer out — it outputs two numbers per passage token: how likely that token is the first word of the answer, and how likely it is the last. The answer is just the span from the best start to the best end. Why it matters

Extractive QA is the quiet engine behind "answer from my documents". Because it returns a span of the source rather than inventing text, the answer is always grounded — it literally can't hallucinate a fact that isn't in the passage. And at 65 MB it runs in a tab, so you can query private documents, contracts, or logs without any of it leaving the device.

  • Doc search that answers — not "here are 10 matching pages" but "here's the exact sentence".
  • FAQ bots over your own text — point it at a help-centre article and let users ask.
  • Grounded RAG — pull the precise supporting span to cite alongside a generated answer.
  • Private extraction — pull dates, amounts, and names out of a pasted document, on-device.

How the API works

The pipeline takes a question and a context and returns the best span with a score:

import { pipeline } from "@huggingface/transformers";

const qa = await pipeline(
  "question-answering",
  "Xenova/distilbert-base-cased-distilled-squad",
); // downloads + caches ~65 MB once

const out = await qa(
  "How tall is the Eiffel Tower?",
  "The tower is 330 metres tall …",
  { topk: 5 },
);
// → [{ answer: "330 metres", score: 0.97, start: 12, end: 22 }, …]

"See inside" reuses that same loaded model instance (qa.model + qa.tokenizer) to read the raw start_logits / end_logits, softmax them over the passage tokens, and draw the distribution. All of it runs in a Web Worker so the UI never janks.
