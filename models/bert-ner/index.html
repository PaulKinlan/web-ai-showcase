See inside — the per-token BIO tags

The model doesn't see "Angela Merkel" as one thing — it tags each WordPiece token independently: B-PER (begin person), I-PER (inside person), and so on. We then stitch consecutive B-/I- runs into whole entities. Below are the raw tags and confidences straight from the model — including the occasional wobble (watch a stray I-ORG on a preposition), which is exactly why the confidence matters.

Why it matters

NER turns free text into structured data — the names and places a downstream system can actually act on. Doing it in the browser means you can index, redact, or link entities without a single byte of the source text leaving the device.

  • PII redaction — mask people and places in a document client-side before it's shared or logged.
  • Contact / data extraction — pull the who-and-where out of emails, notes, or transcripts.
  • Smart search & tagging — auto-tag articles by the people and organizations they mention.
  • Entity graphs — map who appears with whom across a body of text.

How the API works

Token classification is one pipeline call:

import { pipeline } from "@huggingface/transformers";

const ner = await pipeline("token-classification", "Xenova/bert-base-NER");
const out = await ner("Tim Cook runs Apple in California");
// → [{ entity: "B-PER", word: "Tim", score: 0.999, index: 1 },
//    { entity: "I-PER", word: "Cook", score: 0.999, index: 2 },
//    { entity: "B-ORG", word: "Apple", score: 0.998, index: 4 },
//    { entity: "B-LOC", word: "California", score: 0.994, index: 6 }]

The pipeline returns one BIO-tagged row per WordPiece token, but no character offsets. We map each token back to its span in the original text (a cursor-walk) so highlights land exactly, then merge B-/I- runs (and ## sub-word pieces) into whole entities with a pooled confidence. It all runs in a Web Worker so typing stays smooth.
