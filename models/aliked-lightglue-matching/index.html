ALIKED + LightGlue — match two images by their features

A different vision task from the pose and landmark demos: local-feature matching. ALIKED detects and describes up to 1000 distinctive keypoints in each image; LightGlue, a learned attention matcher, finds the correspondences — the keypoint pairs that land on the same physical point across both photos. That's the engine behind panorama stitching, visual place recognition, duplicate detection, and AR tracking — running entirely on your device.

Why it matters

Feature matching is the geometric backbone of computer vision. Doing it on-device — no upload, no API — unlocks private, offline tools that reason about where images relate, not just what they contain.

  • Panorama & document stitching — align overlapping photos or scanned pages from their shared features.
  • Duplicate & near-duplicate detection — decide if two photos show the same scene, robust to crop, rotation, and lighting.
  • Visual place recognition & AR — relocalise a camera against a reference image, or anchor an overlay to a tracked surface.
  • Change detection — align a before/after pair, then diff the registered images.

Use cases

Basics

Match a scene against a rotated/scaled copy and watch the correspondence lines track the transform.

Practical

Duplicate detector: upload two photos and get a "same scene?" verdict from the match count + inliers.

Wild

Auto-align: estimate the rotation/scale between two views from the matches and overlay B onto A.

Multi-model

Retrieve & verify: DINOv2 embeddings shortlist the closest gallery image, then LightGlue confirms it geometrically.

How the API works

No transformers.js matcher class exists, so we run the two ONNX graphs directly with onnxruntime-web:

import * as ort from "onnxruntime-web";
const aliked = await ort.InferenceSession.create(alikedBytes);   // ~6 MB
const lightglue = await ort.InferenceSession.create(lgBytes);    // ~45 MB

// ALIKED: image [1,3,640,640] (RGB, /255) → keypoints [1000,2] (normalised),
//         descriptors [1000,128], scores [1000]
const a = await aliked.run({ image: tensorA });
const b = await aliked.run({ image: tensorB });

// LightGlue: add a batch dim; match the two descriptor sets
const m = await lightglue.run({
  kpts0: batch(a.keypoints), kpts1: batch(b.keypoints),
  desc0: batch(a.descriptors), desc1: batch(b.descriptors),
});
// m.matches0 → [M, 2] index pairs · m.mscores0 → [M] confidences

This page runs exactly that in a Web Worker (with the resize on an OffscreenCanvas), maps the normalised keypoints back to pixels, and draws a line per correspondence.
