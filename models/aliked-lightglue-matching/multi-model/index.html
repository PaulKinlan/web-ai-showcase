Retrieve, then verify

Real visual search is two stages. DINOv2 embeddings give a fast global shortlist — which gallery images look like the query. Then ALIKED + LightGlue geometrically verifies the top hit: do concrete local features actually correspond? That turns a soft similarity into a hard match — the pipeline behind place recognition and re-localisation, all on-device.
