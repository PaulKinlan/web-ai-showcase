StableLM 2 Zephyr 1.6B — Stability AI's tiny chat model on your GPU

A compact, capable chat model from Stability AI, running entirely in your browser. StableLM 2 Zephyr 1.6B is compiled to WebGPU by MLC's WebLLM engine — you get streaming tokens, sampling controls, a real system prompt and full conversation history, with nothing uploaded and no API bill. It's a distinct architecture family from the Llama- and Gemma-based chat models here, and like every WebLLM model it needs a real GPU: no WASM fallback.

Why it matters

A useful chat model that fits in a ~1 GB download changes what you can ship to the client. StableLM 2 Zephyr on WebLLM makes a conversational model a client-side primitive: private by construction, free to run, and available offline once the weights are cached — small enough to reach mid-range laptops and phones, capable enough for real assistant work.

  • Private assistants — a chat helper where the conversation never leaves the device; ideal for health, legal, or journalling apps.
  • Lightweight on-device UX — a 1.6B model with a real system role slots into "summarise", "rewrite", or "explain" buttons without the ~4 GB weight of a 7B model.
  • Zero-cost features — ship conversational features with no inference bill and no rate limits, however many users you have.
  • On-device RAG — pair it with a local embedding model to answer questions grounded in a user's own documents, all in the tab (see the multi-model demo).

How the API works

WebLLM gives you an OpenAI-shaped chat API backed by a WebGPU engine. You create the engine (which downloads and compiles the weights), then stream a completion. This page uses the shared helpers in lib/webllm.js, running inside a Web Worker so the UI stays responsive. Because StableLM 2 Zephyr has a real system role, the system prompt is just a system message: 

import { webGPUAdapterAvailable, createEngine, streamChat }
  from "/web-ai-showcase/lib/webllm.js";

// WebLLM is WebGPU-only — gate before doing anything (headless requestAdapter() → null).
if (!(await webGPUAdapterAvailable())) {
  showNeedsWebGPU();          // never a faked reply
} else {
  const engine = await createEngine({
    model: "stablelm-2-zephyr-1_6b-q4f16_1-MLC",
    onProgress: (p) => setProgress(p.text, p.progress),   // ~1 GB, cached after first load
  });

  // StableLM 2 Zephyr has a real system role — no folding.
  const messages = [
    { role: "system", content: "You are a concise assistant." },
    { role: "user", content: "Explain WebGPU in one sentence." },
  ];

  const full = await streamChat(
    engine,
    { messages, temperature: 0.7, top_p: 0.95, max_tokens: 256 },
    (delta) => appendToken(delta),          // streams token-by-token on your GPU
  );
  // engine.interruptGenerate() stops decoding early; engine.runtimeStatsText() reports tok/s.
}

Everything here runs exactly that path in worker.js, and gates on a real WebGPU adapter first. No adapter → a labelled needs-WebGPU state, never a canned answer.
