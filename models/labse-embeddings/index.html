LaBSE — one meaning space, trained to retrieve translations

LaBSE (Language-agnostic BERT Sentence Embedding) is a dual-encoder Google trained with a translation-ranking objective across 109 languages. Where the other embedders here learned meaning as a side effect, LaBSE was optimised for one thing: pull a sentence and its human translation to nearly the same 768-dimensional point, and push every unrelated sentence away with a margin. That makes it exceptional at bitext mining — given a pile of candidates in many languages, find the one that is the translation of a source sentence. Type mixed-language lines to build a cross-lingual cosine matrix, then watch it match sources to their translations across languages. Nothing leaves your device.

Bitext mining — LaBSE's signature task

This is what LaBSE was built for. Give a column of source sentences and a shuffled pool of candidate translations (mixed languages, plus a distractor that matches nothing). LaBSE embeds both sides and pairs each source with its true translation by cosine — no language labels, no dictionary. The margin column is the gap between the correct match and the best runner-up: a big margin is the model saying "this is clearly the translation".

Each source is matched to its closest candidate by cosine. The extra candidate line matches no source — watch it stay unpicked, or picked only with a tiny margin.

Why it matters

A translation-aware embedder turns "align the parallel data" and "find the matching passage across languages" from a pipeline of dictionaries and MT into a single vector lookup. LaBSE was literally used to mine billions of parallel sentence pairs from the web to train translation systems — and it runs, in full, in a browser tab.

  • Bitext / parallel-corpus mining — pair source and translated sentences from two piles of text with no alignment metadata.
  • Cross-language search & dedup — one 768-d index across 109 languages; an English query finds its French, Japanese or Arabic answer.
  • Translation QA — score whether a candidate translation actually means the same as the source (high cosine) or has drifted.
  • Language-agnostic clustering — group a mixed-language corpus by topic, not by which language it happens to be in.

Use cases

Basics

Is this a translation? One source, several languages, one cross-lingual cosine number each.

Practical

Bitext mining: align a column of sources to their shuffled translations across languages, with accuracy.

Wild

Cluster a mixed-language corpus by meaning — watch the groups ignore language entirely.

Multi-model

Retrieve cross-lingually with LaBSE, then translate the winning passage to English with NLLB-200.

Contrast with the smaller Multilingual MiniLM (384-d, paraphrase objective) and BGE-M3: LaBSE is bigger and tuned specifically for translation retrieval across 109 languages.

How the API works

The same feature-extraction recipe as any sentence embedder — the difference is entirely in the weights:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Xenova/LaBSE",
); // downloads + caches ~490 MB once

// No instruction prefix — embed a source and a candidate IN ANY LANGUAGE the same way:
const out = await extract(["Where is the station?", "最寄りの駅はどこですか？"], {
  pooling: "mean",
  normalize: true,
});
const [q, d] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = q.reduce((s, x, i) => s + x * d[i], 0); // high — the candidate is the translation

This page asks the worker for the un-normalized pooled vectors so "See inside" can show the real pre-normalization magnitude, then normalizes them in JS. Inference runs in a Web Worker so the matrix, projection and bitext mining never jank the page.
