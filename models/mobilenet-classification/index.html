MobileNet — the backbone built to run on a phone

Same 1,000-way ImageNet job as the ViT and ConvNeXt demos, but with a model a fraction of their size. MobileNetV2 is ~3.5M parameters of depthwise-separable convolutions — designed for on-device, real-time inference. Classify an image and read the real millisecond latency; the story here is tiny + fast.

Why fp32 and not q8?

This repo prefers 8-bit (q8) weights, but MobileNetV2's depthwise-separable convolutions quantise degenerately with the generic int8 export: the ConvInteger operator has no WebAssembly kernel, and the uint8/q8 builds emit garbage labels (verified — a tiger photo comes back "envelope"). fp32 is the honest, correct choice and is still tiny at ~14 MB — a fraction of the heavier backbones, which is exactly the point.

Why it matters

MobileNet is the workhorse of on-device vision: small enough to ship inside an app or a web page, fast enough to run on every frame of a camera feed, and accurate enough for a huge range of real tasks. When the constraint is battery, bandwidth, or latency — not the last point of accuracy — this is the backbone you reach for.

  • Real-time camera tagging — classify a live webcam feed frame-by-frame; MobileNet is fast enough to keep up.
  • Instant, offline categorisation — a ~14 MB download tags images the moment they're picked, with no server round-trip.
  • Cheap prefilter — run MobileNet first to triage, and only escalate the hard/uncertain cases to a heavier model like ViT (see the multi-model demo).
  • Low-power edge devices — the small footprint fits constrained hardware where a transformer wouldn't.

How the API works

The convenience path is two lines with Transformers.js:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "image-classification",
  "onnx-community/mobilenet_v2_1.0_224",
  { dtype: "fp32" },
); // downloads + caches ~14 MB once

const output = await classify(imageURL, { top_k: 5 });
// → [{ label: "tiger, Panthera tigris", score: 0.70 }, { label: "tiger cat", score: 0.21 }, …]

This page goes one level deeper for "See inside": instead of the pipeline it runs the model's own processor then model() in a single forward pass, so it can read the raw logits tensor (all 1,001 classes, including the background class at index 0) and compute the softmax, entropy, and margin itself. Inference runs in a Web Worker so the page never janks.
