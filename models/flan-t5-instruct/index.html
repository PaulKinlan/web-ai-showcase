FLAN-T5-small — one model, many tasks, just ask

FLAN-T5 is an encoder-decoder model instruction-tuned on roughly 1,800 tasks. There is no chat template and no system prompt — the whole input is a single natural-language instruction: "summarize: …", "Is this review positive or negative? …", "answer the question based on the context: …". The encoder reads it, the decoder writes the answer one token at a time. Try any instruction and watch it stream. Nothing leaves your device.

This is the small (80M) checkpoint, so calibrate expectations: it is genuinely strong at classification, sentiment, yes/no and reading-comprehension (answering from text you give it) and decent at summarizing — but only roughly translates and is unreliable on closed-book facts. That trade-off is the honest reality of a model this size, and the page shows you the real output, right or wrong — never a canned answer.

Why it matters

One 100 MB model that translates, summarizes, answers, and classifies — driven entirely by plain instructions — is a whole toolbox in a tab. No task-specific model to ship for each feature, no chat template to wrangle, no server bill, and the text never leaves the device.

  • Universal text tool — one input box that does many jobs; swap the instruction, not the model.
  • Offline utilities — quick translate / rephrase / classify in a PWA with no network.
  • Structured extraction — instruct it to pull a field or label and read the answer directly.
  • Pipeline glue — turn transcripts or retrieved snippets into answers on-device.

Use cases

Basics

The same input, five different instructions — feel how the verb changes the task.

Practical

A "universal text tool": one box, a menu of jobs — translate, TL;DR, rephrase, classify.

Wild

"Reason step by step" — coax chain-of-thought out of a tiny model and see it help (or not).

Multi-model

On-device RAG: MiniLM retrieves the right snippet, FLAN-T5 answers from it.

How the API works

The whole task lives in the input string — no template, no roles:

import { pipeline } from "@huggingface/transformers";

const flan = await pipeline(
  "text2text-generation",
  "Xenova/flan-t5-small",
); // downloads + caches ~100 MB once

// the instruction IS the prompt — change the verb to change the task:
await flan("Is the following review positive or negative? Review: The battery dies in an hour.");
// → [{ generated_text: "negative" }]   ← classification is where the small model shines

await flan("Answer the question based on the context.\n" +
  "Context: The museum is closed on Mondays.\nQuestion: When is the museum closed?");
// → [{ generated_text: "Mondays" }]    ← reading-comprehension, not closed-book recall

For the live token stream and the decode timeline, this page passes a TextStreamer so it can timestamp each generated token — all inside a Web Worker so the UI never janks.
