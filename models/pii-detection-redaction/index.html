PII detection & redaction — scrub personal data locally

Paste a message, a support ticket, or a log line and the model tags every piece of personal data — names, emails, phone numbers, cards, IBANs, crypto wallets, SSNs, passwords, IP addresses and street addresses across 60 categories. Flip the Redact switch to replace each with a [CATEGORY] placeholder and copy a clean version — useful before you share, log, or paste text into another tool. Every byte stays on your device.

See inside — the per-token BIO tags

The model doesn't see "jordan.mitchell@gmail.com" as one thing — it tags each sub-word token independently: B-EMAIL (begin email), I-EMAIL (inside email), and so on. We stitch consecutive B-/I- runs into whole values, map them back to the exact characters in your text, and trim stray punctuation so a redaction masks the value — never half of it. Why it matters

Detecting personal data on the device means you can scrub it before it ever leaves — no server sees the raw text. That is exactly the property you want for redaction: the sensitive value never has to be transmitted to be found.

  • Redact before you share — mask names, emails, cards and addresses in a document or screenshot caption client-side.
  • Clean prompts for LLMs — strip PII out of text before pasting it into a chatbot or sending it to an API.
  • Scrub logs & tickets — remove emails, IPs and account numbers from support transcripts before they're stored or indexed.
  • Pre-flight compliance — flag where personal data appears so a human can review it, without shipping the data off-box.

It's a best-effort model, not a guarantee: categories can be approximate (a card number may read as ACCOUNTNUMBER, an IBAN as MASKEDNUMBER), several numbers packed right next to each other can get split or mislabelled, and no detector catches everything — treat it as a strong first pass, then have a human confirm before relying on the redaction.

How the API works

Token classification is one pipeline call:

import { pipeline } from "@huggingface/transformers";

const pii = await pipeline("token-classification", "phatvo/deberta_finetuned_pii-ONNX");
const out = await pii("email me at sam@acme.io");
// → [{ entity: "B-EMAIL", word: "▁sam", score: 0.99, index: 4 },
//    { entity: "I-EMAIL", word: "@",   score: 0.99, index: 5 }, … ]

The pipeline returns one BIO-tagged row per sub-word token, but no character offsets. We map each token back to its span in the original text (a cursor-walk), merge B-/I- runs into whole values, and trim wrapping punctuation. It all runs in a Web Worker so typing stays smooth.
