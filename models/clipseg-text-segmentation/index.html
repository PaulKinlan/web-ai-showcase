CLIPSeg — segment by describing it

Most segmenters know a fixed list of classes. CLIPSeg doesn't: you type a phrase in plain language — the dog, shadows, the road, the warmest area — and it returns a per-pixel score map for exactly that phrase. It's open-vocabulary, text-prompted segmentation: point at things with words, not with a class menu or a mouse click. All on your device.

Why it matters

CLIPSeg turns "select the thing I mean" into a text box. There's no training a new segmentation head per class, no fixed taxonomy, and no click-to-prompt — just words. Because it runs in the browser, you can localize anything a user can describe without a byte of their image leaving the tab.

  • Describe-to-select editing — "mask the sky", "select the shadows" as a photo-tool selection, computed locally.
  • Long-tail / bespoke classes — segment "the forklift", "the cracked tile", "the barcode" with zero per-class training.
  • Content-aware effects — blur / recolour / remove exactly the region a phrase names, e.g. privacy blur "faces" or "screens".
  • Prompted pre-processing — crop to the phrase and hand the region to another model (classify, OCR, caption) — see the multi-model demo.

Use cases

Basics

Type a phrase, get a mask. Compare several phrases side by side on one photo.

Practical

Isolate an object on a transparent background, or privacy-blur exactly what a phrase names.

Wild

Segment abstract concepts — "the warmest area", "the safest place to walk", "where to look first".

Multi-model

CLIPSeg localizes a phrase → crop the region → a ViT classifier names what's actually there.

How the API works

CLIPSeg is a model class, not a one-call pipeline — because it takes both an image and text. For this ONNX repo the processor bundles an image processor only, so we load the text tokenizer separately and batch the phrases against one image:

import {
  AutoTokenizer, AutoProcessor, CLIPSegForImageSegmentation, RawImage,
} from "@huggingface/transformers";

const tokenizer = await AutoTokenizer.from_pretrained("Xenova/clipseg-rd64-refined");
const processor = await AutoProcessor.from_pretrained("Xenova/clipseg-rd64-refined");
const model     = await CLIPSegForImageSegmentation.from_pretrained(
  "Xenova/clipseg-rd64-refined", { dtype: "q8" });

const image  = (await RawImage.read(url)).rgb();
const texts  = ["a cat", "the couch"];
const text   = tokenizer(texts, { padding: true, truncation: true });
const pixels = (await processor(image)).pixel_values;      // [1, 3, 352, 352]

// tile pixels to [texts.length, 3, 352, 352] so every phrase shares the image
const { logits } = await model({
  input_ids: text.input_ids, attention_mask: text.attention_mask, pixel_values,
});
// logits: [texts.length, 352, 352] — a RAW per-pixel score map per phrase.
// mask = sigmoid(logits) > threshold, upscaled to the image.

The score maps are 352×352 (the model's working resolution); we upscale to your photo for display. Everything runs in a Web Worker so the sliders stay smooth while a forward pass is in flight.
