zero-shot-image-classification · vision + language · WASM · ~874 MB

Match an image to labels in any language

Zero-shot image classification isn't just an English trick. Jina-CLIP v2 learned one shared space for pictures and 89 languages — so a cat, un gato, 一只猫, and قطة all land near the same picture of a cat. Type labels in whatever language you like and watch them score. Nothing leaves your device.

Why not AltCLIP? AltCLIP was the obvious multilingual pick, but Transformers.js ships no AltCLIP model class (no Xenova/altclip ONNX either), so it can't run in the browser today. This demo uses jina-clip-v2 instead — a genuinely multilingual CLIP that does run — for the same capability.

See inside

Jina-CLIP turns your image into one 1024-number vector and each label into its own 1024-number vector, all L2-normalised. The score is the cosine similarity (a dot product of the normalised vectors) between the image and each label. Because the text tower is multilingual, translations of the same concept sit close together in that space — so they score alike against the same image. Below are the raw numbers.

Why it matters

Most vision-language models are English-first. A multilingual one means your labels, prompts, and tags can be written by the people who actually use your product — in their language — with no translation step and no server.

  • Global moderation & tagging — score images against policy labels written in the market's own language, client-side.
  • Cross-lingual search — a French query and an image captioned in Chinese meet in the same embedding space.
  • Localised UX for free — one model powers the same feature across dozens of languages; you don't ship a model per locale.
  • Private & offline — the image and the labels never leave the tab.

How the API works

jina-clip-v2 returns embeddings, not logits_per_image, so you take the cosine yourself:

import { AutoModel, AutoProcessor, RawImage } from "@huggingface/transformers";

const model = await AutoModel.from_pretrained("jinaai/jina-clip-v2", { dtype: "q8" });
const processor = await AutoProcessor.from_pretrained("jinaai/jina-clip-v2");

const img = await model({ pixel_values: (await processor.image_processor(image)).pixel_values });
const txt = await model(processor.tokenizer(labels, { padding: true }));

// Both are L2-normalised → cosine is just a dot product.
const cos = dot(img.l2norm_image_embeddings, txt.l2norm_text_embeddings[i]);

This page runs exactly that in a Web Worker so the UI never janks, and keeps the cosine numbers for "See inside".
