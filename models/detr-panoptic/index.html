DETR panoptic — cut a whole scene into labelled pieces

Panoptic segmentation answers "what is every pixel?" in one pass — and it counts. Two cats become two separate segments (that's the "things" half), while the couch, the wall and the floor come back as labelled background regions (the "stuff" half). Drop an image and watch the scene fall apart into colour-coded, named pieces, entirely on your device.

See inside

Panoptic segmentation merges two jobs. Things are countable objects — each instance gets its own mask, so two cats are two rows, not one "cat" class. Stuff is amorphous background — sky, grass, a wall — one region per class. Every pixel is assigned to exactly one segment (we resolve overlaps by the strongest mask), so the areas below are a true partition of the image. This table is the raw parse the model returned.

Why it matters

Boxes tell you where a thing roughly is; a semantic map tells you what kind of thing each pixel is. Panoptic gives you both at once — pixel-exact masks that still count instances — which is exactly what "understand this whole scene" needs. Doing it in the browser turns scene parsing into a local function call: private, offline, free per call.

  • Instance-accurate editing — pull one specific object out of a photo (this cat, not that one) with a pixel mask, no lasso tool.
  • Scene inventory & layout — count distinct objects and measure how much of the frame is sky vs building vs road, on-device.
  • Privacy redaction by region — blur every person instance while leaving the background untouched, before an image is ever shared.
  • Compositing & AR — know the exact silhouette of each object to place graphics behind or between them.

Use cases

Basics

One image, colour-coded segments, a things-vs-stuff split — feel panoptic parsing first-hand.

How the API works

Panoptic parsing is a one-liner with Transformers.js — the default call on this model returns the segment set:

import { pipeline } from "@huggingface/transformers";

const segment = await pipeline(
  "image-segmentation",
  "Xenova/detr-resnet-50-panoptic",
); // downloads + caches ~45 MB once

const segments = await segment(imageURL);
// → [{ score: 0.99, label: "cat",  mask: RawImage }, …]  ← per-instance + per-region
//   each mask is a single-channel image; a pixel value near 255 means "in this segment"

This page runs that call once inside a Web Worker (so the control UI never janks), then composites the per-segment masks into one colour-coded overlay by assigning each pixel to its strongest mask. The opacity slider just re-blends the cached overlay — it never re-runs inference. The canvas carries an aria-label and every segment is also listed as accessible text, because canvas pixels aren't exposed to assistive tech on their own.
