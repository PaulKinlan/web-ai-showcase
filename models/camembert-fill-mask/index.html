CamemBERT — predict the missing French word

CamemBERT is a RoBERTa trained on 138 GB of French text and nothing else. Its one job here is masked language modelling: hide a word behind <mask> and it ranks what belongs there — with a French-native vocabulary, French idioms and French grammar baked in. Type a sentence with a mask and see its guesses, plus the exact logits it computed. Everything runs on your device.

At a glance

ModelXenova/camembert-base (ONNX build of almanach/camembert-base)
Task / pipelinefill-mask — masked language modelling, loaded via AutoModelForMaskedLM for raw logits
OutputA probability over the whole 32,005-token French vocabulary at each <mask>
ArchitectureCamemBERT / RoBERTa-base (12 layers, ~110M params), SentencePiece tokenizer trained on French
LanguageFrench only — mask token is <mask> (RoBERTa style), not [MASK]
Quantizationq8 (8-bit) ONNX
BackendWebAssembly (runs anywhere; no WebGPU needed)
Download~110 MB, cached after first load
LicenseMIT
Web APIsWebAssembly, Web Workers — all Baseline widely available

See inside — logits, not just a guess

A language model doesn't output words — it outputs a logit (a raw, unbounded score) for every one of its 32,005 French vocabulary tokens at the masked position. We softmax those into probabilities. The bars above show the probability; the logit beside each is the actual number the network produced. Watch how a confident prediction ("capitale") sits far above the pack in logit space, while an ambiguous slot spreads the mass out. The – below is the entropy of the top-k — low means CamemBERT is sure, high means it's hedging.

Why it matters

A model trained only on French reads French the way a multilingual model can't afford to — every token in its vocabulary is a French subword, so accents, elisions (l', d'), gender agreement and idioms come out cleaner. Because it scores every word in context, you can ask it what fits, how strongly, and compare alternatives — all locally, with no prompt engineering and no API bill.

  • French autocomplete — suggest the next or missing word in a compose box, offline, with French-native suggestions.
  • Cloze & language drills — generate fill-in-the-blank French exercises with graded difficulty from the entropy.
  • Style & register checks — compare candidate words at a slot (formal vs familiar) using the logits.
  • Grammar-aware transforms — chain a fill with a French POS tagger to build Mad-Libs or agreement checkers.

How the API works

The one-liner uses the fill-mask pipeline:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "Xenova/camembert-base");
const out = await unmask("Paris est la <mask> de la France.");
// → [{ token_str: "capitale", score: 0.71, sequence: "…" }, { token_str: "ville", … }]

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 32005]), slice the row at the <mask> position, and softmax it ourselves. That's the only way to show the real logit values, not just the pipeline's rounded scores. All of it runs in a Web Worker so the page never janks.
