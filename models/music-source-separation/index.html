Music source separation — split a song into stems

Upload a song and Demucs pulls it apart into four stems — vocals, drums, bass and everything else — right on your device. Mute the vocals for instant karaoke, solo them for an acapella, mix and match, and download each stem. The audio never leaves the tab.

Why it matters

Source separation is the magic behind karaoke tracks, remixes and sample-flipping. Doing it in the browser means you can pull the vocals out of a track — or isolate the drums — without uploading your music to anyone's server.

  • Instant karaoke — mute the vocals stem and sing along to the backing.
  • Acapellas & remixing — solo the vocals, or grab just the drum stem for a sample.
  • Practice & transcription — isolate the bass or the "other" (keys/guitar) to learn a part.
  • Music education & accessibility — hear one instrument at a time from any recording.

It separates the first ~7.8-second segment per run (Demucs works on fixed-length windows; a full song would be many segments). The vocals stem is trained on sung vocals, so in the built-in sample the spoken voice lands mostly in "other" — upload a song with singing to hear the vocals isolated. Quality on WASM matches the model; it's just slower than on a GPU.

How the API works

htdemucs takes a stereo waveform segment and returns four stems — the STFT lives inside the graph:

// onnxruntime-web, fp32
const input = new ort.Tensor("float32", stereo, [1, 2, 343980]); // 7.8 s @ 44.1 kHz
const { output } = await session.run({ input });
// output: [1, 4, 2, 343980] → [drums, bass, other, vocals]

We feed raw waveform and get waveform stems back (no manual spectrograms), decode/resample the file to 44.1 kHz stereo in the browser, and run the model in a Web Worker so the page stays responsive during the ~10 s separation.
