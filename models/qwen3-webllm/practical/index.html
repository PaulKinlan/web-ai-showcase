Practical: reasoned extraction to strict JSON

The workhorse task: turn unstructured text into a structured object your code can use. Qwen3 reads a messy sentence and returns JSON matching a schema. With thinking on it reasons about the ambiguous fields first (dates, currencies, implied values) and tends to get them right; then it emits only JSON — which this page runs JSON.parse() on, live, to prove it's machine-readable. All on your GPU.

Why reasoning helps here

Extraction looks easy until a field is implied rather than stated: "next Friday" is a date the model has to compute, "table for 4" is a party size buried in prose, a name is attached to a booking not a person. In non-thinking mode a small model guesses; with thinking on it works the ambiguity out in the trace before committing. Either way, the contract is the same — the answer is strict JSON, and this page enforces it with a real JSON.parse(), so you can see when the model breaks the format and needs a retry.
