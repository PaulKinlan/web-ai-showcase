CodeBERTa — predict the missing code token

BERT learned English by filling in masked words. CodeBERTa — Hugging Face's CodeBERT-family model — learned the same trick on source code: it was pre-trained on millions of functions across six languages (Go, Java, JavaScript, PHP, Python, Ruby). Put a <mask> where a token is missing — a function name, a keyword, an operator — and it ranks what belongs there, with the exact logits it computed. Everything runs on your device.

See inside — how code tokenizes, and the logits

Two things make a code language model different. First, the tokenizer: it's a byte-level BPE trained on code, so self, (), and __init__ are common single tokens, and a leading space is its own marker (shown as ␣). Second, the logits: at the mask, the model produces a raw score for every one of its 52,000 tokens; we softmax those into the probabilities above. The – below is the top-k entropy — low means CodeBERTa is sure, high means it's hedging between several plausible tokens.

Why it matters

A model that scores every token in code context — locally, with no prompt and no API bill — is a surprisingly practical building block. It reads code bidirectionally (both sides of the mask), which is exactly what you want for filling a gap rather than only continuing from the left.

  • In-editor token suggestions — propose a likely identifier, keyword, or API name at the cursor, offline.
  • API-name discovery — mask the method in arr.<mask>() and see the calls the model associates with the receiver.
  • Naming help — mask a function or variable name and rank sensible names given the body.
  • Anomaly / typo signals — a real token that scores far below the model's top guesses is a cheap "this looks off" hint.

How the API works

The one-liner uses the fill-mask pipeline:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "onnx-community/CodeBERTa-small-v1-ONNX");
const out = await unmask("def <mask>(numbers):\n    return sum(numbers) / len(numbers)");
// → [{ token_str: " average", score: 0.09, … }, { token_str: " mean", … }]

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 52000]), slice the row at the <mask> position, and softmax it ourselves. That's the only way to show the real logit values, not just the pipeline's rounded scores. All of it runs in a Web Worker so the page never janks.
