TrOCR — read the text out of a picture

TrOCR is a transformer that does OCR the modern way: no hand-built pipeline, just a Vision Transformer that looks at a cropped line of text and a text decoder that writes it out, one token at a time — GRAND TOTAL: $42.99 — entirely on your device.

See inside — the decode, token by token

OCR here is autoregressive generation, not template matching. The encoder reads the cropped line into patch embeddings once; then the decoder emits the transcription one token at a time, each conditioned on the image and everything decoded so far — which is why TrOCR can fix its own spelling from context. The trace below shows every token as it arrived, with the elapsed time. The first token is the slow one (it includes the image encode); the rest come faster.

Why it matters

Text trapped in an image is invisible to everything that isn't a person looking at it — you can't search it, copy it, translate it, or total it. OCR turns those pixels back into characters, and doing it locally makes it free, private, and offline-capable.

  • Digitise paper — pull totals and line items off receipts and invoices client-side, no OCR API and no upload.
  • Capture the physical world — read a sign, a serial number, a meter, or a label straight from the camera.
  • Accessibility — voice or magnify text that only exists as an image for low-vision users.
  • Feed a pipeline — OCR is the front door to translation, extraction, and search; the text it produces is just more text to work with.

Use cases

Basics

Crop a line — a sign or a receipt total — and read it, token by token.

Practical

Digitise a whole receipt: read every line into an editable, copyable list.

Wild

Live webcam OCR — point your camera at text and read it on the fly.

Multi-model

OCR → translation: read foreign text, then translate it locally.

How the API works

The base call is two lines with Transformers.js:

import { pipeline } from "@huggingface/transformers";

const reader = await pipeline(
  "image-to-text",
  "Xenova/trocr-small-printed",
); // downloads + caches ~185 MB once

const out = await reader(lineCropURL, { max_new_tokens: 48 });
// → [{ generated_text: "GRAND TOTAL: $42.99" }]

The important bit is the input: TrOCR expects one line of text, so this page crops the region you box before handing it over. For "See inside", it passes a TextStreamer into the same call — the streamer fires once per decoded token, so the page renders the transcription as it forms and records each token's arrival time, all off the main thread in a Web Worker. Note: this SROIE-tuned checkpoint tends to normalise to UPPERCASE — that's the model, not a bug.
