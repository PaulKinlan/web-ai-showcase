Basics: similarity as one number, and the prefix effect

An embedding turns text into a point in 768-dimensional space; two texts that mean similar things land close together. The first demo reads that closeness as a single cosine number. The second shows the thing that makes Nomic different from an ordinary encoder: the task prefix literally moves the sentence to a different point.

2 · The prefix effect — same sentence, different point

Nomic bakes the task into the text as a prefix, so one sentence has a different embedding under each prefix. Type a sentence and see how much the vector moves between prefixes — measured as the cosine similarity between the prefixed versions (1.00 would mean "no change").

Off-diagonal values below 1.00 prove the prefix shifts the embedding. search_query: and search_document: are pulled furthest apart — that gap is exactly what powers asymmetric query-to-passage retrieval.
