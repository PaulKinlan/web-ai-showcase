nomic-embed-text-v1 — embeddings with a task prefix

A fully-open, long-context embedder (8192 tokens) that matched OpenAI's text-embedding-ada-002 on release. Its defining quirk: you tell it the task by prepending a prefix to the text — search_query: for what you're searching with, search_document: for what you're searching over. Get the prefix right and retrieval sharpens; get it wrong and it dulls. Everything runs locally; nothing leaves your device.

Why it matters

Embeddings power search, recommendations, dedup, clustering, and RAG. Nomic is a fully open alternative to closed embedding APIs — open weights, open data, an 8192-token context that swallows whole documents — and it still fits in a browser tab at ~140 MB. So you can ship meaning-based search with no vector-DB service, no per-call API bill, and no user text leaving the device.

  • Long-document search — its 8192-token window embeds a full article or transcript without the aggressive chunking a 512-token model forces.
  • Asymmetric retrieval done right — the search_query: / search_document: split is built for short-query → long-passage search.
  • Dedup & clustering — switch to the clustering: prefix to collapse near-duplicate notes, tickets, or reviews with no labels.
  • Private, offline RAG — retrieve the right snippet to feed a local LLM, entirely on-device.

How the API works

Feature-extraction with mean pooling — plus the all-important task prefixes:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Xenova/nomic-embed-text-v1",
); // downloads + caches ~140 MB once

// Nomic uses MEAN pooling (not CLS), then L2-normalize:
const embed = (texts) => extract(texts, { pooling: "mean", normalize: true });

// Prepend the TASK PREFIX — this is the part people miss:
const q  = await embed(["search_query: how to fix yellow leaves"]);
const ps = await embed([
  "search_document: Yellow leaves often mean overwatering; let soil dry out.",
  "search_document: The best time to visit Kyoto is autumn.",
]);

// unit vectors → cosine similarity is just the dot product:
const score = (a, b) => a.reduce((s, x, i) => s + x * b[i], 0);
const q0 = q.tolist()[0], docs = ps.tolist();
docs.map((d) => score(q0, d)); // → [0.62, 0.19] — the plant line wins

This page asks the worker for the un-normalized mean-pooled vectors so "See inside" can show the real pre-normalization magnitude, then normalizes in JS. Inference runs in a Web Worker so ranking never janks the page. Use search_query:/search_document: for retrieval; classification: or clustering: for symmetric jobs.
