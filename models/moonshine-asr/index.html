Moonshine — fast, low-latency speech-to-text

Moonshine listens to audio and writes down what was said — like Whisper, but built for speed at the edge. It doesn't pad every clip to 30 seconds, so a short utterance transcribes in a fraction of its own length. Record from the mic, drop a file, or use a sample, and watch the real-time factor.

Why it matters

On-device transcription used to force a trade-off: Whisper is accurate but pads and chunks, which adds latency on short inputs. Moonshine is tuned for the interactive case — voice commands, live dictation, short replies — where a snappy response matters more than transcribing an hour of audio. It's a free, private, offline capability of your page.

  • Fast voice commands — a hands-free control layer that feels instant, with no per-request inference cost.
  • Live dictation — transcribe short utterances as the user speaks, segment by segment, entirely client-side.
  • Real-time captions — caption a mic or media stream with low latency, nothing uploaded.
  • Edge & offline — a small model that runs where bandwidth or privacy rules out a cloud API.

How the API works

Transcription is one pipeline call — Moonshine uses the standard ASR task:

import { pipeline } from "@huggingface/transformers";

const transcribe = await pipeline(
  "automatic-speech-recognition",
  "onnx-community/moonshine-base-ONNX",
  { device: "wasm", dtype: "q8" }, // WebGPU when a real adapter exists
);

// audio = a 16 kHz mono Float32Array (decode with Web Audio, then resample)
const out = await transcribe(audio);
// out.text → the transcript
// Moonshine has no timestamp tokens — it's built for fast short-form transcription.

This page decodes the mic/file/sample to 16 kHz mono with the Web Audio API on the main thread, then transfers just the Float32 samples to a Web Worker where the pipeline runs — so the model never blocks the interface. It times the call and divides by the clip length to show a real real-time factor.
