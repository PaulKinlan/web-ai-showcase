OuteTTS — a language model that speaks

Most text-to-speech models are custom acoustic networks. OuteTTS takes a different route: it is a plain Qwen2 language model that was taught to predict audio tokens instead of words. Feed it text (and optionally a speaker profile) and it autoregressively generates a stream of discrete codec tokens — then a second model, the WavTokenizer neural codec, decodes that stream into an actual 24 kHz waveform. It is TTS reframed as next-token prediction, and it runs end-to-end in your browser. Nothing is uploaded.

Why it matters

Framing speech as language modelling is what makes modern TTS controllable and composable. Because the generator is an LLM, the same sampling knobs you know from text — temperature, repetition penalty — shape the voice, and a speaker profile conditions it like a system prompt. And it all runs locally.

  • Private narration & read-aloud — turn articles, docs, or messages into audio without shipping text to a TTS API.
  • Voice for offline / on-device apps — games, kiosks, and accessibility tools that speak with no network and no per-character billing.
  • Expressive control — dial temperature for a flatter or livelier delivery; swap speaker profiles for different characters.
  • Composable pipelines — pair with a summariser or chat LLM so a model can write and speak (see the multi-model demo).

How the API works

The outetts library orchestrates the LLM and the codec; you drive it like any TTS:

import { HFModelConfig_v1, InterfaceHF } from "outetts";

const cfg = new HFModelConfig_v1({
  model_path: "onnx-community/OuteTTS-0.2-500M",
  language: "en",
  dtype: "q8",         // 8-bit LLM + codec
  device: "webgpu",    // or "wasm"
});
const tts = await InterfaceHF({ model_version: "0.2", cfg });

const speaker = tts.load_default_speaker("male_1");
const output = await tts.generate({
  text: "Speech synthesis is the artificial production of human speech.",
  temperature: 0.1,          // LLM sampling — lower = steadier delivery
  repetition_penalty: 1.1,
  speaker,
});
// output.audio is a waveform Tensor; output.sr is 24000 Hz.

This page runs all of that in a Web Worker. The LLM tries WebGPU first and falls back to WASM honestly; the WavTokenizer decoder always runs on WASM (WebGPU isn't supported for it yet). We build a 16-bit PCM WAV from output.audio for playback and download.
