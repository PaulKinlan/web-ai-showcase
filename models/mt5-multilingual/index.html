mT5-small — the raw multilingual T5, one blank at a time

mT5 is the multilingual T5: the exact encoder-decoder design of T5, but pretrained on the mC4 corpus across 101 languages. There is one deliberate honesty here — this checkpoint was pretrained on only the span-corruption (denoising) objective and, unlike FLAN-T5 (instruction-tuned) or the English T5 (task-prefixes), it was never fine-tuned on a downstream task. So the one thing it truly knows is to reconstruct masked spans: put a <extra_id_0> blank in a sentence and it fills it — in whatever language you wrote. Try it in several languages and watch the raw multilingual foundation at work. Nothing leaves your device.

This is the small (300M) checkpoint at q8, so calibrate expectations: the fills are often rough or generic — a raw, tiny, quantized model is a foundation, the thing you would fine-tune (the popular mT5 fine-tunes like multilingual summarization aren't ONNX-exported for the browser, so we honestly show the base). What is remarkable is that a single 250k-token multilingual vocabulary lets one model take a blank in Spanish, German, Chinese or English and answer in the same language. The page shows you the real output, right or rough — never a canned answer.

Why it matters

Almost every multilingual NLP product you have used starts from a model like this: a single encoder-decoder pretrained across 101 languages on one objective, then fine-tuned for the job. Seeing the raw checkpoint — before any fine-tuning — makes the recipe legible: one shared vocabulary, one denoising objective, and the ability to operate in a language it was never explicitly told to use. At ~430 MB it runs in a tab, so you can inspect the multilingual foundation on-device with nothing sent anywhere.

  • The multilingual foundation — the base you fine-tune for translation, summarization, or QA in many languages from one model.
  • Language-agnostic cloze — mask a word and get an in-language suggestion, whatever language the sentence is in.
  • Teaching the objective — a legible, on-device demo of span corruption, the pretraining task behind T5-family models.
  • Honest baselines — feel exactly how much a raw multilingual model can and cannot do before task fine-tuning.

How the API works

The whole task is expressed by where you place the sentinel blanks — no template, no roles:

import { pipeline } from "@huggingface/transformers";

const mt5 = await pipeline(
  "text2text-generation",
  "Xenova/mt5-small",
); // downloads + caches ~430 MB once

// mask a span with a sentinel; the target is the sentinels followed by the missing text:
await mt5("Thank you <extra_id_0> me to your party <extra_id_1> week.");
// → [{ generated_text: "<extra_id_0> for inviting <extra_id_1> last <extra_id_2>" }]

// the SAME model, a Spanish blank — it answers in Spanish, from the shared multilingual vocab:
await mt5("La capital de Francia es <extra_id_0>.");
// → [{ generated_text: "<extra_id_0> ..." }]   ← raw + small, so expect rough fills

This page keeps <extra_id_n> sentinels visible (skip_special_tokens: false) so the output format is legible, then reconstructs the sentence by slotting each fill back into its blank. It passes a TextStreamer to timestamp every generated token — all inside a Web Worker so the UI never janks.
