DETR — find and box every object in an image

DETR treats detection as a set-prediction problem: one forward pass, a fixed set of "object queries," and out come labelled boxes with confidence scores. Drop an image and watch it draw a box around every thing it recognises — 80 everyday COCO classes, entirely on your device.

Why it matters

Object detection used to mean a cloud API, an upload, and a per-image bill. DETR in the browser turns "where is everything in this picture?" into a local function call — private, offline, free per call, and fast enough to run on a live camera.

  • Counting & inventory — count people in a queue, cars in a lot, or items on a shelf from a photo, with nothing uploaded.
  • Privacy redaction — find every person or screen in an image and blur it client-side before it's ever shared.
  • Smart cropping & alt-text — know what's in a picture to frame it well or describe it, without a vision API key.
  • Camera-aware UX — react to what the webcam sees (is a person present? a phone? a cup?) entirely on-device.

How the API works

Detection is a one-liner with Transformers.js. The boxes come back in pixel coordinates:

import { pipeline } from "@huggingface/transformers";

const detect = await pipeline(
  "object-detection",
  "Xenova/detr-resnet-50",
); // downloads + caches ~160 MB once

const output = await detect(imageURL, { threshold: 0.5, percentage: false });
// → [{ label: "cat", score: 0.998, box: { xmin, ymin, xmax, ymax } }, …]
//   percentage:false gives pixel coordinates; true gives 0–1 fractions

This page runs the model once at a low floor threshold (0.05) inside a Web Worker, caches every box, and filters the cached list as you drag the slider — so the slider is instant and never re-runs inference. The canvas is drawn at the image's natural resolution and scaled by CSS, so the boxes stay pixel-accurate.
