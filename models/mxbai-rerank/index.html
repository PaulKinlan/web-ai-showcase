mxbai-rerank — rank passages with a DeBERTa-v2 cross-encoder

This is mixedbread's reranker. Like every cross-encoder it reads a query and a passage together and outputs one relevance score — but its backbone is DeBERTa-v2 with disentangled attention, a different architecture from the BERT-family and XLM-RoBERTa rerankers already in this showcase. It's English-focused and tiny (~90 MB at q8). Give it a query and some candidates, watch it re-sort them, then look inside at the raw logit per pair and how decisively the top answer wins. Nothing leaves your device.

Run it

Type a query and one candidate passage per line. The model scores each passage against the query and re-sorts them by relevance logit (higher = more relevant). The default set is full of near-misses — passages that share words with the query but don't actually answer it.

Why it matters

Rerankers are the precision stage of modern search. A fast retriever (a keyword index or embeddings) casts a wide net and returns, say, the top 50 candidates; a cross-encoder then reads each candidate against the query and reorders them with far higher accuracy. mxbai-rerank is a strong, compact English reranker with a DeBERTa-v2 backbone — a different bet from the BERT/RoBERTa rerankers, and small enough (~90 MB) to run in a tab. The precise, expensive step happens on-device with nothing sent anywhere.

  • Better RAG — retrieve broadly with embeddings, then rerank so the LLM reads the genuinely best chunk, not just the most similar-looking one.
  • Answer selection — pick the single best reply from candidate answers, and use the decision margin to know when to escalate to a human.
  • Search precision — reorder a keyword shortlist so the top result actually answers the intent behind the words.
  • De-duping intent — score which FAQ or support article truly matches a user's phrasing across your knowledge base.

Use cases

Basics

Rank a handful of passages against a query by cross-encoder relevance.

Practical

Two-stage RAG: retrieve a shortlist, then rerank it for precision.

Wild

Adversarial hard negatives: distractors built to fool keyword and embedding search.

Multi-model

GTE embeddings retrieve, mxbai-rerank reorders — the classic retrieve-then-rerank.

Compare with the other cross-encoders: the multilingual BGE-reranker (XLM-RoBERTa), the MS MARCO MiniLM (BERT), and the Jina reranker (JinaBERT) — same idea, four different backbones.

How the API works

A cross-encoder scores query–passage pairs; we read the raw logit off the model directly:

import { AutoTokenizer, AutoModelForSequenceClassification } from "@huggingface/transformers";

const model_id = "mixedbread-ai/mxbai-rerank-xsmall-v1";
const tokenizer = await AutoTokenizer.from_pretrained(model_id);
const model = await AutoModelForSequenceClassification.from_pretrained(model_id, { dtype: "q8" });

// Encode the query with each passage AS A PAIR — [CLS] query [SEP] passage [SEP]:
const inputs = tokenizer(
  passages.map(() => query),
  { text_pair: passages, padding: true, truncation: true },
);
const { logits } = await model(inputs); // one relevance logit per pair — higher = more relevant
// sort the passages by logits.data — that's your reranking

This page loads the model through the shared loader as a text-classification pipeline, then reuses pipe.model + pipe.tokenizer to read the raw logits (the pipeline would squash them through a sigmoid). All inference runs in a Web Worker so ranking never janks the page.
