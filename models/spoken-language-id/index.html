Spoken language identification — name the language you hear

Give it speech and it tells you the language — one of 126 — not the words. Meta's MMS-LID listens to the raw waveform on a billion- parameter Wav2Vec2 backbone and returns a language with a confidence score. It's the audio-first cousin of the text language detector: that one reads, this one listens — the router step before any speech pipeline.

See inside

The model outputs one score per language — a softmax over all 126. Two numbers describe the shape of that distribution: the top-1 margin (how far the winner leads the runner-up) and the normalised entropy (0 = certain, 1 = torn). A few seconds of clear speech is peaky and low-entropy; a very short, noisy, or code-switched clip spreads the mass across related languages — watch the runners-up, they're usually neighbours in the same family (Spanish ↔ Galician, Dutch ↔ Afrikaans).

Why it matters

Almost every speech feature has a hidden first question: what language is this? Pick the wrong transcriber or translator and everything downstream is garbage. Answering that on-device, before any upload, makes multilingual audio pipelines both cheaper and more private.

  • Route to the right ASR — detect the language, then load the matching Whisper / speech model instead of guessing.
  • Auto-caption & translate — send audio to the correct translation path without asking the user to set a language.
  • Call-centre & support triage — route a caller to an agent or bot in their language from the first few seconds.
  • Media indexing — tag a library of recordings by language so people can filter without listening.

Use cases

Basics

Identify a sample clip's language and read the top guesses — feel what the model does.

Practical

A language router: detect first, then show which ASR / translator you'd hand the audio to.

Wild

Short & code-switched clips: trim the audio and watch confidence and the runners-up shift.

Multi-model

Detect the language, then transcribe it with the matching Whisper model — a two-model pipeline.

How the API works

Identification is one call with Transformers.js. Feed a 16 kHz mono Float32Array:

import { pipeline } from "@huggingface/transformers";

const identify = await pipeline(
  "audio-classification",
  "Xenova/mms-lid-126",
  { dtype: "q8" },              // large one-time download (~974 MB), then cached
);

const output = await identify(audio, { top_k: 5 });
// audio = Float32Array @ 16 kHz mono (decode with the Web Audio API)
// → [{ label: "eng", score: 0.94 }, { label: "haw", score: 0.01 }, … ]
// labels are ISO-639-3 codes; map them to names for display.

This page decodes files / mic recordings to 16 kHz mono with an OfflineAudioContext, runs the pipeline inside a Web Worker so the UI never janks, and requests top_k: 126 so "See inside" can show the whole language distribution — not just the winner.
