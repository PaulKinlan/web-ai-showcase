Multi-model · low-light → text detection · WASM

Brighten, then read the text in the dark

A dark photo hides its own text from a detector. Here IAT brightens the image first, then the on-device DBNet text detector runs on the result — and finds text regions the shadows were smothering. Two models, chained, entirely local.

Run it

Two models load below (IAT ~0.4 MB, detector ~3 MB). Pick a text-heavy sample, then run the pipeline — we detect on the dark image and on the enhanced one so you can see the gain.

1 · IAT (low-light enhancement)
2 · DBNet (text detection)

Dark → detected

Enhanced → detected

enhance –detect –regions –

Both models run on-device; the image, the enhancement, and the boxes never leave the tab.

Why chain them

Low-light enhancement is a force-multiplier for downstream vision: a text detector, OCR model or classifier all see cleaner structure once the shadows are lifted. This is the on-device version of a classic "read the sign in the dark" pipeline — no server, no upload, each stage a real model running in your browser.
