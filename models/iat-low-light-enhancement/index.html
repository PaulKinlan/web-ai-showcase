IAT — brighten a dark photo

Give this model an under-exposed or night photo and it recovers a properly-lit one: it lifts brightness, fixes colour, and pulls detail out of the shadows. IAT — the Illumination Adaptive Transformer (BMVC 2022) — is a tiny ~90K-parameter network that predicts a per-pixel illumination correction, so it's a learned exposure/tone fix, not a blunt brightness slider. Runs entirely on your device via ONNX Runtime Web; the photo never leaves the tab.

See inside

IAT doesn't apply one global brightness — it predicts an illumination gain map: the mul tensor, a per-pixel multiplier telling each region how much to brighten. This is that map. Warm/amber = high gain (the model decided this area was too dark and lifted it hard); dark indigo = low gain (already well-lit, leave it). That spatial variation is what separates a learned exposure fix from a flat “+40% brightness”.

How to read it

Amber traces the darkest parts of the scene — shadows, night sky, under-lit faces — where IAT applied the most gain. A crude brightness slider would raise everything equally and blow out the highlights; IAT’s map is uneven because it corrects the illumination locally, keeping the already-bright regions in check.

Why it matters

Low light is where phone cameras and webcams fall apart. On-device enhancement means you can rescue a dark shot privately, with no upload and no per-image cloud cost — and the model is so small (~0.4 MB) it loads instantly:

  • Photo apps — a one-tap “fix exposure / night boost” that runs client-side on the user’s phone or laptop, no server.
  • Video calls & webcams — brighten an under-lit caller in real time before the frame is sent, entirely locally.
  • Pre-processing for other models — feed a properly-lit frame into detection, OCR, or face landmarks so they don’t choke on shadow.
  • Archives & scanning — batch-recover under-exposed scans in a static web tool with zero backend.

IAT recovers detail that’s there but buried in the shadows; it can’t invent information that pure black erased. Extreme darkness recovers partially — it’s a rescue, not a light source.

Use cases

Basics

Dial a photo dark, then watch IAT recover it — wipe between dark and enhanced and see the brightness jump.

Practical

Rescue an under-exposed photo so faces and detail are visible again, then download the enhanced image.

Wild

Push it: crank the darkness to near-black and find where recovery starts to run out of signal.

Multi-model

Enhance → colourize: brighten a dark scene, then run the colourizer, or feed it to object detection.

How the API works

transformers.js has no IAT class and no low-light task, so we run the ONNX graph directly with ONNX Runtime Web. In goes the dark image, out comes the enhanced one (plus the gain/offset maps):

import * as ort from "onnxruntime-web";

// weights live in an external-data sidecar → pass it to the session
const session = await ort.InferenceSession.create(iatBytes, {
  executionProviders: ["wasm"],
  externalData: [{ path: "iat_lol_v1.onnx.data", data: dataBytes }],
});

// dark image → [1,3,H,W] float32 in [0,1] (no normalisation; H,W dynamic)
const input = new ort.Tensor("float32", toNCHW(dark, W, H), [1, 3, H, W]);
const { enhanced, mul } = await session.run({ input });

// enhanced is [1,3,H,W] in [0,1] → clamp to RGBA. mul is the illumination gain map.
const lit = tensorToImage(enhanced.data, W, H);

Inference and the whole output composite run in a Web Worker so the UI never blocks, and the tiny model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
