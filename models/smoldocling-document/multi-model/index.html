Multi-model: read the page, then summarise it

One model rarely does the whole job. Here SmolDocling converts a page image into a structured document (as Markdown), then a small on-device LLM — Qwen2.5-0.5B — reads that structure and writes a plain-language brief. A picture of a document becomes a summary you can act on, two models chained in your browser, nothing uploaded.

What's happening

Both models run in their own Web Workers (SmolDocling on a v4-pinned worker, the LLM on the shared v3 build), so neither the document VLM nor the autoregressive summariser blocks the page. The only thing crossing between them is a short Markdown string — the structured document. This is the shape of real document understanding on the edge: a layout model plus a language model, composed client-side, working offline once both are cached.
