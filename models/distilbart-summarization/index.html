Why it matters

Summarization turns a wall of text into something you can act on in seconds — and doing it on-device means you can summarize private, sensitive, or offline content without shipping a word to a server. No API key, no per-call bill, no data-handling agreement.

  • Inbox & doc TL;DRs — collapse a long email thread or PDF to its gist before you read.
  • Meeting recaps — turn a transcript into a paragraph or bullet points, locally.
  • Feed digests — summarize every article in a reader so headlines carry real substance.
  • Pipeline stage — pair with speech-to-text to go from a recording straight to a recap.

How the API works

Summarization is one pipeline call; length is controlled with generation parameters:

import { pipeline } from "@huggingface/transformers";

const summarizer = await pipeline(
  "summarization",
  "Xenova/distilbart-cnn-6-6",
); // downloads + caches ~250 MB once

const out = await summarizer(longText, {
  max_new_tokens: 80,
  min_new_tokens: 20,
  num_beams: 1,          // greedy; raise for tighter (slower) output
  no_repeat_ngram_size: 3,
});
// → [{ summary_text: "…a concise abstractive summary…" }]

For the live token stream and the decode timeline, this page passes a TextStreamer so it can timestamp each generated token — all inside a Web Worker so the UI never janks.
