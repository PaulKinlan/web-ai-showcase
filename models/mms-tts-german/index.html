MMS-TTS German — end-to-end German speech, one network

This is the German checkpoint of Meta's MMS-TTS family — a distinct model from the English MMS-TTS demo: its own German character vocabulary and its own German-trained VITS weights, so it speaks German orthography — umlauts ä/ö/ü, the ß, German phonotactics — that the English model cannot. VITS is a conditional VAE with normalizing flows and a stochastic duration predictor: one end-to-end network from text straight to a waveform, no separate vocoder, and the same German sentence comes out with slightly different rhythm every time.

See inside — the waveform, and the stochastic duration predictor

There is no spectrogram-then-vocoder hop here: VITS predicts the waveform itself. Below is the actual 16 kHz waveform (the bar fills as it plays). The interesting part is timing — VITS decides how long each German sound lasts with a stochastic module, so re-running the same text changes the duration. Why it matters

A German voice that runs entirely on-device: no per-character bill, no text leaving the tab, and a tiny ~38 MB footprint you can ship inside a PWA or kiosk. VITS won't out-act a big expressive model, but it's the model you reach for when you need German speech everywhere, offline, privately.

  • Offline German read-aloud — narrate German articles, forms or messages with no service to sign up for and nothing sent anywhere.
  • Accessibility — a built-in German screen-voice for content, working without a network.
  • Tiny, embeddable German voice — ~38 MB makes it practical inside an app or device UI.
  • Natural variation — the stochastic duration predictor keeps repeated German prompts from sounding mechanically identical.

Use cases

Basics

Type a German line, hear it, see its waveform — the core move.

Practical

A German narrator: read a notification, a weather report or an article snippet aloud.

Wild

Stochastic prosody lab — generate the same German text many times and compare the timing.

Multi-model

Type English, translate it to German with MarianMT, then have MMS speak the German.

How the API works

MMS-TTS German is a one-liner with the text-to-speech pipeline — the German checkpoint is picked by its id:

import { pipeline } from "@huggingface/transformers";

const tts = await pipeline("text-to-speech", "Xenova/mms-tts-deu", { dtype: "q8" });

const out = await tts("Ein einziges Netzwerk verwandelt Buchstaben direkt in eine Schallwelle.");
// out.audio         → Float32Array waveform (mono)
// out.sampling_rate → 16000
// Run it again with the same text → a slightly different waveform (stochastic duration).

This page runs the pipeline inside a Web Worker so synthesis never blocks the interface, encodes the returned Float32 samples to a WAV blob for the <audio> element, and draws the same samples as the waveform under "See inside". The English demo is the same call with a different checkpoint (mms-tts-eng).
