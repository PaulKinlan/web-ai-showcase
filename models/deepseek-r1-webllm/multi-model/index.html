Multi-model: R1 plans, Coder executes

A reasoning model and a coding model, chained. Give a programming task; DeepSeek-R1-Distill reasons out an approach — the plan — and then Qwen2.5-Coder takes that plan and writes the actual code. You see the reasoning, the plan it commits to, and the implementation, each from the model best suited to it. Both on your GPU, nothing uploaded.

Why split planning from coding

A reasoning model is strong at deciding what to build and catching edge cases up front; a code-tuned model is strong at writing correct, idiomatic code. Chaining them means the plan carries the thinking and the Coder just executes it — often better than asking one model to do both at once. The plan is a natural checkpoint you could edit before the code is written. Note the combined VRAM: R1-Distill-Llama-8B (~5 GB) plus Qwen2.5-Coder-1.5B (~1.6 GB) both resident — this one wants a larger GPU.
