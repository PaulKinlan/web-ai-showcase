Bio_ClinicalBERT — the missing word, with a medical education

Same architecture as BERT, a very different reader. Bio_ClinicalBERT is BERT continued-pretrained on roughly two million MIMIC-III clinical notes, so its masked language modelling is steeped in medicine: hide a word behind [MASK] in a clinical or biomedical sentence and it ranks what a clinician — not the open web — would expect. Type a masked sentence, see its guesses and the exact logits, then race it against general BERT to watch the domain gap appear. All on your device.

Why it matters

A general model averages the whole internet; a domain model has read the notes. On clinical and biomedical text that difference is the difference between a plausible guess and the right one — and it comes for free, in the same 110M-parameter, browser-runnable package, with no prompt engineering and no API bill.

  • Clinical autocomplete & templating — suggest the expected term in a notes editor, offline and private.
  • Terminology & abbreviation sense — probe what the model expects for a masked clinical term to disambiguate context.
  • Domain-vs-general auditing — quantify how much a specialised model actually shifts predictions on your text.
  • A feature extractor for downstream tasks — the same encoder feeds clinical NER, classification, and retrieval.

How the API works

The one-liner uses the fill-mask pipeline — only the model id is domain-specific:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "onnx-community/Bio_ClinicalBERT-ONNX", { dtype: "q8" });
const out = await unmask("The patient was admitted with acute [MASK] failure.");
// → [{ token_str: "respiratory", score: 0.95, sequence: "…" }, { token_str: "heart", … }]

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 28996]), slice the row at the [MASK] position, and softmax it ourselves. The comparison loads general bert-base-cased the same way, in the same Web Worker, so both are exercised identically and the page never janks.
