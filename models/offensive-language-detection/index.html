hate_speech_en — rate how offensive a comment is, privately

A BERT fine-tuned by the IMSyPP project on 100k+ labelled English comments. It reads any text and picks a spot on a four-step severity ladder — acceptable → inappropriate → offensive → violent — as a single softmax distribution (the four add up to 100%). This is a broader signal than hate or toxicity: it catches profanity, insults, and general hostility, not just protected-group attacks. It is an imperfect moderation signal, not a verdict. Every score is computed on your device; no comment is ever sent to a server.

Run it

Type or paste a comment; it re-scores automatically a moment after you stop typing. The offensive-or-worse threshold decides the verdict: if the combined offensive + violent probability lands at or above it, the comment is flagged for review — otherwise it's allowed. The samples run from friendly to hostile so you can watch the distribution slide up the ladder.

See inside — one softmax over a severity ladder

toxic-bert emits six independent sigmoids that need not add up. This model is the other shape: its final layer produces four raw logits, and a single softmax turns them into competing probabilities that sum to 100%. So the model is forced to spread one unit of belief across acceptable, inappropriate, offensive, and violent. Because those classes are ordered by severity, we can read a single "offensive-or-worse" number off the top two rungs — that is what the threshold acts on. A single-label softmax is the right tool when the classes are mutually exclusive rungs of one scale — a comment has one severity. Contrast that with toxicity, where a comment can be several kinds of bad at once and independent sigmoids keep each signal separate.

Why it matters

"Offensive" is a much broader net than "hateful" or "toxic". A comment can be crude, insulting, or aggressive without attacking a protected group — and a moderation queue still wants to see it. A fast, private, offline severity score is a first triage pass: it runs the instant someone hits submit, without the text leaving the tab, and hands a human only what actually needs eyes.

  • Comment-queue triage — sort an incoming stream so violent and offensive rise to the top and the acceptable majority is auto-approved.
  • Pre-submit nudges — warn a user their draft reads as offensive before they post it, entirely client-side.
  • A severity dial, not a yes/no — the ordered ladder lets you set different actions for inappropriate vs violent instead of one blunt ban.
  • Cheap pre-filter — clear the easy cases on-device before spending a human reviewer or a big LLM on the hard ones.

Honesty note: this is a single model trained mostly on YouTube comments. It misreads sarcasm, quoted abuse, and reclaimed language, and it inherits the biases of its labels. Treat its output as one signal that routes a human — never as an automatic verdict.

How the API works

The convenience path is one call; ask for every class with topk: null:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline("text-classification", "onnx-community/hate_speech_en-ONNX");
const out = await classify("shut up you idiot", { topk: null });
// → [{ label: "LABEL_2", score: 0.99 }, …]  (softmax over 4 classes; LABEL_2 = "offensive")

The config only ships generic LABEL_0…3 ids, so this page maps them to the model card's documented names (acceptable / inappropriate / offensive / violent). For "See inside" it runs the tokenizer then the model() directly, reads the raw four-value logits, and applies the softmax itself — so it can show both the logit and the probability, and compute your offensive-or-worse threshold. All of it runs in a Web Worker so typing never janks.
