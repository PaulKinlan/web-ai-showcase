SigLIP — CLIP, but every label gets its own probability

SigLIP learned the same shared space for pictures and words as CLIP — but it was trained with a sigmoid loss instead of a softmax one. The practical upshot: each label is scored independently. Two labels can both read 90%, and if none fit, all of them can read near 0. That "none of the above" honesty is exactly what tagging and moderation need. Nothing leaves your device.

See inside — sigmoid vs softmax, on the same numbers

SigLIP turns your image into one 768-number vector and each label into its own, then takes the cosine between them. It scales that cosine by a learned temperature and adds a learned bias to get a logit. Here's the fork in the road: SigLIP squashes each logit through an independent sigmoid (its training objective), so every label is a standalone yes/no. CLIP would instead run a softmax across all labels, forcing them to compete and sum to 100%. Same logits, two very different answers — watch the two right-hand columns diverge.

The sigmoid column can total more (or far less) than 100% — that's the point. The softmax column always totals 100% because it makes the labels fight over a fixed pie.

Why it matters

CLIP's softmax always crowns a winner even when the truth is "none of these," and it can't say "this image is both a cat and cozy." SigLIP's per-label sigmoid fixes both: it's a natural multi-label tagger with a real "reject everything" state, it's cheaper to train at scale (which is why the SigLIP family tops many zero-shot benchmarks), and the label set is still just an input you decide at runtime.

  • Honest moderation — score against policy labels with a threshold, and let an image trip several flags or none, instead of forcing a single softmax "winner."
  • Multi-label tagging — a photo really can be beach, sunset, and people at once; independent sigmoids let all three fire.
  • Aesthetic / vibe scoring — rate an image on abstract qualities (minimalist, nostalgic) as absolute 0–1 scores you can compare across images.
  • Private, offline, free — the image and the labels never leave the tab.

How the API works

The pipeline call is identical to CLIP — the difference is in how you read the scores:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "zero-shot-image-classification",
  "Xenova/siglip-base-patch16-224",
); // downloads + caches ~200 MB once

const output = await classify(imageURL, ["a cat", "a dog", "a city street"]);
// → [{ label: "a cat", score: 0.91 }, …]
//   Each score is an INDEPENDENT sigmoid probability — they need not sum to 1.
//   (CLIP's pipeline returns a softmax over the labels instead.)

For "See inside" this page reuses the pipeline's own model, processor, and tokenizer to read the raw logits_per_image and the image/text embeddings from a single forward pass — then it computes the sigmoid (SigLIP's real read-out) and a softmax over the same logits so you can see how differently the two objectives answer. The SentencePiece tokenizer is padded to max_length: 64, the length SigLIP trained on. Inference runs in a Web Worker so the page never janks.
