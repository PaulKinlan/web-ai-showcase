Practical: search & dedup a multilingual knowledge base

The everyday job of a retrieval embedder. Below is a small knowledge base of long passages in several languages. Ask a question in any language and BGE-M3 ranks every passage by meaning; then it flags the near-duplicate passages — the same content written in two languages — that you'd want to collapse in a real index. One model, one 1024-d space, no translation round-trip.

Every pair of passages above whose cosine exceeds the threshold — these are candidates to merge in a real index (usually the same content in two languages).

Because retrieval happens in one shared space, you never maintain a separate index per language, and you never translate the corpus up front. Long passages stay whole — BGE-M3's 8192-token window means you can embed a full paragraph instead of shredding it into sentences that lose context.
