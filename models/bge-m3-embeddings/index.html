BGE-M3 — one meaning space for 100+ languages and whole documents

BGE-M3 is BAAI's flagship multi-functionality embedder. This page runs its dense-retrieval head: the 1024-number vector you drop into a vector database. Two things set it apart from the smaller embedders on this site — it covers 100+ languages in one shared space, and it reads up to 8192 tokens, so it embeds whole paragraphs and documents, not just sentences. Paste passages in any mix of languages, build a cross-language cosine matrix, and search long documents by meaning. Nothing leaves your device.

Honest about the size. At ~570 MB (q8) this is far heavier than the 118 MB multilingual MiniLM or the ~30–90 MB English embedders — a real download and a real memory footprint, especially on phones. You buy 100+ languages, an 8192-token window, and top-tier retrieval quality; if you don't need those, a smaller embedder is the right call. It caches after the first load and clears with the button below.

Why it matters

BGE-M3 collapses the "one index per language" and "chunk everything to a sentence" problems at once. You embed questions, documents, tickets and reviews — long or short, in any of 100+ languages — into one 1024-d space, and nearest-neighbour search just works across the whole corpus. It's the dense head that powers a lot of production multilingual RAG, and it runs here with no server and no data leaving the tab.

  • Multilingual RAG — one retriever for a corpus written in dozens of languages; a query in any language finds the right passage in any other.
  • Long-document search — embed a whole paragraph or section (up to 8192 tokens) instead of shredding it into sentences that lose context.
  • Cross-language dedup — collapse "same content, different language" documents, tickets or reviews into one.
  • Language-agnostic clustering — group a mixed-language corpus by topic, not by which language it happens to be in.

Use cases

Basics

Do two passages mean the same thing across languages, or across a paraphrase? One cosine number.

Practical

Long-document, cross-language semantic search and dedup over a mixed-language knowledge base.

Wild

Cluster a mixed-language pile of text by meaning — watch the groups ignore language entirely.

Multi-model

Retrieve cross-lingually with BGE-M3, then translate the winning passage to English with M2M100.

Contrast with the lighter multilingual MiniLM (384-d, ~50 languages, 118 MB) and the English-only GTE-small: BGE-M3 trades size for language breadth, an 8192-token window, and 1024-d retrieval quality.

How the API works

Same feature-extraction recipe as any sentence embedder — with CLS pooling for BGE-M3's dense head:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Xenova/bge-m3",
); // downloads + caches ~570 MB once

// CLS pooling, no instruction prefix — embed a query and a document IN ANY LANGUAGE the same way:
const out = await extract(["How long does the battery last?", "このバッテリーは一日中もちます"], {
  pooling: "cls",
  normalize: true,
});
const [q, d] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = q.reduce((s, x, i) => s + x * d[i], 0); // high — same meaning, different languages

This page asks the worker for the un-normalized CLS vectors so "See inside" can show the real pre-normalization magnitude, then normalizes them in JS. Inference runs in a Web Worker so the matrix, projection and search never jank the page.
