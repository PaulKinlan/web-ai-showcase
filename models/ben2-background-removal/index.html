BEN2 — matte the subject out of any photo

BEN2 (Background Erase Network 2) is a confidence-guided matting model: it decides, pixel by pixel, how much each pixel belongs to the foreground — and it's tuned for crisp, detailed edges (hair, fur, thin structures). You get a soft alpha matte and a transparent cutout, entirely on your device. Nothing is uploaded.

BEN2 never sees "a cat" — it outputs an alpha matte: a confidence from 0 (background) to 255 (subject) for every pixel. The cutout above is just the photo with that matte stamped into its transparency. The soft, in-between values along the boundary are where the model expresses partial coverage — wisps of fur, hair, motion blur — and BEN2's confidence-guided design is built to keep those edges crisp and detailed.

Why it matters

"Remove the background" used to mean a round-trip to a paid API or twenty minutes in Photoshop. BEN2 does it in a web page, on-device — and its confidence-guided matting is aimed squarely at the hard part: edges. That changes what you can build:

  • Product shots — drop any catalogue photo onto a clean white or brand-colour backdrop, consistently, at scale.
  • Avatars & profile pics — swap a background live, with the image never leaving the browser.
  • Stickers & cutouts — turn a photo into a transparent PNG (or an outlined sticker) for chat, decks, and design tools.
  • Hair & fur — the crisp-edge matte is where a matting model earns its keep vs a hard segmentation mask.

How the API works

BEN2 uses a custom architecture (model_type: "ben"), which transformers.js registers as a custom image-segmentation model. Load the model + processor directly and read the alpha tensor:

import { AutoModel, AutoProcessor, RawImage } from "@huggingface/transformers";

const model = await AutoModel.from_pretrained("onnx-community/BEN2-ONNX", {
  device: "webgpu", dtype: "fp16",   // fp16 is the only exported precision
});
const processor = await AutoProcessor.from_pretrained("onnx-community/BEN2-ONNX");

const image = await RawImage.read(imageURL);
const { pixel_values } = await processor(image);          // [1,3,1024,1024]
const { alphas } = await model({ pixel_values });          // [1,1,1024,1024], 0..1

// Scale to 0–255 and resize the matte back to the original photo:
const matte = (await RawImage.fromTensor(alphas[0].mul(255).to("uint8")))
  .resize(image.width, image.height);
// then copy `matte` into the image's alpha channel → the cutout

This page runs that in a Web Worker and does the per-pixel composite in the worker too (OffscreenCanvas → transferToImageBitmap), so the main thread only draws the finished cutout — the interface never janks even at full resolution.
