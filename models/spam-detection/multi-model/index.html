Why this is the interesting part

A classifier and a summarizer are each useful alone. Chained, they're a template for a whole class of tool: an assistant that filters your messages and hands you a briefing, a support queue that drops the spam and summarizes the real tickets, a comment moderator that recaps only the genuine feedback. The gate matters — feeding junk into a summarizer wastes compute and pollutes the digest, so classify first, summarize second. Both models run locally, no server, no API bill.
