Spam detector — is this message spam or ham?

A DistilBERT classifier fine-tuned to separate junk from real messages. Paste any message or email and it scores it as SPAM or HAM (not spam) with a confidence — then look inside to see which exact words tripped the spam signal. Everything runs on your device.

See inside — the spam-signal words

This is occlusion attribution: we re-run the model with each word removed and see how much the SPAM score shifts (measured in log-odds, because the model is so confident that raw percentages barely move). Remove a word that was pushing "spam" and the score drops — so that word gets a red weight. Remove a word that was keeping it looking legitimate and the score rises — green. No gradients, no hand-waving: just the model's own predictions, N+1 of them.

pushes SPAM pushes not-spam brighter = larger effect

Why it matters

Spam filtering is the oldest applied text classifier there is — and running it in the browser means you can triage messages the instant they arrive, without a round-trip and without their contents ever leaving the device. That unlocks fast, private, always-available filtering.

  • Client-side inbox triage — sort junk from real mail before it renders, with no server pass.
  • Form & comment guarding — flag spammy submissions client-side, cutting server load and PII exposure.
  • Chat & DM filters — catch scam and promo messages in a messaging app, offline.
  • Private email tooling — score sensitive mail locally with no data-handling agreement.

How the API works

Classification is one call; top_k returns both class scores:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "text-classification",
  "onnx-community/tanaos-spam-detection-v1-ONNX",
); // downloads + caches ~136 MB once

const out = await classify("Claim your FREE prize now!", { top_k: 2 });
// → [{ label: "spam", score: 0.9994 }, { label: "not_spam", score: 0.0006 }]

"See inside" re-uses that same call: it classifies the full message plus one copy per word with that word deleted (batched in a single pass), then subtracts the SPAM log-odds to attribute the shift to each word. All of it runs in a Web Worker so typing never janks.
