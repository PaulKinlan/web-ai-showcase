Multilingual toxicity — moderate 15 languages, privately

A multilingual BERT (mBERT) fine-tuned for the TextDetox 2025 shared task. It reads text in 15 languages — Arabic, Hindi, Chinese, Japanese, Hebrew, Russian, Ukrainian, German, Spanish, Italian, French, English and more — and returns a single toxic vs. neutral probability. This is defensive content-safety tooling: set a threshold, triage what to review. Every score is computed on your device; no text is ever sent to a server. It is a probability, not a judgment about a person — and, as the model card is candid about, accuracy varies a lot by language.

Run it

Type or paste text in any of the 15 languages; it re-scores automatically a moment after you stop typing. The moderation threshold decides the verdict: if the toxic probability lands at or above it, the text is flagged for review — otherwise it's allowed. The samples pair a benign and a mild hostile line per language so you can watch the classifier separate them.

See inside — two logits, one softmax

The English toxic-bert demo is multi-label: six independent sigmoids that don't add up. This model is the other shape — binary, single-label. The final layer produces two raw logits (neutral, toxic); a single softmax turns them into two probabilities that do sum to 100%. The table shows both, so you can see exactly how the threshold turns numbers into a decision. The multilingual trick is upstream: mBERT shares one WordPiece vocabulary and one embedding space across 104 languages, so the same two-logit head works whether the input is English or Arabic.

Why it matters

Communities are rarely monolingual. A moderation queue that only understands English is blind to abuse in every other language its users actually write in. A single small multilingual model — running for free, on-device, the instant someone hits submit — is a first line of defence that scales across borders without a per-call bill and without the text ever leaving the tab.

  • Cross-lingual pre-submit guardrails — warn a user their draft reads as toxic before they post, in whatever language they wrote it.
  • Multilingual queue triage — sort a mixed-language comment stream so the most likely-toxic rises to the top for human review.
  • Private-by-default safety — score sensitive text (support tickets, health forums) in many languages without a server round-trip or a third-party API.
  • Honest about its limits — the model card reports test F1 from 0.95 (Ukrainian) down to 0.51 (Arabic); a responsible deployment routes low-confidence and weak-language cases to people.

Use cases

Basics

Score one line in any language and read the two-class toxic/neutral probability.

Practical

Multi-model

How the API works

The convenience path is one call — the model is single-label, so the pipeline returns the winning class:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline("text-classification",
  "onnx-community/bert-multilingual-toxicity-classifier-ONNX");
const out = await classify("أنت شخص غبي");   // Arabic
// → [{ label: "toxic", score: 0.98 }]   (idx 0 = neutral, idx 1 = toxic)

This page goes one level deeper for "See inside": instead of the pipeline it runs the tokenizer then model() in a single forward pass, reads the raw two-value logits tensor, and applies the softmax itself — so it can show both the logit and the probability, and apply your threshold. Same weights, same result. All of it runs in a Web Worker so typing never janks.
