text2text-generation · text · WASM · ~98 MB

Text simplification — rewrite dense text into plain language

This is an encoder-decoder (FLAN-T5) model fine-tuned on one job: text simplification. It rewrites complex, jargon-laden or bureaucratic English into plainer words and simpler sentences — while preserving the meaning (unlike a summariser, which drops content). The whole input string is the sentence to simplify; the encoder reads it, the decoder writes the plain version one token at a time. Paste something dense and watch it get simpler — with a live before/after readability score and a word-level diff of exactly what changed. Nothing leaves your device.

This is the small (~80M) FLAN-T5 checkpoint, so calibrate expectations: it is good at swapping fancy words for plain ones and untangling sentences, but on very long or highly technical input it can be conservative or lightly paraphrase. The page shows you the real output, plainer or not — never a canned answer.

Why it matters

A private, offline plain-language rewriter that fits in a tab. No API key, no per-request bill, and the text never leaves the device — so it is safe for medical letters, legal notices, internal docs, and anything you would not paste into a cloud service.

  • Accessibility — rewrite dense content into plain language for readers with lower literacy, cognitive load, or English as a second language.
  • Plain-language compliance — help meet plain-language mandates for government, health and finance communications, on-device.
  • Onboarding & support — turn jargon-heavy documentation into something a new user can actually follow.
  • Pipeline stage — simplify text before display, before TTS reads it aloud, or before a smaller downstream model has to process it.

Use cases

Basics

Simplify one sentence and watch the reading-ease score jump — see what "plainer" means, measured.

Practical

A plain-language rewriter for accessibility: simplify a whole passage line by line with a running readability delta.

Wild

Throw dense legalese and academic abstracts at it and see how far a small model can untangle them.

Multi-model

Simplify → speak: rewrite in plain language, then read it aloud with MMS-TTS.

How the API works

The whole task is the input string — no template, no instruction verb needed:

import { pipeline } from "@huggingface/transformers";

const simplify = await pipeline(
  "text2text-generation",
  "grazh/agentlans-flan-t5-small-simplifier-onnx-v2",
); // downloads + caches ~98 MB once

await simplify("The utilization of this methodology facilitates the amelioration of comprehension.");
// → [{ generated_text: "Using this method makes it easier to understand." }]

await simplify("Notwithstanding the foregoing, the lessee shall remit payment prior to occupancy.");
// → [{ generated_text: "The tenant must pay before moving in." }]

For the live token stream and the decode timeline, this page passes a TextStreamer so it can timestamp each generated token — all inside a Web Worker so the UI never janks. The readability scores (Flesch reading ease + Flesch–Kincaid grade) and the word-level diff are computed on the page from the input and the model's output.
