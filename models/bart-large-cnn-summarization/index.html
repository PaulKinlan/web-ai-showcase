BART-large-CNN — the canonical news summarizer

The full-size BART fine-tuned on CNN/DailyMail news — the model most abstractive-summary papers benchmark against. It reads long text and writes a shorter abstractive summary in fresh sentences. It's larger and slower than DistilBART, and usually tighter and more faithful. Paste something and watch the summary stream in. Nothing leaves your device.

Why it matters

Summarization turns a wall of text into something you can act on in seconds — and doing it on-device with the higher-quality original means fewer factual slips and tighter phrasing, without shipping a word to a server. No API key, no per-call bill, no data-handling agreement.

  • News & report TL;DRs — collapse a long article or filing to its gist before you read.
  • Research triage — skim abstracts of dozens of long docs, locally.
  • Quality tier — reach for BART-large when a DistilBART summary reads too rough.
  • Pipeline stage — pair with speech-to-text or TTS for recording → recap → read-aloud.

How the API works

Summarization is one pipeline call; length is controlled with generation parameters:

import { pipeline } from "@huggingface/transformers";

const summarizer = await pipeline(
  "summarization",
  "Xenova/bart-large-cnn",
); // downloads + caches ~460 MB once

const out = await summarizer(longText, {
  max_new_tokens: 100,
  min_new_tokens: 30,
  num_beams: 1,          // greedy; raise for tighter (slower) output
  no_repeat_ngram_size: 3,
});
// → [{ summary_text: "…a concise abstractive summary…" }]

For the live token stream and the decode timeline, this page passes a TextStreamer so it can timestamp each generated token — all inside a Web Worker so the UI never janks.
