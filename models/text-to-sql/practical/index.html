Ask a question, T5 writes the SQL, and the in-browser database actually runs it — you see the real rows. When the tiny model produces invalid SQL, you see the honest database error instead of a fake result. Nothing leaves your device.

Why this is the shippable shape

A generate-and-execute loop against a schema you own is exactly how an in-app query assistant works: the model drafts SQL from the user's words, your database engine runs it, and the user sees rows — or a real error they can act on. Here both the model and the database are in the browser, so a whole "chat with your data" feature ships with zero backend and total privacy.
