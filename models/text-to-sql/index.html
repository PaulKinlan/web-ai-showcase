Text-to-SQL — ask in English, get a query

This is a small encoder-decoder (T5-small, ~60M) fine-tuned for one job: writing SQL from a database schema and a plain-language question. Unlike a general text model, it is schema-conditioned — you paste your CREATE TABLE statements in front of the question, and the model grounds its query in your columns and table names. The decoder writes the SELECT one token at a time; watch it stream. Nothing leaves your device.

Calibrate expectations: at 60M params it is genuinely good at single-table filters, counts, GROUP BY aggregations, and simple joins over a schema you give it — and it will sometimes get complex multi-join logic wrong. The page shows the real generated SQL, correct or not, never a canned answer. Always review generated SQL before running it against a real database.

Why it matters

Natural-language querying is one of the most-requested database features, and it usually means shipping every schema and question to a cloud LLM. An 80 MB model in a tab does the common cases privately, offline, and for free — the schema and the question never leave the device.

  • Query assistant in your app — let non-SQL users ask questions of a database whose schema you already know; show them the generated SQL to run or edit.
  • Private analytics on local data — build a "ask your CSV/SQLite" tool that composes queries client-side, with nothing sent to a server.
  • SQL learning aid — a student types a question and sees the query, then reads the token stream to understand how the schema shaped it.
  • Draft-then-review workflows — generate a first-pass query on-device, then send only the (small) SQL for validation instead of the whole prompt.

How the API works

The whole task lives in the input string — the schema is the conditioning, no template roles:

import { pipeline } from "@huggingface/transformers";

const text2sql = await pipeline(
  "text2text-generation",
  "Xenova/t5-small-awesome-text-to-sql",
); // downloads + caches ~80 MB once

// paste the schema in front of the question — this exact shape is what it was trained on:
const prompt =
  "tables:\n" +
  "CREATE TABLE students (student_id INT, name VARCHAR, gpa FLOAT);" +
  " CREATE TABLE attendance (student_id INT, course_id INT)\n" +
  "query for:List the names of students who never attended a course.";

await text2sql(prompt, { max_new_tokens: 128 });
// → [{ generated_text: "SELECT name FROM students WHERE NOT student_id IN
//        (SELECT student_id FROM attendance)" }]

For the live token stream and the decode timeline, this page passes a TextStreamer so it can timestamp each generated token — all inside a Web Worker so the UI never janks.
