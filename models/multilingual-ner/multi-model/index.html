Multi-model: link the same entity across languages

One model finds the entities; a second decides which of them are the same real-world thing named in different languages. Multilingual NER tags the spans, then the built cross-lingual embedder (paraphrase-multilingual-MiniLM) turns each entity string into a vector in a shared 50-language meaning space — so 阿里巴巴 lands next to Alibaba, and Estados Unidos next to United States. Two small models, one on-device pipeline, nothing uploaded.

How the chain works

  1. Extract — each line is tagged by multilingual NER; we collect the merged entity spans with their class and source language.
  2. Embed — every unique entity string is embedded once by the cross-lingual model into a shared 384-d space where translations sit together.
  3. Link — within each class we join mentions whose vectors are within the cosine threshold (a union-find), producing clusters that ignore which language wrote them.

This is the core of cross-lingual entity resolution: the same building block behind merging a multilingual knowledge graph or de-duplicating a global customer list — done entirely on the device.
