Multilingual NER — name the people, places & dates in ten languages

Named-entity recognition reads a sentence and labels each span: who (PersonPER), what organization (OrgORG), where (LocationLOC), and when (DateDATE). Unlike the English BERT-NER page, this one runs on a multilingual BERT backbone and tags the same four classes across Arabic, German, English, Spanish, French, Italian, Latvian, Dutch, Portuguese and Chinese — one model, no language flag. Pick a language below and watch the entities light up. All on your device.

See inside — the per-token BIO tags

Multilingual BERT breaks every language into WordPiece tokens (whole words, sub-word fragments marked ##, or single CJK characters), and tags each token independently: B-PER (begin person), I-PER (inside person), and so on. We then stitch consecutive B-/I- runs into whole entities. Below are the raw tags and confidences straight from the model — the same machinery, whether the input is French, Chinese or Arabic.

Why it matters

Most NER models are English-only. A single model that structures text in ten languages means you can index, redact, or link entities across a multilingual corpus without a translation step and without a per-language pipeline — and, because it runs in the browser, without a byte of the source text leaving the device.

  • Cross-border PII redaction — mask people, places and dates in documents in any of the ten languages, client-side, before they're shared or logged.
  • Multilingual contact / data extraction — pull the who-where-when out of emails, notes or transcripts regardless of the language they arrived in.
  • One tagging pipeline, many locales — retire ten single-language NER services for one model.
  • Cross-lingual entity graphs — link the same organisation or person named in different languages (see the multi-model demo).

How the API works

Token classification is one pipeline call — the multilingual model is a drop-in for the English one:

import { pipeline } from "@huggingface/transformers";

const ner = await pipeline(
  "token-classification",
  "Xenova/bert-base-multilingual-cased-ner-hrl",
);
const out = await ner("Angela Merkel a rencontré Barack Obama à Berlin en 2015.");
// → [{ entity: "B-PER",  word: "Angela", score: 0.99, index: 1 },
//    { entity: "I-PER",  word: "Merkel", score: 0.99, index: 2 },
//    { entity: "B-PER",  word: "Barack", score: 0.99, index: 5 },
//    { entity: "I-PER",  word: "Obama",  score: 0.99, index: 6 },
//    { entity: "B-LOC",  word: "Berlin", score: 0.99, index: 8 },
//    { entity: "B-DATE", word: "2015",   score: 0.98, index: 10 }]

The pipeline returns one BIO-tagged row per WordPiece token, but no character offsets. We map each token back to its span in the original text (a cursor-walk) so highlights land exactly — across Latin, CJK and RTL scripts — then merge B-/I- runs (and ## sub-word pieces) into whole entities with a pooled confidence. It all runs in a Web Worker so typing stays smooth.
