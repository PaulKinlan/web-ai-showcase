Clothes Parsing — give every pixel of a person a garment name

This is human parsing: not "cut out the person" and not "find the face" — every pixel of a person is classified into one of 18 clothing and body classes (upper-clothes, dress, skirt, pants, hat, scarf, belt, bag, shoes, hair, face, arms, legs). Pick a photo and watch it become a colour-blocked map of garments, entirely on your device.

A different job from the segmentation demos already here: portrait matting cuts the whole person out as one silhouette, face parsing labels facial regions, and background removal is binary foreground/background. Clothes parsing labels which garment each pixel belongs to — the raw material for fashion tagging, virtual try-on, and per-garment edits.

Why it matters

A per-pixel garment map is the raw material for fashion, retail, and avatar features — and having it run locally means you can analyse or edit what someone is wearing without uploading their photo to a server.

  • Fashion tagging & search — turn a photo into "wearing a dress + hat + bag" tags, or index a catalogue by garment, all client-side.
  • Virtual try-on prep — the upper-clothes / dress / pants masks are exactly what a try-on or garment-swap pipeline needs to composite a new item.
  • Per-garment edits — recolour only the "upper clothes," extract just the "dress," or blur everything except the outfit — precise region edits with no manual masking.
  • Dataset & retail QA — auto-label garments to bootstrap a training set, or check that a product shot frames the item, without sending photos off the device.

Use cases

Basics

Parse a person and read the colour map and legend — see human parsing plainly.

Practical

Fashion tagger + garment cut-out: get "what are they wearing" tags and extract one garment on a transparent background.

Wild

Outfit recolour: restyle the upper-clothes, dress, or pants to a new colour, on-device.

Multi-model

Clothes parsing → CLIP: extract the outfit, then ask CLIP what style it is.

How the API works

Clothes parsing is the standard Transformers.js semantic-segmentation pipeline. It returns one entry per class present, each with a full-resolution mask:

import { pipeline } from "@huggingface/transformers";

const parse = await pipeline(
  "image-segmentation",
  "Xenova/segformer_b2_clothes",
); // downloads + caches ~29 MB once

const output = await parse(imageURL);
// → [{ score: null, label: "Upper-clothes", mask: RawImage },
//    { score: null, label: "Pants", mask: RawImage },
//    { score: null, label: "Hair", mask: RawImage }, … ]
//   each mask is 0/255 for that class; together they tile the whole image

This page runs that in a Web Worker, then folds the per-class masks into a single colour-coded overlay (argmax per pixel) and a per-class pixel count — so the map, legend, and coverage bars are all views of the one label map the model produced. Nothing is uploaded.
