Nested templates and deliberately messy text

How far does a 0.5B extraction model go? These examples use nested templates, lists of objects, and messy, run-on, multi-entity text. NuExtract fills the whole tree in one pass — and because the output is validated with JSON.parse(), you see honestly whether it held the structure or not.

What's happening

A nested template asks the model to hold a deeper structure while still copying values verbatim. Small models can drop a field or mis-close a bracket on the hardest inputs — this rung is honest about that: valid JSON lights up green, invalid lights up red with the raw output shown. That transparency is the point; nothing is hidden to make the demo look perfect.
