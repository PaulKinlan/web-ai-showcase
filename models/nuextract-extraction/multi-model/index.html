Multi-model: image → text → structured JSON

Extraction usually starts from a document, not clean text. So chain two models: TrOCR reads the printed text off a receipt image, then NuExtract turns that text into JSON. Pixels in, a typed record out — both models running in your browser, nothing uploaded.

What's happening

TrOCR is a single-line printed-text recogniser, so it reads the total line off the receipt image; that string flows into NuExtract, which fills your template. Both models run in their own Web Workers — the ViT-decoder OCR and the CPU-bound extraction never block the page. This is the shape of real document automation on the edge: a reader model plus a structuring model, composed client-side, working offline once cached.
