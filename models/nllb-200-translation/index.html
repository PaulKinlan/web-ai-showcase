NLLB-200 — translate 200 languages, no language left behind

Meta's No Language Left Behind covers 200 languages — direct, many-to-many, with no English pivot — and reaches deep into low-resource languages that other models skip: Zulu, Yoruba, Chhattisgarhi, Dzongkha, Tamasheq. Search a source and target, type, and watch it translate on-device. NLLB steers the output language with a single forced target-language token, which you can see below.

See inside — FLORES-200 codes, forced BOS token & tokens

NLLB is a sequence-to-sequence translator: it encodes the source, then decodes the target one token at a time. What selects the output language is a single trick — the decoder is forced to begin with the target language's special token (forced_bos_token_id). Change the target and only that first token changes; the same input then decodes into a different language. Why it matters

Most translation stops at the ~100 highest-resource languages. NLLB-200 doubles that reach — and its whole point is the long tail: languages spoken by hundreds of millions of people that commercial APIs under-serve or omit. Running it on-device means that reach is private, offline, and free of per-call fees.

  • Reach under-served languages — Zulu, Yoruba, Igbo, Nuer, Dzongkha, Santali: pairs many models don't offer at all.
  • Private & offline — translate messages, health, or civic text where nothing can leave the device, with no signal needed.
  • Relay translation — bridge two rare languages through a high-resource hop (A→English→C) entirely locally.
  • Localisation drafts at the edges — first-pass UI/content strings for languages you'd otherwise skip, then have a human review.

How the API works

Translation is one pipeline call; the FLORES-200 language pair is set per call:

import { pipeline } from "@huggingface/transformers";

const translator = await pipeline(
  "translation",
  "Xenova/nllb-200-distilled-600M",
); // downloads + caches ~900 MB once

const out = await translator("Hello, how are you today?", {
  src_lang: "eng_Latn",   // FLORES-200 code: ISO 639-3 + ISO 15924 script
  tgt_lang: "zul_Latn",   // Zulu (Latin script)
});
// → [{ translation_text: "Sawubona, unjani namuhla?" }]

// Under the hood, tgt_lang selects a forced BOS token id that the decoder
// must emit first — that single token is what picks the output language.

The 200 FLORES-200 codes are three-letter ISO 639-3 language plus a four-letter script tag (zul_Latn, zho_Hans, kas_Deva) — so the same language in two scripts is two codes. This page streams tokens via a TextStreamer inside a Web Worker, so decoding never blocks the interface.
