all-mpnet-base-v2 — the default sentence embedder

If you've ever read a semantic-search tutorial, it probably used this model. all-mpnet-base-v2 is the canonical sentence-transformers workhorse — roughly 250 million downloads a month — an MPNet encoder fine-tuned on 1.17 billion sentence pairs that maps sentences and short paragraphs to 768-dimensional vectors. Mean pooling, no query/document prefixes, one symmetric meaning space. It is the baseline every newer embedder on this site is compared against — and it runs entirely in your browser. Nothing leaves your device.

At a glance

ModelXenova/all-mpnet-base-v2 (ONNX export of sentence-transformers/all-mpnet-base-v2)
Task / pipelinefeature-extraction (dense sentence embedding)
ArchitectureMPNet (109M params, 12 layers) — pre-trained with masked and permuted language modelling, then contrastively fine-tuned on 1.17B sentence pairs
Outputone 768-dimensional vector per input; cosine similarity compares meanings
Poolingmean pooling over every token's vector, then L2-normalize — the sentence vector is literally the average of the token vectors (see inside, below)
Prefixesnone — the space is symmetric: queries and documents are embedded identically (unlike arctic-embed's "query: " or e5's query:/passage:)
Contextinputs truncate at 384 word pieces — a sentence-and-short-paragraph encoder, not a long-document one
LanguageEnglish (for multilingual, see the multilingual MiniLM or arctic-embed-m-v2.0)
Quantizationq8 (8-bit) ONNX
BackendWebAssembly (runs anywhere; no WebGPU needed)
Download~110 MB (q8), cached after first load
LicenseApache-2.0
Web APIsWebAssembly, Web Workers — all Baseline widely available

Why it matters

This is the embedding model an entire generation of semantic search was built on. It's the default in the sentence-transformers docs, the baseline row in embedding benchmarks, and the vector behind countless FAQ bots, dedup pipelines and RAG prototypes. Because it needs no prefixes and no special tokens, it drops into any pipeline: embed everything once, compare with cosine, done. In the browser that means private, serverless semantic features — search, clustering, dedup, recommendations — with zero per-query cost.

  • Semantic FAQ / help-centre search — match a user's question to answers by meaning, not keywords, fully client-side.
  • Duplicate detection — find near-duplicate tickets, reviews or forum posts with a cosine threshold instead of brittle string matching.
  • Clustering & triage — group free-text feedback into themes locally, before anything is sent anywhere.
  • The recall stage of retrieve-then-rerank — pair it with a cross-encoder for the classic two-stage search pipeline (see the multi-model demo).
  • Ship it commercially — Apache-2.0, so it can go straight into a product.

Use cases

Basics

One cosine number for "do these two sentences mean the same thing?" — plus odd-one-out.

Practical

A client-side FAQ answerer and a near-duplicate detector for tickets and reviews.

Wild

Semantic Hot & Cold — a guessing game played entirely in embedding space — and sentence algebra.

Multi-model

Retrieve-then-rerank: MPNet recall + MS MARCO cross-encoder precision, the classic two-stage pipeline.

Contrast with the built all-MiniLM-L6-v2 (same family, 5× smaller and faster, 384-d, a little less accurate), the GTE / BGE generation (newer training recipes) and arctic-embed-m-v2.0 (multilingual, long-context): this page is the reference point they're all measured against.

How the API works

The model-card recipe is mean pooling + L2-normalize — in Transformers.js that's two options on the feature-extraction pipeline. No prefixes, no special tokens:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Xenova/all-mpnet-base-v2",
  { dtype: "q8" },
); // downloads + caches ~110 MB once

const sentences = [
  "How long does the battery last?",
  "Battery life is excellent — a full day on one charge.",
  "I planted tomatoes in the garden.",
];
const emb = await extract(sentences, { pooling: "mean", normalize: true });
// emb.dims === [3, 768] — one unit-length vector per sentence

// unit vectors → cosine similarity is just the dot product:
const [a, b, c] = emb.tolist();
const sim = (u, v) => u.reduce((s, x, i) => s + x * v[i], 0);
sim(a, b); // high — same meaning, different words
sim(a, c); // low — unrelated

This page asks the worker for the un-normalized mean-pooled vectors so "See inside" can show the real pre-normalization magnitude — and the per-token vectors so it can show the average being taken — then normalizes in JS. Inference runs in a Web Worker so the matrix, projection and search never jank the page.
