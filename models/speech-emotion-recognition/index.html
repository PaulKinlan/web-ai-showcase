Speech Emotion Recognition — hear the feeling, not the words

Two people can say "I'm fine" and mean opposite things. This model listens to how something is said — the acoustics of the voice, not the transcript — and names the emotion: angry, happy, sad, fearful, disgust, or neutral. It's a wav2vec2 network that reads the raw waveform, running entirely on your device.

An honest caveat: emotion models learn from acted emotional speech. They pick up real acoustic cues (pitch, energy, tremor), but they are not a lie detector and real neutral speech can get surprising labels. Read the confidence bars, not just the top label — and treat this as a signal, not a verdict.

Why it matters

Tone carries meaning that transcripts throw away. Detecting it on-device means an app can react to how a user sounds without shipping their voice to a server — private by construction.

  • Call-centre & support triage — flag calls where a caller sounds angry or distressed so a supervisor can step in, without recording or uploading the audio.
  • UX & product research — measure emotional reaction in spoken user-testing feedback, locally, instead of hand-coding hours of tape.
  • Accessibility & wellbeing — a journaling or mood app that notices tone over time, all data staying on the device.
  • Expressive interfaces — a character or avatar that mirrors the emotion in your voice in real time, no cloud round-trip.

How the API works

Classification is one call with Transformers.js. Feed a 16 kHz mono Float32Array:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "audio-classification",
  "onnx-community/wav2vec2-base-Speech_Emotion_Recognition-ONNX",
  { dtype: "q8" },              // downloads + caches ~95 MB once
);

const output = await classify(audio, { top_k: 6 });
// audio = Float32Array @ 16 kHz mono (decode with the Web Audio API)
// → [{ label: "ANGRY", score: 0.62 }, { label: "NEUTRAL", score: 0.15 }, … ]

This page decodes files / mic recordings to 16 kHz mono with an OfflineAudioContext, runs the pipeline inside a Web Worker so the UI never janks, and renders the full probability distribution — not just the winner.
