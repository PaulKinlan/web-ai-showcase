SpeechT5 — one model, many voices, chosen by a vector

SpeechT5 is a speaker-conditioned text-to-speech model. Unlike a model with a fixed set of named voices, its voice is set by a 512-number speaker embedding (an "x-vector"). Feed it a different x-vector and the same sentence is spoken by a different person — voice-cloning-adjacent, entirely in your browser. A HiFi-GAN vocoder turns the model's mel-spectrogram into the waveform you hear.

Why it matters

SpeechT5 decouples what is said from who says it. Because the voice is just a vector, you can store, swap, blend, or derive voices at runtime — no per-voice model, no cloud TTS bill, and no text ever leaving the tab. It's a different design point from a fixed-voice model like Kokoro: more knobs, a smaller download, and a built-in path toward personalized voices.

  • Speaker-selectable narration — let a user (or a brand) pick a voice from a library of x-vectors, chosen at runtime.
  • Private read-aloud — give any article, form, or message a natural voice with nothing uploaded.
  • Voice for on-device agents — pair with a local LLM or translator so a private assistant speaks back in a chosen voice.
  • Voice experiments — because the identity is a vector, you can interpolate between speakers or explore the embedding space directly.

Use cases

Basics

Type a line, pick a speaker x-vector, hear it — the core move.

Practical

Read-aloud: paste an article, choose a voice, and listen sentence by sentence.

Wild

Voice swap: speak one line through every x-vector and compare the same words in each voice.

Multi-model

Translate → speak: M2M100 translates your text, then SpeechT5 reads the translation aloud.

How the API works

One pipeline call. The voice is the speaker_embeddings argument — a 512-dim vector:

import { pipeline } from "@huggingface/transformers";

const tts = await pipeline(
  "text-to-speech",
  "Xenova/speecht5_tts",
  { dtype: "q8" }, // WASM fallback when WebGPU is absent; the HiFi-GAN vocoder auto-loads
);

// A 512-float x-vector — from a file, a URL, or one you derived yourself.
const speaker_embeddings = new Float32Array(await (await fetch(xvectorUrl)).arrayBuffer());

const out = await tts("Hello from your browser.", { speaker_embeddings });
// out.audio         → Float32Array waveform
// out.sampling_rate → 16000

This page runs the pipeline inside a Web Worker so synthesis never blocks the interface, encodes the returned Float32 samples to a WAV blob for the <audio> element, and draws the real x-vector, spectrogram, and waveform under "See inside". The six speakers are real CMU-Arctic x-vectors bundled next to this page — nothing hard-coded about the audio.
