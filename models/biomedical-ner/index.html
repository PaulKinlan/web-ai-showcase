Biomedical NER — read a clinical note like a clinician

General named-entity recognition knows people, places and organizations. This one knows medicine. It labels each span in clinical and biomedical text with one of 40+ domain entity types — a diseaseDisease disorder, a drugMedication, a lab valueLab value, a procedureDiagnostic procedure — that generic NER would miss entirely. Paste a case note and watch it light up. All on your device.

The model doesn't see "myocardial infarction" as one thing — it tags each WordPiece token independently: B-Disease_disorder (begin), I-Disease_disorder (inside), and so on, across 43 types. We stitch consecutive B-/I- runs into whole entities. Why it matters

A generic NER model tells you a note mentions a person and a place. A biomedical one tells you the diagnosis, the drug, the dose, and the lab value — the structured clinical facts a downstream system can actually act on. Doing it in the browser means you can mine or de-identify clinical text without a single byte leaving the device — the difference between "possible" and "compliant" for regulated health data.

  • Literature mining — pull diseases, drugs and findings out of papers and abstracts to build a structured index.
  • Clinical de-identification (concept) — surface Age / Sex / Date / person spans to redact before text is shared or logged, on-device.
  • Coding & structuring assist — turn a free-text note into a first-pass list of conditions, medications and procedures for a human to confirm.
  • Cohort & safety signals — scan notes for a medication plus an adverse sign co-occurring, entirely locally.

A research/education demo — not a medical device, and not for clinical decisions. Model output is probabilistic and will make mistakes; the confidence scores are shown for exactly that reason.

How the API works

Token classification is one pipeline call — only the model id is domain-specific:

import { pipeline } from "@huggingface/transformers";

const ner = await pipeline("token-classification", "onnx-community/biomedical-ner-all-ONNX", { dtype: "q8" });
const out = await ner("started on aspirin 300mg for chest pain");
// → [{ entity: "B-Medication", word: "aspirin", score: 0.99, index: 3 },
//    { entity: "B-Dosage", word: "300", score: 0.99, index: 4 },
//    { entity: "I-Dosage", word: "##mg", score: 0.99, index: 5 },
//    { entity: "B-Sign_symptom", word: "chest", score: 0.99, index: 7 },
//    { entity: "I-Sign_symptom", word: "pain", score: 0.99, index: 8 }]

The pipeline returns one BIO-tagged row per WordPiece token, but no character offsets. We map each token back to its span in the original text (a cursor-walk) so highlights land exactly, then merge B-/I- runs (and ## sub-word pieces) into whole entities with a pooled confidence. Because there are 43 fine-grained types, the UI colours them by eight clinically meaningful groups (condition, anatomy, procedure, medication, measurement, temporal, person, descriptor) while always showing the exact type. It all runs in a Web Worker so typing stays smooth.
