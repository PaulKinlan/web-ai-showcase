MobileLLM-R1 360M — a reasoning model that runs on your CPU

MobileLLM-R1 is Meta's 2025 on-device reasoning family — sub-billion-parameter models post-trained on math, code, and science to reason step by step on a phone. This page runs the 360M checkpoint entirely in your browser via Transformers.js on the universal WebAssembly/CPU path — no GPU required. Give it a problem and watch it think inside <think> tags, then answer — all streamed on your device. No API key, no server, nothing uploaded.

A different family from the other small LLMs here (Qwen, Llama, Gemma, SmolLM, Phi, ERNIE): MobileLLM-R1 uses Meta's compact llama4_text architecture (deep-and-thin with grouped-query attention). It is a reasoning model for math/code, not a general chatbot — steer it with problems, not trivia. At 360M in 4-bit it is small, so expect small-model reasoning: often right, sometimes confidently wrong.

Why it matters

MobileLLM-R1 is a bet that reasoning — not just fluent chat — can fit in a sub-billion model that runs on the device in your pocket. Because it is small and CPU-friendly, it makes step-by-step problem solving a local primitive: no GPU, no API bill, no data leaving the machine, and it works offline once cached.

  • On-device math & logic helpers — a homework or spreadsheet assistant that shows its working, entirely in the page.
  • Local coding aids — draft a small function or explain a snippet without shipping code to a server.
  • Transparent reasoning — the <think> trace is visible, so you can see how it got there and catch mistakes.
  • Reason over your own data — pair it with a local embedding model for grounded, private question answering (see the multi-model demo).

How the API works

It's the standard Transformers.js text-generation pipeline — pass a messages array (the pipeline applies MobileLLM-R1's Llama-4 chat template), a TextStreamer for token-by-token output, and your sampling settings: 

import { pipeline, TextStreamer } from "@huggingface/transformers"; // v3.7.5

// Universal WASM/CPU path — no WebGPU needed. 4-bit weights (~676 MB, cached once).
const generator = await pipeline(
  "text-generation", "onnx-community/MobileLLM-R1-360M-ONNX",
  { device: "wasm", dtype: "q4" },
);

const messages = [
  { role: "system", content: "Think step by step, then give the final answer." },
  { role: "user", content: "What is 17 * 24? Work it out step by step." },
];

const streamer = new TextStreamer(generator.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });

await generator(messages, {
  max_new_tokens: 320, do_sample: true, temperature: 0.6, top_p: 0.9, streamer,
});
// → "<think> 17 * 24 = 17 * 20 + 17 * 4 = 340 + 68 = 408 </think> The answer is \boxed{408}."

The "see inside" panel runs one raw forward pass — generator.model(generator.tokenizer(prompt)) — reads the last position's logits, softmaxes them, and shows the real top-k. Everything runs in a Web Worker so generation never blocks the page; the loader auto-inits an already-cached copy and never silently re-downloads.
