wav2vec2-large — Finnish speech, one character per frame

The Russian XLSR, Thai XLSR and other demos here are XLSR-53 fine-tunes. This one is a close cousin, but honestly not the same checkpoint: neither jonatasgrosman's nor aapot's Finnish XLSR-53 model ships a browser-loadable ONNX, so a true Finnish XLSR-53 can't run in a tab today. What can run is Finnish-NLP/wav2vec2-large-uralic-voxpopuli-v2-finnish: a wav2vec2-large model pre-trained on the VoxPopuli Uralic-languages subset (a different cross-lingual pretraining than XLSR-53) and fine-tuned on Finnish. Same CTC mechanics; the output alphabet is the 34-symbol Latin-Finnish set (a–z + ä å ö). Pick a clip and watch the frames collapse into Finnish.

Why it matters

Most of the world's ~7,000 languages have little labelled speech data. Cross-lingual self-supervised pre-training is how the field reaches them: this model learned speech structure from the Uralic languages in VoxPopuli, so fine-tuning to Finnish needed far fewer transcripts. That is what puts a usable Finnish recogniser inside a browser tab — private, offline once cached, no server, no key. And because it's CTC (frames, not generated tokens) it can't hallucinate an unspoken sentence, and every character is pinned to a moment in time.

  • On-device Finnish dictation and captions — for apps that must keep audio on the device (health, legal, journalism) in Finnish-speaking contexts.
  • One family, many languages — the same wav2vec2-large + CTC recipe (whether XLSR-53 or VoxPopuli-Uralic pretraining) produces recognisers across this showcase.
  • Finnish forced alignment — real per-word timings for karaoke, subtitles, and pronunciation feedback, straight from the frames.
  • Privacy-first field work — transcribe sensitive Finnish audio with nothing leaving the device.

How the API works

Normally the one-liner is the ASR pipeline — but this repo ships no config.json and keeps its ONNX at the repo root, so we load the model with a config borrowed from the base repo, normalise the audio ourselves, and do the CTC collapse (which is exactly what the pipeline does internally): 

import { AutoConfig, AutoModelForCTC, Tensor } from "@huggingface/transformers";

const model_id = "KalleLaht/wav2vec2-large-uralic-voxpopuli-v2-finnish-ONNX";
// the export repo has no config.json → borrow the architecture config from the base repo
const config = await AutoConfig.from_pretrained("Finnish-NLP/wav2vec2-large-uralic-voxpopuli-v2-finnish");
const model = await AutoModelForCTC.from_pretrained(model_id, {
  device: "wasm", dtype: "fp32",       // the only export is a single fp32 file…
  subfolder: "",                       // …at the repo ROOT…
  model_file_name: "wav2vec2_model",   // …under this name
  config,
});

// audio = 16 kHz mono Float32Array → normalise to zero mean / unit variance (wav2vec2 feature step)
const x = normalize(audio);
// this graph needs BOTH input_values and an int32 attention_mask (all-ones for a single clip)
const { logits } = await model({
  input_values: new Tensor("float32", x, [1, x.length]),
  attention_mask: new Tensor("int32", new Int32Array(x.length).fill(1), [1, x.length]),
});
// logits shape [1, frames, 34]  ← 34 Latin-Finnish symbols

// greedy CTC decode: argmax per frame, merge duplicates, drop the blank ([PAD]=31); | → space
const ids  = argmaxPerFrame(logits);
const text = collapse(ids);             // "hei…" in Finnish — no decoder involved

Inference runs in a Web Worker so the UI never janks. The worker returns the raw per-frame argmax and real per-word timings (from the frame each character was emitted at) — nothing is faked or pre-computed.
