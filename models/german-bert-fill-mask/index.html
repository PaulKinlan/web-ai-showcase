German DistilBERT — predict the missing German word

This is a DistilBERT trained only on German text. Its one job here is masked language modelling: hide a word behind [MASK] and it ranks what belongs there — with a German-native vocabulary, German casing and German compound words baked in. Type a sentence with a mask and see its guesses, the exact logits it computed, and how it chops German words into subword pieces. Everything runs on your device.

See inside — logits + how German splits into subwords

A language model doesn't output words — it outputs a logit (a raw, unbounded score) for every one of its 31,102 German vocabulary tokens at the masked position. We softmax those into probabilities. The bars above show the probability; the logit beside each is the actual number the network produced. The – below is the entropy of the top-k — low means the model is sure, high means it's hedging.

Why it matters

A model trained only on German reads German the way a multilingual model can't afford to — every token in its vocabulary is a German subword, so umlauts, the ß, capitalised nouns and famously long compound words come out cleaner. Because it scores every word in context, you can ask it what fits, how strongly, and compare alternatives — all locally, with no prompt engineering and no API bill.

  • German autocomplete — suggest the next or missing word in a compose box, offline, with German-native suggestions.
  • Cloze & language drills — generate fill-in-the-blank German exercises with graded difficulty from the entropy.
  • Register & word-choice checks — compare candidate words at a slot (formal vs colloquial, Sie vs du) using the logits.
  • Compound-word insight — see how German words decompose, useful for spell-assist, hyphenation, and learner tools.

How the API works

The one-liner uses the fill-mask pipeline:

import { pipeline } from "@huggingface/transformers";

const unmask = await pipeline("fill-mask", "onnx-community/distilbert-base-german-cased-ONNX");
const out = await unmask("Die Hauptstadt von Deutschland ist [MASK].");
// → [{ token_str: "Berlin", score: 0.61, sequence: "…" }, { token_str: "Deutschland", … }]

For the "see inside" surface we go one level lower — AutoModelForMaskedLM — so we can read the exact logits tensor ([1, sequence, 31102]), slice the row at the [MASK] position, and softmax it ourselves. That's the only way to show the real logit values, not just the pipeline's rounded scores. All of it runs in a Web Worker so the page never janks.
