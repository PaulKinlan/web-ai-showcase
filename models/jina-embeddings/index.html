Jina v2 — the embedder that reads the whole document

Most sentence embedders stop at 512 tokens — feed them a long document and they quietly ignore everything after the first few paragraphs. Jina embeddings v2 swaps learned positional embeddings for ALiBi, so a single 768-number vector can represent up to 8192 tokens — a whole article. Type passages to build a live similarity matrix, search long text by meaning, and watch Jina catch an answer buried in a tail that a 512-token model never even reads. Nothing leaves your device.

See inside — the vector, and the long-context advantage

Why it matters

Long context changes what you can embed. Instead of chopping a document into 512-token chunks and hoping the right chunk survives, you embed the whole thing once — fewer vectors, no lost tail, and a representation that actually reflects the full text. All private, because nothing leaves the tab.

  • Whole-document search — index articles, contracts or transcripts as one vector each, not a pile of fragments.
  • Long-form dedup & clustering — group near-duplicate documents by their full content.
  • RAG without chunk-juggling — retrieve whole passages for a local LLM instead of stitching fragments.
  • Robust classification — embed the entire text so the label reflects the ending, not just the intro.

Use cases

Basics

Do two passages mean the same thing? One cosine number — even for long text.

Practical

Long-document semantic search and near-duplicate detection over full paragraphs.

Wild

Cluster a pile of passages into themes with k-means over the embeddings.

Multi-model

Jina retrieves a shortlist, then a cross-encoder reranker re-scores the top hits.

How the API works

Feature-extraction with mean pooling — and no prefix, no 512-token ceiling:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Xenova/jina-embeddings-v2-base-en",
); // downloads + caches ~139 MB once

// Jina uses MEAN pooling and needs NO prefix — and accepts up to 8192 tokens:
const out = await extract([longDocument, query], { pooling: "mean", normalize: true });
const [doc, q] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = q.reduce((s, x, i) => s + x * doc[i], 0);

This page pools with normalize:false so "See inside" can show the real pre-normalization magnitude, then normalizes in JS. Inference runs in a Web Worker, so the matrix, projection and long-context test never jank the page.
