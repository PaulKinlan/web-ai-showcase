wav2vec2 — speech to text, one character per frame

wav2vec2 is a different kind of speech model. Whisper, Moonshine and Distil-Whisper are seq2seq — a decoder generates text token by token (and can hallucinate). wav2vec2 is a CTC model: it labels every ~20 ms frame of audio with a single character (or a blank), then collapses that per-frame strip into words. No decoder, no autoregression, no invented words. Pick a clip and watch the collapse happen.

At a glance

ModelXenova/wav2vec2-base-960h (facebook/wav2vec2-base-960h, English)
Task / pipelineautomatic-speech-recognition — but loaded as AutoModelForCTC so we can see the frames
ArchitectureCTC (Connectionist Temporal Classification): a 12-layer transformer encoder over raw audio → per-frame logits across a 32-symbol vocab (blank + "|" word boundary + 26 letters + apostrophe). ~94M params. No decoder.
Quantizationq8 (8-bit) ONNX — onnx/model_quantized.onnx
BackendWebAssembly (WebGPU when a real adapter is present), honest fallback
Download~95 MB (single encoder model), cached after first load
LicenseApache-2.0 (facebook wav2vec2)
Web APIsWebAssembly / WebGPU, Web Workers, Web Audio (decode + resample), MediaRecorder (mic)

Why it matters

Because CTC labels frames instead of generating tokens, it has properties Whisper doesn't. It can't hallucinate a plausible-but-unspoken sentence when the audio is silent or noisy — worst case it emits blanks. And because every character is pinned to a frame, you get timing for free: exactly where in the audio each word (and letter) was said. That's the basis for forced alignment, karaoke, and pronunciation feedback.

  • Forced alignment — line up a known transcript to audio at the word/character level, from the frame positions.
  • Pronunciation & reading feedback — compare what was said, frame by frame, against the target words.
  • Hallucination-free captions — for compliance/medical contexts where an invented word is worse than a blank.
  • Phoneme/character research — inspect the raw acoustic posteriors, not just a cleaned string.

How the API works

The one-liner is the ASR pipeline — but to see the frames we load the model directly and do the CTC collapse ourselves (which is exactly what the pipeline does internally): 

import { AutoProcessor, AutoModelForCTC } from "@huggingface/transformers";

const processor = await AutoProcessor.from_pretrained("Xenova/wav2vec2-base-960h");
const model = await AutoModelForCTC.from_pretrained(
  "Xenova/wav2vec2-base-960h",
  { device: "wasm", dtype: "q8" },
);

// audio = 16 kHz mono Float32Array
const inputs = await processor(audio);
const { logits } = await model(inputs); // shape [1, frames, 32]

// greedy CTC decode: argmax per frame, merge duplicates, drop the blank (id 0)
const ids = argmaxPerFrame(logits);     // e.g. [0,0,7,7,0,12,...]
const text = collapse(ids);             // "THE ..." — no decoder involved

Inference runs in a Web Worker so the UI never janks. The worker returns the raw per-frame argmax and real per-word timings (from the frame each character was emitted at) — nothing is faked or pre-computed.
