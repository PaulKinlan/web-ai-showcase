wav2vec2-XLSR-53 — Korean speech, one Hangul syllable per frame

The French, Spanish and Russian demos are all the same cross-lingual cousin — XLSR-53, pre-trained on 53 languages (56,000 hours of multilingual audio) — fine-tuned per language. This one is fine-tuned on Korean, and it does something the Latin-script versions can't: its CTC head is not an alphabet of letters. It emits whole Hangul syllable blocks — 가, 각, 간 … 힘 — from a 1205-symbol vocabulary. Every ~20 ms frame is labelled with one complete syllable, and the transcript is just those frames collapsed. Pick a clip and watch the syllables assemble into Korean.

Why it matters

Most of the world's ~7,000 languages have little labelled speech data. Cross-lingual self-supervised pre-training is how the field reaches them: XLSR-53 learns speech structure from 53 languages at once, so fine-tuning to a new language needs far fewer transcripts. That is what puts a usable Korean recogniser inside a browser tab — private, offline once cached, no server, no key. And because it's CTC (frames, not generated tokens) it can't hallucinate an unspoken sentence, and every syllable is pinned to a moment in time.

  • On-device Korean dictation and captions — for apps that must keep audio on the device (health, legal, journalism).
  • A syllable-level output that maps to Hangul directly — the units the model emits are the units Korean is written in, so alignment and highlighting are natural.
  • Korean forced alignment — real per-syllable timings for karaoke, subtitles, and pronunciation feedback, straight from the frames.
  • One backbone, many languages — the same XLSR-53 recipe produces the French, Spanish, Russian and Korean recognisers in this showcase, plus dozens more fine-tunes.

How the API works

Normally the one-liner is the ASR pipeline — but this repo ships no config.json and keeps its ONNX at the repo root, so we load the model directly (borrowing an XLSR-53 sibling's architecture config) and do the CTC collapse ourselves (which is exactly what the pipeline does internally): 

import { AutoConfig, AutoModelForCTC } from "@huggingface/transformers";

const model_id = "FinDIT-Studio/wav2vec2-large-xlsr-53-korean-onnx";
// Repo has no config.json → borrow a reachable XLSR-53 sibling's config (architecture only, no weights)
const config = await AutoConfig.from_pretrained("jonatasgrosman/wav2vec2-large-xlsr-53-spanish");
config.vocab_size = 1205;         // …then match the Korean CTC head (1205 Hangul syllables)
config.pad_token_id = 1204;       // [PAD] is the CTC blank

const model = await AutoModelForCTC.from_pretrained(model_id, {
  device: "wasm", dtype: "fp32",  // the only export is a single fp32 model.onnx…
  subfolder: "", config,          // …at the repo ROOT, not the usual onnx/ subfolder
});

// audio = 16 kHz mono Float32Array, normalised to zero-mean/unit-variance
const { logits } = await model({ input_values }); // shape [1, frames, 1205]

// greedy CTC decode: argmax per frame, merge duplicates, drop the blank (id 1204)
const ids  = argmaxPerFrame(logits);   // e.g. [1204,1204,26,26,1204,1140,...]
const text = collapse(ids);            // "가수…" in Hangul — no decoder involved

Inference runs in a Web Worker so the UI never janks. The worker returns the raw per-frame argmax (whole syllables) and real per-word timings (from the frame each syllable was emitted at) — nothing is faked or pre-computed.
