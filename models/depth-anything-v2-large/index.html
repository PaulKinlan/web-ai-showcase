Depth Anything V2 Large — the sharp, big-encoder depth model

Same task as the small Depth Anything V2 demo — estimate the distance to every pixel from one ordinary photo — but this is the ViT-Large checkpoint. The bigger encoder resolves thinner structures (wires, railings, hair, foliage) and gives cleaner depth edges, at the cost of a larger download and slower inference. It runs entirely on your device; the photo never leaves the tab.

Run it

Pick or drop a photo, then estimate depth. Warmer / brighter means closer; darker means further away. Switch the colour map to read the scene differently. On a WebGPU device this runs fp16; otherwise it uses the q8 WASM build — both real inference.

Large (ViT-L) vs Small (ViT-S) — the honest contrast

Both are the same Depth Anything V2 family, differing only in encoder size. The ViT-Large here (~340M params) produces sharper depth boundaries and separates thin, overlapping objects the ViT-Small (~25M params) smears together — most visible on fences, hair, plant stems, and where a foreground object meets a busy background. The trade-off is real: Large is roughly 4× the download and noticeably slower per image. Pick Small for speed and Apache licensing; pick Large when edge fidelity matters. Open the same photo in the small model to compare directly.

Why it matters

Depth used to need special hardware — a stereo rig, a LiDAR sensor, a structured-light camera. Depth Anything estimates it from a single frame. The Large checkpoint pushes the quality end of that trade-off: when the depth map feeds a downstream effect, sharper edges mean fewer halos and cleaner masks. Once you know how far away each pixel is, a whole class of effects opens up client-side:

  • Cleaner portrait-mode blur — sharper subject/background boundaries mean less blur bleeding onto hair and edges.
  • 3D parallax "live photos" — tilt a flat image and watch near objects shift more than far ones, with crisper separation.
  • AR occlusion & compositing — place a virtual object at the right depth, or pop a subject out in front of a scene, without ragged edges.
  • Relighting & fog — drive atmospheric effects and depth-of-field from a high-fidelity distance map.

How the API works

Depth is a one-liner with Transformers.js. The pipeline returns both a ready-to-display image and the raw tensor:

import { pipeline } from "@huggingface/transformers";

const estimator = await pipeline(
  "depth-estimation",
  "onnx-community/depth-anything-v2-large",
  { device: "webgpu", dtype: "fp16" },   // ~669 MB fp16 / ~347 MB q8, cached after first load
);

const { depth, predicted_depth } = await estimator(imageURL);
// depth          → RawImage: grayscale, min–max normalized to 0–255 (great for display)
// predicted_depth → Tensor:  the raw per-pixel relative depth (dims = [H, W]); bigger = nearer

This page colourises depth on a canvas and reads predicted_depth for the real range + distribution in "See inside". Inference runs in a Web Worker, so estimating a large photo never freezes the page.
