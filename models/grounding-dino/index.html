Grounding DINO — point at things by describing them in a sentence

Grounding DINO is a phrase-grounding detector: instead of a fixed label list, you give it a caption — a cat. a remote control. a laptop. — and it grounds each phrase to the regions that match, with a score. It fuses a Swin vision backbone with a BERT text encoder so the words and the image are compared in one model. New description, no retraining, on your device.

Why it matters

Because the prompt is language, Grounding DINO detects things a fixed detector never learned — and it reads referring phrases, not just nouns. "a person wearing red", "the empty shelf", "a dog on the left" are all valid prompts. That turns bespoke detection from a training project into a sentence, running client-side with nothing uploaded.

  • Describe-to-find inventory / inspection — ground "a forklift", "a missing bolt", or "an empty pallet" by phrase, no per-site model, nothing leaves the device.
  • Grounded captions & alt-text — tie each phrase of a description to the exact region it refers to, for accessible, verifiable image text.
  • Natural-language visual search — let users type what to find in their own photos ("my blue mug", "a bike") and box it locally.
  • Open-vocabulary region proposals — Grounding DINO finds "where"; hand each region to a segmenter or captioner for "what exactly", chaining tiny local models.

How the API works

Phrase grounding is a one-liner with Transformers.js — the caption's phrases are the argument:

import { pipeline } from "@huggingface/transformers";

const detect = await pipeline(
  "zero-shot-object-detection",
  "onnx-community/grounding-dino-tiny-ONNX",
); // downloads + caches ~205 MB once

// Each phrase should be lowercased and end with a period.
const phrases = ["a cat.", "a remote control.", "a person wearing red."];
const output = await detect(imageURL, phrases, { threshold: 0.3 });
// → [{ label: "a cat", score: 0.45, box: { xmin, ymin, xmax, ymax } }, …]
//   label is the matched phrase; boxes are in image pixel coordinates

This page runs the model once at a low floor threshold inside a Web Worker, caches every box, and filters the cached list as you drag the slider — so the slider is instant and never re-runs inference. Changing the caption re-runs the model (new text embeddings). The canvas is drawn at the image's natural resolution and scaled by CSS, so boxes stay pixel-accurate.
