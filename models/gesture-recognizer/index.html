GestureRecognizer — name the gesture

MediaPipe's GestureRecognizer finds a hand and classifies it into one of eight built-in gestures — 👍 Thumb_Up, 👎 Thumb_Down, ✌️ Victory, ✊ Closed_Fist, ✋ Open_Palm, ☝️ Pointing_Up, 🤟 ILoveYou, or None — with a confidence score, and hands you the 21 hand landmarks too. It's a real downloadable .task model that runs on-device via WebAssembly (with a GPU delegate when available). Drop a photo or turn on your webcam.

Why it matters

A named gesture is a ready-made button. GestureRecognizer turns "what is the hand doing?" into a single local call — no training data, no depth camera, no cloud. Eight canonical gestures cover a surprising amount of UI: yes/no, next/back, select, stop.

  • Touchless controls — 👍 confirm, ✊ pause, ✋ stop, ☝️ select: a remote you never touch.
  • Presentations & kiosks — advance slides or trigger actions hands-free, hygienic and cable-free.
  • Games & play — rock-paper-scissors, reaction games, or emoji reactions on a video call.
  • Accessibility — map a comfortable gesture to any action for hands-busy or motor-limited use.

How the API works

MediaPipe Tasks loads a .task bundle and gives you a recognizer you call per image or per video frame:

import { FilesetResolver, GestureRecognizer } from "@mediapipe/tasks-vision";

const vision = await FilesetResolver.forVisionTasks(".../wasm");
const rec = await GestureRecognizer.createFromOptions(vision, {
  baseOptions: {
    modelAssetPath: ".../gesture_recognizer.task",  // ~8 MB
    delegate: "GPU",                                 // or "CPU"
  },
  numHands: 1,
  runningMode: "IMAGE",                              // or "VIDEO"
});

const result = rec.recognize(imageElement);
// result.gestures  → [ [ { categoryName:"Thumb_Up", score } , … ] , … ]  (ranked per hand)
// result.landmarks → [ [ {x,y,z} × 21 ], … ]  (normalized 0–1)
// result.handedness → [ [ { categoryName:"Right", score } ], … ]

This page loads the model through the showcase's shared auto-init loader and draws the landmarks on a canvas; for the webcam it switches the task to VIDEO mode and calls recognizeForVideo each frame.
