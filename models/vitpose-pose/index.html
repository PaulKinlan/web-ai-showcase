ViTPose — a plain transformer that finds 17 body keypoints

ViTPose skips the hand-built pose architecture: it's a plain Vision Transformer with a tiny decoder head. Give it a person and it emits one heatmap per keypoint — the joint is simply the hottest cell. This is a top-down estimator (a different design from the MediaPipe PoseLandmarker): feed it a person box, get back 17 COCO keypoints and a skeleton, all on your device.

See inside — heatmap → arg-max

ViTPose doesn't regress coordinates directly. For each of the 17 keypoints it outputs a small 64×48 heatmap — a probability surface over the person box — and the keypoint is the arg-max (the single hottest cell), rescaled back to image pixels. The confidence is that peak value. Pick a joint to see its real heatmap and where the peak landed.

Why it matters

Pose estimation used to mean a depth camera or a cloud vision API. ViTPose shows that a plain transformer — the same ViT family behind image classification — is a state-of-the-art pose estimator with almost no task-specific machinery. That makes "where is this person's body?" a local function call, private and offline.

  • Fitness & physio — check squat depth, plank alignment, or arm angle from joint coordinates, with nothing uploaded.
  • Sports & dance analysis — compare a learner's joints to a target and score the match, frame by frame.
  • Ergonomics & safety — flag slouching or unsafe lifting posture from shoulder/hip/knee angles, on-device.
  • Crowd pose — pair a person detector with ViTPose to estimate everyone in a scene, each from their own crop.

Use cases

Basics

Pose a person and read all 17 COCO keypoints, confidences, and the box ViTPose worked from.

Practical

Form & posture analysis — compute joint angles (elbow, knee, back) and get plain-language feedback.

Wild

A stick-figure puppet: drop a photo and drive a clean skeleton avatar from the detected joints.

Multi-model

Detector → ViTPose: find every person with an object detector, then estimate each one's pose from its own crop.

How the API works

ViTPose isn't a one-line pipeline — it's a model + image processor. You pass a person box; the processor crops and normalises, the model emits heatmaps, and the processor decodes them into keypoints:

import { AutoModel, AutoImageProcessor, RawImage } from "@huggingface/transformers";

const model = await AutoModel.from_pretrained(
  "onnx-community/vitpose-base-simple", { dtype: "q8" });        // ~87 MB, cached once
const processor = await AutoImageProcessor.from_pretrained(
  "onnx-community/vitpose-base-simple");

const image = await RawImage.read(url);
const boxes = [[[0, 0, image.width, image.height]]];             // [image][person][x,y,w,h]
const inputs = await processor(image, { boxes });
const { heatmaps } = await model(inputs);                         // [1, 17, 64, 48]
const [person] = processor.post_process_pose_estimation(heatmaps, boxes)[0];
// person.keypoints → [[x,y] × 17] in image pixels · person.scores → 17 confidences

This page runs exactly that in a Web Worker, and keeps the raw heatmaps tensor so "See inside" can show the arg-max decode. For multiple people, pass one box per person (see the multi-model demo).
