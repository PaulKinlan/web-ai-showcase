Practical: fast on-device RAG rerank

In retrieval-augmented generation, you fetch a shortlist of chunks, then rerank them so the LLM reads the genuinely best one. That rerank is a cross-encoder pass over every (query, chunk) pair — usually a server cost. Here it runs on-device. Rerank a support knowledge base below and watch the real timing: total latency, per-pair time, and tokens processed. Add passages to see how the cost scales.
