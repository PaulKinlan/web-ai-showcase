Practical: on-device tool routing

The move that turns a chat model into an agent: given a set of tools and a user request, decide which tool to call and produce a valid call for it. Phi-4-mini is specifically trained for function calling, so it's good at this — and here it runs entirely on your GPU, emitting strict JSON (WebLLM's JSON mode) that your app could dispatch to a real API. Nothing leaves the device.

How tool routing works

      

        The tools are described in the system prompt, and the model is asked to return a single JSON object
        { "tool": "…", "arguments": { … } }. Passing
        response_format: { type: "json_object" } constrains WebLLM's sampler so the output is
        always valid JSON — no stray prose around it. The page then JSON.parses the result and
        checks the chosen tool exists and its arguments match the declared parameters. In a real app you'd
        take that object and actually call the function; here we just show it, so you can see the model's
        routing decision on-device.
