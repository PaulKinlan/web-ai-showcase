Transcribe speech, then clean it up

Speech-to-text output is often the perfect input for a grammar model: raw transcripts run on, drop punctuation, and keep disfluencies ("um", "like", "you know"). Here two on-device models chain up — Moonshine transcribes an audio clip (or your mic), then the grammar model rewrites the raw transcript into clean prose. Why chain them

An ASR model optimises for what was said, not for readability — so its output is great data but poor prose. A grammar model is the natural second stage: it adds punctuation, fixes agreement, and trims disfluencies without a cloud round-trip. The same pattern extends to a third stage (text-to-speech) to read the cleaned text back.
