Math-formula OCR — a picture of an equation → LaTeX

Paste, drop or upload an image of a mathematical equation and the model reads it and writes the LaTeX that produces it — fractions, integrals, sums, roots, super/subscripts and all. It's OCR-free end to end: a vision encoder looks at the whole formula and a decoder generates the markup. Everything runs on your device — the image never leaves the tab.

Why it matters

Math is hard to search, edit or read aloud when it's trapped in a picture. Turning an equation image back into LaTeX — on-device — means you can lift formulas out of textbooks, papers, whiteboards or screenshots without uploading anything.

  • Digitise notes & textbooks — photograph an equation and paste editable LaTeX into your document.
  • Accessibility — recover LaTeX/MathML so a screen reader can speak the maths.
  • Q&A & search — make a formula searchable or feed it to a solver, from just an image.
  • Authoring tools — a "paste a photo of an equation" shortcut inside an editor.

It's a best-effort recogniser for a single, reasonably clean equation: it nails structure (fractions, integrals, sums, roots) but can slip on an ambiguous character (a stylised E vs H) or very dense/handwritten input. Treat the LaTeX as a strong first draft — the re-typeset preview below makes any slip easy to spot and fix.

How the API works

It's one image-to-text call — the model generates the LaTeX autoregressively:

import { pipeline } from "@huggingface/transformers";

const texify = await pipeline("image-to-text", "Xenova/texify");
const out = await texify(equationImage, { max_new_tokens: 256 });
// → [{ generated_text: "$$x^{2}+y^{2}=z^{2}$$" }]

A Donut-Swin encoder embeds the whole equation image; an mBART decoder generates LaTeX tokens with a KV-cache (merged decoder). It all runs in a Web Worker so the page stays responsive while it decodes.
