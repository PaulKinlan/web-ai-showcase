Qwen3-Embedding 0.6B — instruction-aware, multilingual retrieval, on your CPU

Most embedders are BERT encoders that mean-pool their tokens. Qwen3-Embedding is different: it's a 0.6B Qwen3 decoder LLM turned into an embedder. It reads the sentence with causal attention and takes the last token's hidden state as the vector, it accepts a one-line instruction per query to steer the objective, and it embeds 100+ languages into one shared space. It sits at the top of the MTEB multilingual board — and it runs here on the CPU in your browser. Nothing is uploaded; no key, no server.

See inside — the instruction changes the vector

This is the model's signature move. A query is wrapped as Instruct: {task}\nQuery:{query}; a document is embedded raw. The same query text, embedded with vs. without the instruction, produces a different vector — and for a genuine query→passage pair the instruction should raise the cosine to the right passage. Why it matters

Retrieval quality is the ceiling on every RAG system, semantic search box, and dedup pipeline. A top-tier multilingual embedder that runs on the client means your search index, your document clustering, and your "find similar" features can be private, offline-capable, and free to run — with no embedding API bill and no data leaving the device.

  • Instruction steering — one embedder, many objectives: search, classification, code, STS — just change the instruction.
  • Cross-lingual by default — a query in one language finds passages in another, in the same vector space.
  • Right-size the index — Matryoshka truncation trades dimensions for storage without re-embedding.
  • On-device RAG — pair it with a local LLM (see the multi-model demo) for retrieval-augmented answers with zero server.

How the API works

It's the standard feature-extraction pipeline with two Qwen3-specific choices: pooling: "last_token" and the instruction wrapper on queries only. 

import { pipeline } from "@huggingface/transformers";

const extractor = await pipeline(
  "feature-extraction",
  "onnx-community/Qwen3-Embedding-0.6B-ONNX",
  { device: "wasm", dtype: "q8" },   // CPU / WebAssembly — no WebGPU
);

const task = "Given a web search query, retrieve relevant passages that answer the query";
const detailed = (q) => `Instruct: ${task}\nQuery:${q}`;

// Queries get the instruction; documents are embedded raw.
const input = [ detailed("What is the capital of China?"), "The capital of China is Beijing." ];

const out = await extractor(input, { pooling: "last_token", normalize: true });
// out.data holds the 1024-d unit vectors → cosine = dot product.

Everything runs in a Web Worker so the CPU-bound embedding never blocks the page; the shared loader auto-inits an already-cached copy and never silently re-downloads.
