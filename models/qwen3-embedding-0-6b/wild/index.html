One query, many languages, one vector space

Qwen3-Embedding maps 100+ languages into a single shared embedding space. So a query written in one language lands right next to the passages that answer it — no matter what language they are written in. Ask in English and watch the model surface the matching sentence in French, Spanish, German, or Japanese, ranked by meaning, not by words.

Why this works

During training the model saw parallel and mixed-language data, so semantically equivalent sentences — regardless of language — are pulled close together. Retrieval is just cosine similarity in that shared space; the language barrier disappears because the vectors, not the words, are what's compared. This is the backbone of multilingual search, deduplication across locales, and "find this in any language" features — all running locally.
