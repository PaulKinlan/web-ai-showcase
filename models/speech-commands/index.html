Speech Commands — spot a spoken keyword on-device

Wake-words and voice control don't need the cloud. This is an Audio Spectrogram Transformer — the same model that names 527 general sounds — but fine-tuned on Google Speech Commands v2 to do one narrow thing extremely well: recognise 35 short spoken keywords like yes, no, up, down, left, right, on, off, stop, go and the digits. Say a word into your mic and it tells you which command it heard, with a confidence — entirely on your device.

See inside — the short-utterance spectrogram

The model doesn't hear a waveform — it looks at a picture of it. Each ~1-second command becomes a log-mel spectrogram: time runs left→right, frequency runs bottom→top, brightness is energy. A spoken word is a compact shape — the sweep of a vowel, the burst of a consonant — and the transformer attends over that little image to pick a keyword. Why it matters

Keyword spotting is the piece of voice UI that has to be always-on, private, and cheap — you can't stream every second of mic audio to a server just to notice someone said "stop". A tiny on-device classifier makes hands-free control a local function call: no key, no bill, no audio leaving the page.

  • Hands-free control — drive a page, a kiosk, or a game with a handful of spoken words while your hands are busy or messy.
  • Wake-word gating — a cheap always-listening keyword detector that only wakes a heavier assistant (or an ASR model) when it hears its trigger.
  • Accessibility — a simple, reliable voice layer for users who can't easily use a keyboard or touch, running offline.
  • Voice-controlled hardware demos — WebUSB/WebBluetooth toys that respond to "on"/"off"/"up"/"down" with no phone app and no network.

How the API works

Recognition is one call with Transformers.js. Feed a 16 kHz mono Float32Array:

import { pipeline } from "@huggingface/transformers";

const recognise = await pipeline(
  "audio-classification",
  "Xenova/ast-finetuned-speech-commands-v2",
  { dtype: "q8" },              // downloads + caches ~83 MB once
);

const output = await recognise(audio, { top_k: 5 });
// audio = Float32Array @ 16 kHz mono, ~1 s (decode with the Web Audio API)
// → [{ label: "stop", score: 0.99 }, { label: "up", score: 0.004 }, … ]

This page decodes files / mic recordings to 16 kHz mono with an OfflineAudioContext, runs the pipeline inside a Web Worker so the UI never janks, and computes the log-mel spectrogram you see — the same representation the model classifies.
