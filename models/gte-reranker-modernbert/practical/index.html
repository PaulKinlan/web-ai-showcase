Practical: two-stage retrieval for RAG

The pattern that powers production search and RAG: a cheap first stage returns a shortlist, then the cross-encoder reranks it for real precision. Here the knowledge base has some long passages — the kind a 512-token reranker would have to truncate — and ModernBERT's 8192-token context reads them whole to find the one that answers your query.
