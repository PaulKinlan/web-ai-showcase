Multilingual MiniLM — one meaning space for 50+ languages

Most embedders on this site speak English. This one is cross-lingual: it was distilled so that a sentence and its translation land at almost the same point in a shared 384-dimensional space. "The weather is nice today", "Il fait beau aujourd'hui" and "今日はいい天気です" get near-identical vectors — so cosine similarity measures meaning, not language. Type lines in any mix of languages to build a cross-language similarity matrix, then search with an English query and watch it surface the matching French or Japanese passage above an unrelated English one. Nothing leaves your device.

See inside — the vector, and cross-lingual alignment

The encoder emits one 384-number vector per token; the model mean-pools them (mask-aware) and L2-normalizes, so every sentence becomes a unit-length arrow and cosine similarity is a plain dot product. Pick a sentence to see its real vector and pre-normalization magnitude.

Alignment — the same sentence, N languages

Type one sentence and translate it into a few languages (edit the lines). All the vectors should be mutually close — a bright, uniform matrix — because they all mean the same thing. A monolingual embedder would score these near zero; this one keeps them together. The last "distractor" line means something different and should stay dark.

Why it matters

A cross-lingual embedder collapses the "one index per language" problem into a single vector space. You embed everything — questions, documents, product reviews, support tickets — into the same 384-d space regardless of language, and nearest-neighbour search just works across the whole multilingual corpus. No translation round-trip, no per-language model, and the text never leaves the device.

  • Cross-language search — an English query retrieves the best answer even if it was written in French, Spanish or Japanese.
  • Multilingual dedup — collapse "same content, different language" reviews, tickets or docs into one.
  • Language-agnostic clustering — group text by topic across a mixed-language corpus, not by which language it happens to be in.
  • Zero-shot classification in any language — embed your English label set once, then tag incoming text in any of 50+ languages by nearest label.

How the API works

Same feature-extraction recipe as any sentence embedder — the difference is entirely in the weights:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Xenova/paraphrase-multilingual-MiniLM-L12-v2",
); // downloads + caches ~118 MB once

// Mean pooling, no instruction prefix — embed a query and a doc IN ANY LANGUAGE the same way:
const out = await extract(["I can't sign in", "¿Cómo restablezco mi contraseña?"], {
  pooling: "mean",
  normalize: true,
});
const [q, d] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = q.reduce((s, x, i) => s + x * d[i], 0); // high — same meaning, different languages

This page asks the worker for the un-normalized pooled vectors so "See inside" can show the real pre-normalization magnitude, then normalizes them in JS. Inference runs in a Web Worker so the matrix, projection and search never jank the page.
