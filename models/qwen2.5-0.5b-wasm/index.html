Qwen2.5 0.5B on the CPU — a capable instruct LLM in pure WebAssembly

No GPU, no problem. This runs the 8-bit build of Qwen2.5-0.5B-Instruct on the WebAssembly backend — pure CPU — to make one point: a genuinely useful instruct LLM now fits inside a browser tab on ordinary hardware. Chat with it, steer it with a system prompt and sampling controls, and watch the real probability distribution it samples each token from. Nothing is uploaded; there's no API key and no server.

Want the fast path? The same weights run on WebGPU on the Qwen2.5 0.5B (WebGPU) page. This page is deliberately the CPU story.

Why it matters

WebGPU is still uneven across devices and browsers — but every browser has a fast WebAssembly engine. Proving a capable instruct model runs on plain CPU means client-side language becomes a universal primitive: it works on the locked-down work laptop, the older phone, the kiosk with the GPU blocklisted. Private, offline-capable, free to run, no cold start once cached.

  • Runs everywhere — no GPU adapter, no driver flags; if the browser runs WASM, it runs.
  • Structured extraction — turn messy text into clean JSON for a form or API, on-device.
  • On-device agents — classify, route, and summarise inside the page with zero per-call cost.
  • Offline & edge — field apps and privacy-first tools that work with no network.

How the API works

The only difference from the GPU path is two options: device: "wasm" and dtype: "q8". Everything else is the standard text-generation pipeline — pass a messages array (the pipeline applies the chat template), a TextStreamer for token-by-token output, and your sampling settings: 

import { pipeline, TextStreamer } from "@huggingface/transformers";

const generator = await pipeline(
  "text-generation",
  "onnx-community/Qwen2.5-0.5B-Instruct",
  { device: "wasm", dtype: "q8" },   // CPU / WebAssembly — no WebGPU
);

const messages = [
  { role: "system", content: "You are a helpful, concise assistant." },
  { role: "user", content: "Explain WebAssembly in one sentence." },
];

const streamer = new TextStreamer(generator.tokenizer,
  { skip_prompt: true, skip_special_tokens: true, callback_function: (t) => append(t) });

await generator(messages, {
  max_new_tokens: 200, do_sample: true, temperature: 0.7, top_p: 0.9, streamer,
});

The "see inside" panel runs one raw forward pass — generator.model(generator.tokenizer(prompt)) — reads the last position's logits, softmaxes them, and shows the real top-k. Everything runs in a Web Worker so the CPU-bound generation never blocks the page; the loader auto-inits an already-cached copy and never silently re-downloads.
