GTE-ModernBERT — a long-context embedder on a newer backbone

Most in-browser embedders (GTE-small, MiniLM, BGE, E5) sit on the classic BERT encoder with its 512-token ceiling. This one is built on ModernBERT — a from-scratch redesign of the encoder (rotary positions, alternating local/global attention, an 8192-token context) — fine-tuned by Alibaba's GTE team into a 768-dimensional retriever that tops MTEB for its class. It also pools differently: the [CLS] token, not the mean. Paste a few passages to build a live similarity matrix, search by meaning, and inspect the vector. Nothing leaves your device.

Why it matters

Embeddings power search, recommendations, deduplication, clustering and retrieval-augmented generation. GTE-ModernBERT's pitch is long context on a modern, efficient backbone: you can embed a whole document — a full support article, a contract clause, a long chat turn — without chopping it into 512-token windows and losing the thread, and get top-tier retrieval quality, all in a tab and all private.

  • Long-document retrieval — index whole articles or chapters as single vectors, no window-stitching.
  • RAG over long chunks — fewer, longer chunks means fewer embeddings and less lost context.
  • Dedup & clustering — group "same thing, different words" across long passages.
  • Retrieve → rerank pipelines — a fast long-context first pass, then a cross-encoder for precision (see the multi-model demo).

Use cases

Basics

Do two passages mean the same thing? One cosine number — and the long-context angle.

Practical

Long-document semantic search and near-duplicate detection over your own passages.

Wild

Cluster a pile of mixed passages into themes with k-means over the 768-d vectors.

Multi-model

GTE-ModernBERT retrieves a shortlist, then a cross-encoder reranks the top hits.

How the API works

Feature-extraction with CLS pooling is the whole recipe — note it is cls, not mean, and there is no prefix:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "Alibaba-NLP/gte-modernbert-base",
); // downloads + caches ~150 MB once

// GTE-ModernBERT pools the [CLS] token and needs NO instruction prefix — up to 8192 tokens per input:
const out = await extract([
  "how do neural networks pay attention?",
  "Attention lets a model focus on the most relevant parts of the input…",
], { pooling: "cls", normalize: true });
const [q, d] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = q.reduce((s, x, i) => s + x * d[i], 0); // high — same meaning

This page asks the worker for the un-normalized CLS vectors so "See inside" can show the real pre-normalization magnitude, then normalizes them in JS. Inference runs in a Web Worker, so the matrix, projection and comparison never jank the page.
