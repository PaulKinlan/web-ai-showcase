AST — name the sound in any audio clip

The Audio Spectrogram Transformer turns a sound into a 128-band log-mel spectrogram and reads that spectrogram like an image — the same trick a Vision Transformer plays on photos. Give it a clip and it returns the most likely sounds from 527 AudioSet classes — speech, a dog, a siren, applause, a guitar — entirely on your device.

See inside

AST doesn't hear a waveform — it looks at a picture of it. Every clip becomes a log-mel spectrogram: time runs left→right, frequency runs bottom→top (low to high), and brightness is energy. The transformer cuts that image into patches and attends over them, exactly like a Vision Transformer over an image. Why it matters

Sound is a channel most apps ignore because "understanding audio" used to mean a cloud API and an upload. AST makes it a local function call — private, offline, free per call — so the microphone becomes a sensor your page can actually reason about.

  • Accessibility sound alerts — flash the screen when a smoke alarm, doorbell, or baby cry is heard, for Deaf and hard-of-hearing users, with no audio leaving the device.
  • Ambient monitoring — a shop or workshop dashboard that notices glass breaking, a machine whining, or a dog barking, on-device.
  • Auto-tagging & search — label a library of recordings ("music", "speech", "traffic") so people can filter without listening to each one.
  • Context-aware UX — a page that knows it's noisy, or that music is playing, and adapts — no key, no bill.

How the API works

Classification is one call with Transformers.js. Feed a 16 kHz mono Float32Array:

import { pipeline } from "@huggingface/transformers";

const classify = await pipeline(
  "audio-classification",
  "Xenova/ast-finetuned-audioset-10-10-0.4593",
  { dtype: "q8" },              // downloads + caches ~90 MB once
);

const output = await classify(audio, { top_k: 5 });
// audio = Float32Array @ 16 kHz mono (decode with the Web Audio API)
// → [{ label: "Speech", score: 0.71 }, { label: "Music", score: 0.06 }, … ]

This page decodes files / mic recordings to 16 kHz mono with an OfflineAudioContext, runs the pipeline inside a Web Worker so the UI never janks, and computes the log-mel spectrogram you see — the same representation AST classifies.
