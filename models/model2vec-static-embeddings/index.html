Model2Vec — embeddings with no transformer at inference

Most text embedders run a neural network for every sentence. Model2Vec does not. It is a distilled lookup table: each token has one precomputed vector, and to embed a sentence you just gather those vectors and average them. No attention, no layers, no forward pass — so it does a tiny fraction of the work of an encoder like MiniLM (often 10–100× faster), and the model itself costs almost nothing; the little wall-clock you see is mostly tokenization. Type below and watch meaning become geometry, instantly. Nothing leaves your device.

Why it matters

Embeddings power search, dedup, clustering, recommendations, and retrieval for LLMs. The usual cost is running a transformer per item — fine for a few sentences, painful for millions or on a phone. Model2Vec collapses that to a lookup, so you can embed a huge corpus, or re-embed on every keystroke, on hardware that could never run an attention model at that rate. It trades a little quality for a lot of speed and a tiny footprint.

  • Edge & on-device search — index and query a large corpus in the browser with no server and no GPU.
  • Real-time, as-you-type — re-rank suggestions on every keystroke because embedding is effectively free.
  • Cheap prefilter — shortlist with Model2Vec, then spend a slow, accurate cross-encoder only on the top few.
  • Huge-batch pipelines — deduplicate or cluster millions of short texts where a transformer would be too slow.

How the API works

Model2Vec's ONNX is a torch EmbeddingBag (mean) — it needs input_ids and offsets, so we run it directly with ONNX Runtime Web and tokenize with Transformers.js:

import * as ort from "onnxruntime-web";
import { AutoTokenizer } from "@huggingface/transformers";

const tok = await AutoTokenizer.from_pretrained("minishlab/potion-base-8M");
const session = await ort.InferenceSession.create(onnxBytes); // onnx/model.onnx

// Bag every token of every sentence; offsets mark where each sentence starts.
const ids = [], offsets = [];
for (const text of texts) {
  offsets.push(ids.length);
  for (const id of tok(text, { add_special_tokens: false }).input_ids.data) ids.push(BigInt(id));
}
const out = await session.run({
  input_ids: new ort.Tensor("int64", BigInt64Array.from(ids), [ids.length]),
  offsets:   new ort.Tensor("int64", BigInt64Array.from(offsets.map(BigInt)), [offsets.length]),
});
// out.embeddings → [batch, 256], already L2-normalized → cosine is a dot product.

No pipeline() forward pass, no attention. The "model" is the embedding table baked into that ONNX; inference is a gather and a mean. It runs in a Web Worker, and the ONNX is stored via the Cache API so it auto-initialises offline on return visits.
