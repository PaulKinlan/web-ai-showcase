Live semantic matching — no debounce

Normally you debounce an embedding search: wait until the user stops typing, because running a model per keystroke would jank. Model2Vec is fast enough — a couple of milliseconds per short query, off the main thread — that you don't have to. Type in the box and the candidate list below re-ranks by meaning on every keystroke, with the live embed time shown. Try typing "a small feline" and watch "kitten" climb.

backend –last embed –keystrokes 0

Candidates, re-ranked live

The candidate list is embedded once; each keystroke only embeds your (short) query and recomputes cosine — that is the microsecond part.
