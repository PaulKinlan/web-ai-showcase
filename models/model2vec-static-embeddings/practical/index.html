Semantic search where speed is the point

Paste a corpus — one item per line — and embed the whole thing at once. Then type a query and it is ranked by meaning, not keyword overlap. Because each item is just a lookup+average, you can index thousands of lines in the tab and re-query instantly, with no server and no vector database. This is the setting Model2Vec is built for: lots of text, tight latency, modest hardware.
