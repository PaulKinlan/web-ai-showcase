Kokoro — natural text-to-speech, on your device

Kokoro is a tiny (82M-parameter) speech model that sounds far bigger than it is. Type any text, choose from dozens of named voices, set the speaking speed, and hear it spoken — synthesized entirely in your browser. No API key, no per-character bill, nothing uploaded.

Why it matters

Cloud TTS is metered per character and sends your text to a third party. Kokoro turns high-quality narration into a free, private, offline feature of your page — the text never leaves the tab, and there's no bill no matter how much you generate.

  • Accessibility read-aloud — give any article, form, or message a natural voice, with no service to sign up for.
  • Narration & voiceover — turn scripts, docs, or notifications into audio clips, downloadable as WAV, entirely client-side.
  • Voice for agents & assistants — pair with an on-device LLM so a private assistant can actually speak back.
  • Localized, branded voices — dozens of voices across accents and genders, chosen at runtime, no per-voice licensing.

How the API works

Kokoro has its own tiny library (which wraps Transformers.js). Two calls: load, then generate.

import { KokoroTTS } from "kokoro-js";

const tts = await KokoroTTS.from_pretrained(
  "onnx-community/Kokoro-82M-v1.0-ONNX",
  { dtype: "q8", device: "webgpu" }, // WASM fallback when WebGPU is absent
);

const audio = await tts.generate("Hello from your browser.", {
  voice: "af_heart",  // one of tts.voices — dozens across accents/genders
  speed: 1,
});
// audio.audio         → Float32Array waveform
// audio.sampling_rate → 24000
// audio.toBlob()      → a WAV Blob you can play or download

This page runs from_pretrained and generate inside a Web Worker so synthesis never blocks the interface, encodes the returned Float32 samples to a WAV blob for the <audio> element, and draws the same samples as the waveform under "See inside". The voice list comes straight from tts.voices — nothing hard-coded.
