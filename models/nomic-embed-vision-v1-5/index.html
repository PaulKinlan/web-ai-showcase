Nomic multimodal embeddings — search images by meaning

Two aligned models project pictures and words into the same 768-number space, so a phrase and an image become comparable. Type a dog and the dog photo rises to the top; click an image and find the ones most like it. CLIP-style cross-modal search — from the Nomic family — with real embeddings you can see, all on your device.

Modelsnomic-ai/nomic-embed-vision-v1.5 (image) + nomic-ai/nomic-embed-text-v1.5 (text)

See inside

Every image and every phrase becomes one 768-number vector, unit-normalised so its length is 1. Similarity is the cosine — here a plain dot product — between the query vector and each image vector. Below is the query's vector as a colour fingerprint (warm = positive, blue = negative), then the exact cosine to each gallery image. Same gallery, different phrase → different ranking. That is the whole trick, and it is cross-modal: text and image share one space.

Why it matters

A shared image–text space turns "search" into "compare vectors." No fixed tags, no per-image API call — you embed once, then rank by meaning, in the browser, for free.

  • Natural-language photo search — find images by a phrase you type, over a private library, with nothing uploaded.
  • Visual similarity / dedup — "more like this" recommendations and near-duplicate detection from image embeddings alone.
  • Zero-shot tagging & moderation — score images against label phrases you invent at runtime (like CLIP), no training.
  • On-device RAG for images — build a local vector index of a photo set and retrieve by text, offline and private.

Use cases

Basics

Type a phrase and watch the gallery re-rank by meaning — feel cross-modal search first-hand.

Practical

Zero-shot image tagging: score any image against label phrases you type, no training.

Wild

Multi-model

How the API works

Both towers load with the low-level Transformers.js classes and share a 768-d space:

import { AutoModel, AutoProcessor, AutoTokenizer, RawImage } from "@huggingface/transformers";

const proc  = await AutoProcessor.from_pretrained("nomic-ai/nomic-embed-vision-v1.5");
const vis   = await AutoModel.from_pretrained("nomic-ai/nomic-embed-vision-v1.5", { dtype: "q8" });
const tok   = await AutoTokenizer.from_pretrained("nomic-ai/nomic-embed-text-v1.5");
const txt   = await AutoModel.from_pretrained("nomic-ai/nomic-embed-text-v1.5", { dtype: "q8" });

// image → 768-d (CLS token of last_hidden_state)
const img = await (await vis(await proc(await RawImage.read(url)))).last_hidden_state;
// text  → 768-d (mean-pooled), prefixed "search_query: " for queries
const q   = await txt(tok("search_query: a dog", { padding: true }));
// cosine of the L2-normalised vectors = similarity

This page runs both towers in a Web Worker and L2-normalises every vector there, so ranking the gallery on the main thread is a handful of dot products. Nothing is uploaded.
