reranker (text-classification) · text · WASM · ~571 MB

BGE-reranker-v2-m3 — rank passages by relevance, across 100+ languages

BAAI's second-generation multilingual cross-encoder. Where the original bge-reranker-base used an XLM-RoBERTa-base backbone, v2-m3 is built on the larger M3 backbone (XLM-RoBERTa-large, trained across 100+ languages) — a stronger, broader reranker. Instead of turning a query and a passage into separate vectors and comparing them, it reads both together and outputs a single relevance score, so it ranks a Chinese or Spanish passage against an English query as easily as an English one. Give it a query and candidates and watch it re-sort them; then look inside at the raw logit per pair. Nothing leaves your device.

Run it

Type a query and one candidate passage per line. The model scores each passage against the query and re-sorts them by relevance logit (higher = more relevant). The default set mixes English, Chinese and Spanish so you can see cross-language ranking.

Why it matters

Rerankers are the precision stage of modern search. A fast retriever (a keyword index or embeddings) casts a wide net and returns, say, the top 50 candidates; a cross-encoder then reads each candidate against the query and reorders them with far higher accuracy. BGE-reranker-v2-m3 pairs that precision with the widest language reach in the family — one model that ranks relevance across 100+ languages, so a query in one language surfaces the best answer written in another. At ~571 MB it runs in a tab, so the precise, expensive step happens on-device with nothing sent anywhere.

  • Global multilingual RAG — retrieve across a mixed-language corpus, then rerank so the LLM reads the genuinely best chunk whatever language it's in.
  • Cross-language site search — one reranker serves every locale instead of a model per language, with the larger M3 backbone's accuracy.
  • Answer selection — pick the best reply from candidate answers, even when they're phrased very differently from the question.
  • Upgrade path — a drop-in precision boost over v1 for products that outgrew bge-reranker-base's coverage.

Use cases

Basics

Rank a handful of passages against a query by cross-encoder relevance.

Practical

Two-stage RAG: retrieve a multilingual shortlist, then rerank with v2-m3.

Wild

Cross-lingual hard negatives: distractors that fool keyword and embedding search across languages.

Multi-model

Multilingual E5 embeddings retrieve, BGE-reranker-v2-m3 reorders — the classic retrieve-then-rerank.

Contrast with the first-generation BGE-reranker-base: same cross-encoder idea, but v2-m3 uses the larger M3 backbone for stronger accuracy and broader language coverage — at roughly double the download.

How the API works

A cross-encoder scores query–passage pairs; we read the raw logit off the model directly:

import { AutoTokenizer, AutoModelForSequenceClassification } from "@huggingface/transformers";

const model_id = "onnx-community/bge-reranker-v2-m3-ONNX";
const tokenizer = await AutoTokenizer.from_pretrained(model_id);
const model = await AutoModelForSequenceClassification.from_pretrained(model_id, { dtype: "q8" });

// Encode the query with each passage AS A PAIR — [CLS] query [SEP] passage [SEP]:
const inputs = tokenizer(
  passages.map(() => query),
  { text_pair: passages, padding: true, truncation: true },
);
const { logits } = await model(inputs); // one relevance logit per pair — higher = more relevant
// sort the passages by logits.data — that's your reranking (works even when query and passage
// are in different languages, because the M3 backbone is multilingual XLM-RoBERTa-large)

This page loads the model through the shared loader as a text-classification pipeline, then reuses pipe.model + pipe.tokenizer to read the raw logits (the pipeline would squash them through a sigmoid). All inference runs in a Web Worker so ranking never janks the page.
