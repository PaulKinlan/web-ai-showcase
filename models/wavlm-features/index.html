WavLM — turning speech into features, not text

Every other speech model here hands you words. WavLM hands you numbers. It's a self-supervised speech model: for every ~20 ms frame of audio it emits a 768-dimensional embedding — a learned description of what the sound is, with no transcript and no labels. Those embeddings let you compare two clips, search a recording for a sound, and cluster audio by ear — entirely on this device.

BackendWebAssembly (WebGPU when a real adapter is present), honest fallback
Download~95 MB (single encoder), cached after first load

Why it matters

A transcript throws almost everything away — who spoke, how, in what language, over what noise. WavLM's self-supervised embeddings keep it. Because they were learned by predicting masked speech (even under simulated overlap and noise), a single 768-dim vector captures speaker, phonetic content and acoustic conditions at once — and you can do maths on it without any labelled training data of your own.

  • Speaker / voice matching — is this the same person? Compare pooled embeddings, no enrolment model needed.
  • Search audio by example — find every place a sound or phrase occurs in a long recording, by sliding a window and comparing.
  • De-dup and cluster a media library — group takes, callers, or sound effects by how they actually sound.
  • A backbone for fine-tuning — the same frames feed diarization, emotion, keyword-spotting and ASR heads (this is the model those tasks start from).

Use cases

Basics

Audio similarity: embed a bank of clips and read the full pairwise cosine matrix.

Practical

Find a sound: brush a query on a recording and search for where it recurs.

Wild

Cluster by ear: project every clip's embedding to 2D and watch voices group.

Multi-model

WavLM retrieves the nearest clip by sound → wav2vec2 reads out what it says.

How the API works

There's no pipeline for "give me the frames" — so we load the model directly and read last_hidden_state, then mean-pool + L2-normalise into one comparable vector: 

import { AutoProcessor, AutoModel } from "@huggingface/transformers";

const processor = await AutoProcessor.from_pretrained("Xenova/wavlm-base-plus");
const model = await AutoModel.from_pretrained(
  "Xenova/wavlm-base-plus",
  { device: "wasm", dtype: "q8" },
);

// audio = 16 kHz mono Float32Array
const inputs = await processor(audio);
const { last_hidden_state } = await model(inputs); // [1, frames, 768]

// one comparable vector per clip: average over time, then normalise
const emb = meanPool(last_hidden_state);           // 768-dim
const score = cosine(embA, embB);                  // -1 … 1

Inference and the heatmap composite both run in a Web Worker, and the RGBA heatmap is transferred back (not copied) so the UI never janks — nothing is faked or pre-computed.
