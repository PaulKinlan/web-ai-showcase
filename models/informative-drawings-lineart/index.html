Informative Drawings — turn a photo into a line drawing

Give this model any photo and it renders an artistic line drawing: clean contour lines that trace the scene's shapes and structure, ink on white paper. Informative Drawings (CVPR 2022) is a generator trained to draw lines that convey geometry and semantics — not a Sobel/Canny edge filter, but a learned artist's rendering of what matters in the picture. It's the kind of line/soft-edge extractor used to condition image generators (ControlNet), and it runs entirely on your device via ONNX Runtime Web; the photo never leaves the tab.

How to read it

Indigo = a line the model drew; the faded photo behind it shows what that line was tracing. Notice the lines land on meaningful boundaries — the edge of a building, the silhouette of a subject, the frame of a window — rather than on every patch of contrast, which is what makes it read as a drawing and not an edge map.

Why it matters

A clean line drawing of a photo is useful far beyond art — it's a compact, structure-only representation that both people and models can work with. On-device, it's private and free per image:

  • Creative & photo apps — a one-tap "turn my photo into a sketch / colouring page" that runs client-side, no server.
  • ControlNet & image generation — extract the line/soft-edge conditioning image locally before sending only the lines (not the photo) to a generator.
  • Design & CAD hand-off — trace a reference photo into clean contours to sketch over, vectorise, or use as a drawing guide.
  • Accessibility & printing — high-contrast line renderings for tactile graphics, colouring books, or low-ink printing.

Informative Drawings renders a plausible artistic line drawing; it's a stylised interpretation of the scene, not a precise geometric edge measurement.

How the API works

transformers.js has no informative-drawings class and no line-art task, so we run the ONNX graph directly with ONNX Runtime Web. In goes the photo, out comes a single-channel line map:

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(modelBytes, { executionProviders: ["wasm"] });

// photo → [1,3,H,W] float32 in [0,1] (plain RGB, no normalisation)
const input = new ort.Tensor("float32", toNCHW(photo, W, H), [1, 3, H, W]);
const { output } = await session.run({ input });

// output is [1,1,H,W] in [0,1]: ~1 = white paper, low = ink → render as a grayscale drawing
const drawing = lineMapToImage(output.data, W, H);

Inference and the whole output composite run in a Web Worker so the UI never blocks, and the ~17 MB model is fetched through Cache Storage, so a returning visit auto-initialises with no re-download.
