SmolLM3-3B — a hybrid thinking model on Transformers.js

SmolLM3 is Hugging Face's fully-open 3B model, and it can think out loud on demand: flip thinking on and it emits a <think>…</think> chain of reasoning before answering — you watch the trace stream live — or flip it off for a fast, direct reply. One model, two behaviours, toggled with SmolLM3's /think and /no_think modes (its chat template's enable_thinking switch). Unlike the WebLLM chat pages, this one runs on Transformers.js (the ONNX build), so you also see the exact templated prompt. WebGPU only — a ~2.1 GB download, cached after first load.

Why it matters

SmolLM3 is a fully-open 3B model — open weights, open data recipe — with a genuinely useful hybrid reasoner. Running it on Transformers.js puts that on the client: for easy prompts you turn thinking off and get a fast reply; for hard ones — multi-step maths, planning, careful extraction — you turn it on and the model works through the problem in a visible trace before committing. All private by construction, free to run, and available offline once cached.

  • Show your work — the reasoning trace is a first-class UI surface: audit how the model reached an answer, or hide it for a clean reply.
  • Spend compute only when it helps — a per-request thinking toggle lets you pay the reasoning cost on the prompts that need it and stay fast on the rest.
  • Fully-open, self-hostable — Apache-2.0 weights running on the standard Transformers.js stack: no proprietary runtime, no API key, no data leaving the device.
  • On-device RAG with reasoning — pair it with a local embedding model for grounded answers it can reason over (see the multi-model demo).

How the API works

Transformers.js runs the ONNX build through the standard pipeline("text-generation", …) path on the WebGPU backend. SmolLM3's hybrid mode is selected by the chat template's enable_thinking switch, so this page applies the template itself with that flag, then streams the continuation with a TextStreamer, splitting the <think> reasoning from the answer in worker.js: 

import { hasWebGPU } from "/web-ai-showcase/lib/webai.js";
import { pipeline, TextStreamer } from "@huggingface/transformers";  // 3.7.5

// A 3B model needs a GPU — gate on a real adapter (headless requestAdapter() → null).
if (!(await hasWebGPU())) { showNeedsWebGPU(); }   // never a faked reply
else {
  const gen = await pipeline("text-generation", "HuggingFaceTB/SmolLM3-3B-ONNX",
    { device: "webgpu", dtype: "q4f16", progress_callback: setProgress });

  const messages = [
    { role: "system", content: "You are a concise assistant." },
    { role: "user",   content: "Is 97 prime?" },
  ];
  // enable_thinking picks /think vs /no_think — we template it ourselves so it's inspectable.
  const prompt = gen.tokenizer.apply_chat_template(messages,
    { add_generation_prompt: true, tokenize: false, enable_thinking: true });

  const streamer = new TextStreamer(gen.tokenizer, { skip_prompt: true, callback_function: onPiece });
  await gen(prompt, { max_new_tokens: 512, do_sample: true, temperature: 0.6, top_p: 0.95, streamer });
  // worker.js splits the streamed … trace from the answer.
}

Everything here runs exactly that path in worker.js, off the main thread, and gates on a real WebGPU adapter first. No adapter → a labelled needs-WebGPU state, never a canned answer.
