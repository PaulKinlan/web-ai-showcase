Practical: reasoned text → structured JSON

A shippable LLM trick with a twist: extract clean, typed data from messy human text, and let the model reason first. Paste a note and SmolLM3 thinks through the fields in a <think> trace (toggleable), then returns strict JSON matching your schema — parsed and validated live. All on-device, so the source text (often sensitive) never leaves.

Why reason before extracting

Plain extraction fails on ambiguity — a date like "next Thursday", an implied severity, a field that's split across two sentences. Letting the model reason in a <think> trace first gives it room to resolve those before it commits to the JSON, and you can read exactly how it decided. Turn thinking off for speed when the text is clean. Either way the answer is on-device and machine-readable — drop it straight into a form or an API call.
