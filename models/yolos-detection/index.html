YOLOS-Tiny — fast object detection with a plain Vision Transformer

YOLOS ("You Only Look at One Sequence") strips detection back to a bare Vision Transformer: no CNN backbone, no anchors, no region proposals — just image patches plus a handful of learned "detection tokens," and out come labelled boxes. The tiny build is about a fifth the size of DETR and quick enough to run on a live camera, entirely on your device.

YOLOS-Tiny vs DETR — why you'd pick the small one

Both do the same job — 80 COCO classes, boxes + scores from one forward pass. They trade differently: DETR's ResNet-50 backbone is more accurate on small/cluttered objects; YOLOS-Tiny is a fraction of the download and fast enough to run per webcam frame. Why it matters

A capable detector that downloads in a few seconds and runs per frame changes what's practical in a web page. YOLOS-Tiny turns "what's in this camera feed, right now?" into a local function call — private, offline, free per call — without shipping a 160 MB model first.

  • Lightweight on-device counting — tally people in a queue, cars in a lot, or items on a shelf, with a model small enough to bundle into a PWA.
  • Live camera UX — react to what the webcam sees (is a person present? a phone? a cup?) at interactive frame rates on modest devices.
  • Cheap first pass — run YOLOS everywhere as a fast trigger, and only reach for a heavier detector when the stakes justify the bytes.
  • Edge & offline — kiosks, field tools, and flaky-network apps get real detection with nothing to upload.

How the API works

Detection is a one-liner with Transformers.js — YOLOS uses the exact same object-detection pipeline as DETR, so swapping models is a one-line change. The boxes come back in pixel coordinates:

import { pipeline } from "@huggingface/transformers";

const detect = await pipeline(
  "object-detection",
  "Xenova/yolos-tiny",
); // downloads + caches ~30 MB once

const output = await detect(imageURL, { threshold: 0.4, percentage: false });
// → [{ label: "cat", score: 0.96, box: { xmin, ymin, xmax, ymax } }, …]
//   percentage:false gives pixel coordinates; true gives 0–1 fractions

This page runs the model once at a low floor threshold (0.05) inside a Web Worker, caches every box, and filters the cached list as you drag the slider — so the slider is instant and never re-runs inference. The canvas is drawn at the image's natural resolution and scaled by CSS, so the boxes stay pixel-accurate.
