Prompt-injection detection (LLM guardrail)

The built text classifiers judge sentiment or toxicity. This one judges intent against an LLM: is a prompt a normal request, or an attempt to hijack the model — "ignore your instructions", jailbreaks, system-prompt extraction, role-play escapes? It's the input filter you put in front of an LLM app to catch adversarial prompts before they reach the model. Type a prompt or try an example. Runs entirely on your device.

A guardrail, not a fix

Injection detection is a filter: it flags prompts that try to override an application's instructions, so you can block or review them before they reach your LLM. It's a classifier, so it can miss novel attacks or occasionally over-flag an unusual-but-benign prompt — the confidence score shows how sure it is. Running it locally means user prompts never leave the device to be screened.
