CREPE — trace the pitch of any sound

Pitch is the note you'd hum back. CREPE answers one focused question, many times a second: what is the fundamental frequency right now? It's a small (~2 MB) convolutional network that reads audio in 1024-sample windows and returns a 360-bin pitch salience — the confidence-weighted peak is the frequency, in Hz and as a musical note. It's the engine behind tuners, vocal-pitch monitors, auto-tune, and melody transcription. Here it runs entirely on your device.

CREPE never outputs "the note is A4". For every window it outputs a 360-bin salience curve — how much energy sits at each of 360 pitch candidates from C1 to B7. The top map is that raw salience over time (brighter = higher activation); the white line is the f0 we read out of it, and the peak brightness at each moment is the confidence. Why it matters

Pitch tracking is small, fast infrastructure that a surprising amount of music and voice tech is built on — and it belongs on-device, because it's cheap and nobody needs to upload their singing to a server to get a note back.

  • Instrument & voice tuners — turn f0 into a note + cents-off readout with zero latency and no cloud round-trip.
  • Vocal-pitch training — show a singer their pitch against a target in real time; score how close they held a note.
  • Melody transcription & search — hum a tune and turn the contour into notes you can search or notate ("query by humming").
  • Pitch-aware audio tools — auto-tune, pitch-correction, expressive-performance analysis, and prosody features for speech, all computed locally.

Use cases

Basics

Play or hum a single note and read the pitch back — Hz, the nearest note, and how many cents off.

Practical

A live chromatic tuner: sing or play into your mic and watch the needle settle on the note.

Wild

Hum a melody and CREPE traces the contour, then snaps it to notes — query-by-humming.

Multi-model

CREPE + Whisper: transcribe the words and track the pitch of speech — prosody, side by side.

How the API works

There's no pipeline() for pitch — you drive onnxruntime-web directly. Frame the audio into per-frame-normalised 1024-sample windows, run the CNN, and read the fundamental out of the 360-bin salience.

import * as ort from "onnxruntime-web";

const session = await ort.InferenceSession.create(
  "https://huggingface.co/niobures/CREPE/resolve/main/models/onnx/crepe_onnx_tiny.onnx",
  { executionProviders: ["wasm"] },
);

// One 1024-sample window @16 kHz, normalised (subtract mean, divide by std):
const frame = normalise(win1024);
const out = await session.run({ frames: new ort.Tensor("float32", frame, [1, 1024]) });
const act = out.probabilities.data;                 // 360 salience bins, C1…B7, 20 cents apart

// f0 = confidence-weighted centre of the peak, in cents → Hz:
const peak = argmax(act);
const cents = weightedCents(act, peak);             // avg over ±4 bins around the peak
const f0 = 10 * 2 ** (cents / 1200);                // Hz
const confidence = act[peak];                       // 0…1 voicing confidence

This page decodes files / mic recordings to 16 kHz mono with an OfflineAudioContext, batches the windows and runs them inside a Web Worker (so the UI never janks), caches the ~2 MB model in Cache Storage (return visits auto-initialise with no download), and converts each f0 to the nearest equal-tempered note plus a cents deviation.
