question-answering · text · WASM · ~295 MB

Multilingual QA — find the answer inside a passage, in any language

This is extractive question answering with one twist: the model is XLM-RoBERTa, pretrained on 100 languages. Give it a passage and a question — in French, German, Spanish, Japanese, Arabic, English… — and instead of writing a new sentence it points at the exact span of the passage that answers you. No translation step, no per-language model. Pick a sample below and watch it highlight the answer, then look inside at the real start/end token probabilities it used. Nothing leaves your device.

See inside — the start/end token probabilities

The model never "reads" an answer out — it outputs two numbers per passage token: how likely that token is the first word of the answer, and how likely it is the last. The answer is the span from the best start to the best end. Because XLM-R uses a SentencePiece tokenizer, a "word" is often several subword pieces (the ▁ marks word boundaries) — the strip shows the real softmaxed distribution over the passage tokens for your last question, with the chosen start/end tokens outlined.

Why it matters

Most "answer from my documents" tooling is English-first. A single multilingual extractive model collapses that: one 295 MB download answers questions over passages in dozens of languages, and because it returns a span of the source rather than inventing text, the answer is always grounded — it can't hallucinate a fact that isn't in the passage. It runs in a tab, so multilingual documents, contracts, or support tickets can be queried without any of it leaving the device.

  • Multilingual FAQ & help-centre search — one model over content in every language you ship.
  • Grounded multilingual RAG — pull the precise supporting span to cite, whatever the source language.
  • Cross-lingual lookup — thanks to the shared representation, an English question can often hit a non-English passage.
  • Private extraction — pull dates, amounts, and names out of a foreign-language document, on-device.

How the API works

The pipeline takes a question and a context and returns the best span; we read the raw logits from the same model instance to get clean, highlightable multilingual spans:

import { pipeline } from "@huggingface/transformers";

const qa = await pipeline(
  "question-answering",
  "onnx-community/xlm-roberta-base-finetuned-squad2-ONNX",
  { dtype: "q8" },
); // downloads + caches ~295 MB once

// The stock pipeline drops inter-word spaces for SentencePiece models, so we read the raw
// start_logits / end_logits off qa.model, pick the best span, and re-locate it in the ORIGINAL
// passage (whitespace-insensitive) to get a correctly-spaced, highlightable answer.
const enc = qa.tokenizer(question, { text_pair: context, padding: true, truncation: true });
const { start_logits, end_logits } = await qa.model(enc);

All of it runs in a Web Worker so the UI never janks. See worker.js for the span picker and the whitespace-insensitive re-location.
