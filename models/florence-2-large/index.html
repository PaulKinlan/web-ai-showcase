Florence-2-large — same API, sharper answers

This is the large Florence-2 checkpoint: 0.77B parameters against the base model's 0.23B. The interface is identical — one network steered by a task token to caption, detect, read text (OCR), describe regions, or segment a phrase — but the extra capacity buys noticeably richer captions, more complete detections, and better OCR. Pick an image, pick a task, and watch the bigger model run on your GPU with nothing uploaded.

Why it matters

The large checkpoint is the one you reach for when the base model's answer is close but not quite good enough — a caption that misses a detail, a detection that drops a small object, OCR that garbles a hard font. Same single download, same task-token API; you're trading download size and GPU memory for accuracy.

  • Higher-fidelity alt-text — richer, more specific descriptions for accessibility.
  • More complete detection — catches smaller and more crowded objects for moderation, inventory, or analytics.
  • Tougher OCR — reads harder fonts, angles, and low-contrast text than the base model.
  • Precise grounding — "segment the red car" lands more accurately for smart cut-out and annotation.

How the API works

Identical to the base model — only the checkpoint id changes. Build the prompt from a task token, run the processor over the image and prompt, generate, decode, then post-process: 

import { Florence2ForConditionalGeneration, AutoProcessor, load_image }
  from "@huggingface/transformers";

const model = await Florence2ForConditionalGeneration.from_pretrained(
  "onnx-community/Florence-2-large", { dtype: "fp16", device: "webgpu" });   // WebGPU required
const processor = await AutoProcessor.from_pretrained("onnx-community/Florence-2-large");

const image = await load_image(imageURL);
const task = "<OD>";                              // or <CAPTION>, <OCR>, <REFERRING_EXPRESSION_SEGMENTATION>…
const prompts = processor.construct_prompts(task);
const inputs = await processor(image, prompts);

const ids = await model.generate({ ...inputs, max_new_tokens: 512 });
const raw = processor.batch_decode(ids, { skip_special_tokens: false })[0];
const result = processor.post_process_generation(raw, task, image.size);
// result["<OD>"] = { bboxes: [[x1,y1,x2,y2], …], labels: [ … ] }

This page runs exactly that in a Web Worker, and gates on a real WebGPU adapter first — with none it shows a labelled needs-WebGPU state instead of a fake result.
