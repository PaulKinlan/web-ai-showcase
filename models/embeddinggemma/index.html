EmbeddingGemma — Google's on-device embedder

The built embedders here sit on the classic BERT encoder (MiniLM, GTE, BGE, E5) or ModernBERT. EmbeddingGemma-300m is different: it is the first embedding model on Google's Gemma 3 backbone (2025), trained for retrieval across 100+ languages, it is instruction-prompted (queries and documents get different task prefixes), and it is a Matryoshka model — its 768-d vector can be truncated to 512 / 256 / 128 and still work, so you can store 4× smaller vectors on-device. Paste passages to build a live similarity matrix, search across languages, and watch the vector shrink. Nothing leaves your device.

Matryoshka — how much accuracy survives truncation?

EmbeddingGemma is trained so the first 512 / 256 / 128 numbers of its 768-d vector are each a usable embedding on their own. Why it matters

Embeddings power search, recommendations, deduplication, clustering and retrieval-augmented generation. EmbeddingGemma's pitch is a small, multilingual, storage-flexible embedder built for the device: it runs on a phone-class footprint, retrieves across languages without a translation step, and lets you dial vector size down to 128-d when storage or compute is tight — all in a tab, all private.

  • On-device semantic search — index your notes, docs or emails locally; no vectors ever leave the machine.
  • Cross-lingual retrieval — one index, many languages; an English query finds the Spanish passage.
  • Tiny vector stores — Matryoshka-truncate to 128-d for 4× smaller indexes on constrained devices.
  • Retrieve → rerank / RAG — a fast first pass, then a cross-encoder or a local LLM (see the multi-model demo).

How the API works

Feature-extraction with mean pooling — the one thing to get right is EmbeddingGemma's task prefixes:

import { pipeline } from "@huggingface/transformers";

const extract = await pipeline(
  "feature-extraction",
  "onnx-community/embeddinggemma-300m-ONNX",
); // downloads + caches ~300 MB once

// EmbeddingGemma is instruction-prompted: wrap queries and documents differently.
const query = "task: search result | query: how do neural networks pay attention?";
const doc   = "title: none | text: Attention lets a model focus on the most relevant parts of the input…";

const out = await extract([query, doc], { pooling: "mean", normalize: true });
const [q, d] = out.tolist();

// unit-length vectors → cosine similarity is just the dot product:
const sim = q.reduce((s, x, i) => s + x * d[i], 0); // high — same meaning

// Matryoshka: the first 128 numbers are a usable smaller embedding (re-normalize after slicing).
const small = (v) => { const h = v.slice(0,128); const n = Math.hypot(...h); return h.map(x => x/n); };

This page asks the worker for the un-normalized mean vectors so "See inside" can show the real pre-normalization magnitude, then normalizes them in JS. Inference runs in a Web Worker, so the matrix, projection and Matryoshka comparison never jank the page.
